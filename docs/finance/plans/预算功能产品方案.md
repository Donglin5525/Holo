## 一、背景与目标

### 当前状况

HOLO 记账模块已实现：交易管理、多账户系统、统计分析（总览/明细/类别三 tab）、分类管理（二级分类）、导入导出、AI 对话集成。但**缺少预算管理环节**。

PRD (`docs/_common/HoloPRD.md`) 已将预算管理列为 P1 优先级：

> 预算管理 | 设置月度/周度预算，追踪执行情况 | P1

### v1 → v2 方案变更

v1 方案（独立 Tab）经对抗性审查发现 6 个严重问题（Tab Bar 布局冲突、组件不匹配、日期计算 Bug 等）。经讨论，改为**账户级预算**：

- 每个账户独立设置预算
- 嵌入 AccountDetailView，不修改 FinanceView Tab 结构
- Phase 1 仅支持总预算，Phase 2 扩展分类预算

### 目标

1. **账户级总预算**：每个账户设置月度/周度/年度支出上限
2. **实时追踪**：预算 vs. 实际支出的可视化对比
3. **预警提示**：接近或超出预算时的颜色提醒

## 三、数据模型设计

### 3.1 BudgetPeriod 枚举

```swift
// Models/BudgetPeriod.swift
enum BudgetPeriod: String, CaseIterable, Identifiable {
    case week = "week"
    case month = "month"
    case year = "year"

    var id: String { rawValue }

    var displayName: String {
        switch self {
        case .week: return "每周"
        case .month: return "每月"
        case .year: return "每年"
        }
    }

    var icon: String {
        switch self {
        case .week: return "calendar.badge.clock"
        case .month: return "calendar"
        case .year: return "calendar.badge.plus"
        }
    }
}
```

### 3.2 Budget Core Data 实体

**字段定义**：

| 字段 | 类型 | 可选 | 说明 |
|------|------|------|------|
| `id` | UUID | 否 | 唯一标识 |
| `accountId` | UUID | 否 | 所属账户 |
| `categoryId` | UUID | 是 | nil=总预算，非nil=分类预算（Phase 2） |
| `amount` | NSDecimalNumber | 否 | 预算金额 |
| `period` | String | 否 | BudgetPeriod.rawValue |
| `startDate` | Date | 否 | 预算起始日期 |
| `createdAt` | Date | 否 | 创建时间 |
| `updatedAt` | Date | 否 | 更新时间 |

**设计决策**：
- `accountId` 和 `categoryId` 均为轻量 UUID 引用（非 Core Data Relationship）
- 与 Category 的 `parentId` 模式一致，避免级联删除复杂性
- Phase 1 仅使用 `categoryId == nil`（总预算），Phase 2 扩展分类预算

### 3.3 BudgetStatus 值类型

struct BudgetStatus: Identifiable {
    let id: UUID              // budget.id

## 四、Repository 层设计

### 4.1 BudgetRepository

```swift
@MainActor class BudgetRepository {
    static let shared = BudgetRepository()
    private lazy var context = CoreDataStack.shared.viewContext
    private init() {}  // 零 I/O

    // CRUD
    func getBudgets(forAccount accountId: UUID) -> [Budget]
    func getTotalBudget(forAccount accountId: UUID, period: BudgetPeriod) -> Budget?
    func addBudget(accountId: UUID, amount: Decimal, period: BudgetPeriod, startDate: Date) throws -> Budget
    func updateBudget(_ budget: Budget, amount: Decimal?, startDate: Date?) throws
    func deleteBudget(_ budget: Budget) throws

    // 计算引擎（async，依赖 FinanceRepository）
    func computeBudgetStatus(budget: Budget) async throws -> BudgetStatus
    func computeTotalBudgetStatus(forAccount accountId: UUID, period: BudgetPeriod) async throws -> BudgetStatus?

    // 周期范围计算
    func currentPeriodRange(for budget: Budget) -> (start: Date, end: Date)
}
```

### 4.2 支出计算策略

**核心方法 `computeBudgetStatus(budget:) async throws`**：

1. 调用 `currentPeriodRange(for:)` 计算当前周期日期范围
2. 调用 `FinanceRepository.shared.getTransactions(from:to:)` 获取日期范围内交易
3. **内存过滤**（与现有 `getSubCategoryAggregations` 模式一致）：
   - `tx.account.id == budget.accountId`（仅统计该账户的交易）
   - `tx.transactionType == .expense`（仅统计支出）
4. 汇总 `tx.amount` 得到 `spentAmount`
5. 返回 `BudgetStatus`

### 4.3 周期日期范围计算（修复 v1 Bug）

v1 使用 `dateComponents([.month], from:to:)` 有月末溢出 Bug。改用**循环方式**：

```swift
func currentPeriodRange(for budget: Budget) -> (start: Date, end: Date) {
    let cal = Calendar.current
    let now = Date()
    var periodStart = budget.startDate

    // 从 startDate 开始，每次加一个周期长度，找到包含 now 的区间
    while true {
        guard let periodEnd = cal.date(byAdding: budget.budgetPeriod.calendarComponent,
                                        value: 1, to: periodStart) else {
            return (periodStart, now)
        }
        if now < periodEnd {
            return (periodStart, periodEnd)
        }
        guard let nextStart = cal.date(byAdding: budget.budgetPeriod.calendarComponent,
                                        value: 1, to: periodStart) else {
            return (periodStart, now)
        }
        periodStart = nextStart
    }
}
```

**BudgetPeriod 扩展**：
```swift
var calendarComponent: Calendar.Component {
    switch self {
    case .week: return .weekOfYear
    case .month: return .month
    case .year: return .year
    }
}
```

### 4.4 校验规则

- 每个账户每个周期最多一个总预算
- 预算金额必须 > 0
- 不允许修改 `period`（需删除重建），仅允许修改 `amount` 和 `startDate`
- 删除预算不影响任何交易数据
