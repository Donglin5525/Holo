# PRD：统计图表分析模块

## 1. 产品概述

### 1.1 背景

用户通过 HOLO 记录日常收支后，积累了大量财务数据。但单纯的交易列表无法帮助用户：
- 理解消费习惯和趋势
- 发现异常支出
- 做出更好的财务决策

统计图表模块将原始数据转化为可视化洞察，帮助用户「看懂」自己的钱。

### 1.2 目标用户

- 有记账习惯、积累了一定数据量的用户
- 希望了解自己消费结构的用户
- 想要优化支出、控制预算的用户

### 1.3 核心价值

> **从「记账」到「理财」的跨越**

不只是记录，更要理解。让用户在数据中发现规律、识别问题、指导决策。

## 2. 页面结构

### 2.1 顶部时间选择器

**支持的统计区间**：

| 区间类型 | 说明 | 默认选中 |
|---------|------|---------|
| 日 | 当天 | — |
| 周 | 本周一 ~ 今天 | — |
| **月** | 本月1日 ~ 今天 | ✅ 默认 |
| 季度 | 本季度第一天 ~ 今天 | — |
| 年 | 本年1月1日 ~ 今天 | — |
| 自定义 | 用户选择起止日期 | — |

**交互规范**：
- 切换时间范围后，所有 Tab 内的图表和列表同步刷新
- 保留上次选择的时间范围（记忆）
- 自定义区间使用日期范围选择器

---

### 2.2 顶部 Tab 导航

```
┌─────────────────────────────────────────────────────┐
│  [日] [周] [月] [季度] [年] [自定义]                 │  ← 时间选择器
├─────────────────────────────────────────────────────┤
│   总览   |   明细   |   类别                        │  ← Tab 导航
└─────────────────────────────────────────────────────┘
```

三个 Tab 分别对应不同的可视化模块，每个 Tab 采用不同的图表组件。

---

## 3. Tab 1：总览

### 3.1 核心可视化：柱状图

**布局**：右侧柱状图形式

**显示内容**：
- 收入（绿色柱）
- 支出（红色柱）
- 总计/结余（收入 - 支出的金额，单独显示或作为净值柱）

**用户问题**：这个统计周期内，我赚了多少？花了多少？还剩多少？

**场景描述**：
> 小明月底打开 HOLO，切换到「总览」Tab。一眼看到柱状图：本月收入 ¥15,000（绿色），支出 ¥12,800（红色），结余 ¥2,200。他立刻知道这个月有盈余，可以安心。

**提供的信息**：
- 收入总额
- 支出总额
- 结余金额（收入 - 支出）

**解决的问题**：
- 快速了解收支全貌
- 判断是否有盈余或赤字
- 为预算规划提供依据

---

### 3.2 下方内容：TOP3 一级类目

**布局**：三个并排的卡片区域

**显示内容**：按不同统计口径的前三个最大一级类目

| 统计口径 | 说明 | 数据来源 |
|---------|------|---------|
| 支出 TOP3 | 支出金额最高的3个一级分类 | 仅统计支出交易 |
| 收入 TOP3 | 收入金额最高的3个一级分类 | 仅统计收入交易 |
| 频次 TOP3 | 交易笔数最多的3个一级分类 | 统计所有交易 |

**每个类目卡片显示**：
- 一级分类名称
- 金额
- 占比（该分类金额 / 总金额 × 100%）

**场景描述**：
> 小红发现本月支出比预期高，她看向「支出 TOP3」区域：餐饮 ¥3,200（25%）、购物 ¥2,800（22%）、交通 ¥1,500（12%）。原来餐饮是大头，决定下个月多自己做饭。

**提供的信息**：
- 各分类的金额排名
- 占比百分比
- 快速定位「钱去哪了」

**解决的问题**：
- 快速识别主要消费去向
- 为预算分配提供依据
- 发现潜在的优化空间

---

## 4. Tab 2：明细

### 4.1 核心可视化：趋势图

**布局**：顶部趋势图区域

**显示内容**：
- X 轴：时间（根据选择的区间自动调整粒度）
  - 日区间 → 按小时
  - 周区间 → 按天
  - 月区间 → 按天
  - 季度区间 → 按周
  - 年区间 → 按月
- Y 轴：金额

**切换功能**：
- 支出明细（默认选中）
- 收入明细

**用户问题**：我的消费/收入在这个周期内是如何变化的？哪天/哪周花得最多？

**场景描述**：
> 小李想看看本月的消费趋势。他切换到「明细」Tab，默认显示「支出明细」折线图。他看到月中（15号左右）有一个明显的高峰——那是双11购物。折线图让他一眼看到消费的时间分布。

**提供的信息**：
- 随时间变化的收支走势
- 消费/收入的高峰和低谷
- 趋势方向（上升/下降/平稳）

**解决的问题**：
- 识别消费高峰时段
- 发现异常消费（突然的峰值）
- 判断整体趋势是否健康

**设计要点**：
- 折线图，带数据点标记
- 颜色区分：支出（红色）、收入（绿色）
- 点击数据点可查看当日/当周明细
- 空数据点（无交易）显示为 0 或断开

---

### 4.2 下方内容：消费明细列表

**布局**：趋势图下方，可滚动列表

**显示内容**：当前统计周期内的所有交易

**列表项信息**：
- 日期
- 分类图标 + 名称
- 金额（支出红色，收入绿色）
- 备注（如有）

**交互功能**：
- 按日期分组（天/周/月根据区间）
- 点击交易项可进入编辑
- 支持搜索/筛选（未来功能）

**场景描述**：
> 小张在趋势图上点击了15号的峰值点，下方的消费明细列表自动滚动到15号，显示当天所有交易。他看到：「京东购物 ¥299」、「淘宝购物 ¥158」... 原来是双11囤货。

**提供的信息**：
- 每笔交易的详细信息
- 与趋势图的联动交互
- 快速定位具体消费

**解决的问题**：
- 从宏观到微观的消费分析
- 点击趋势图快速定位明细
- 逐笔检查消费记录

## 5. Tab 3：类别

### 5.1 核心可视化：饼图

**布局**：居中显示的环形饼图

**显示内容**：
- 各一级分类占总支出的百分比
- 中间显示总金额
- 扇区颜色跟随分类颜色体系

**切换功能**：
- 支出（默认选中）
- 收入

**用户问题**：我的钱都花在哪些类别了？哪个类别占比最高？

**场景描述**：
> 小王想知道钱都去哪了。他切换到「类别」Tab，看到饼图：餐饮占 35%（最大扇区），购物占 28%，交通占 15%... 他点击「餐饮」扇区，进入二级分类详情，发现「外卖」占了餐饮的 70%。

**提供的信息**：
- 各分类占总体的百分比
- 直观的占比对比
- 一级分类 → 二级分类的下钻

**解决的问题**：
- 快速识别「钱去哪了」
- 发现占比过高的消费类别
- 下钻到二级分类找出具体原因

**设计要点**：
- 环形饼图，中间显示总金额
- 扇区颜色跟随分类颜色
- 小于 5% 的分类合并为「其他」
- 点击扇区进入二级分类视图

---

### 5.2 下钻功能：一级 → 二级分类

**交互流程**：

```
┌─────────────────────────────────┐
│  ← 返回                         │  ← 二级分类视图
│                                 │
│  餐饮 (35%, ¥3,200)             │  ← 一级分类标题
│                                 │
│  ┌───────────────────────────┐  │
│  │      二级分类饼图          │  │  ← 仅显示该一级分类下的二级分类
│  │                           │  │
│  │      外卖 70%              │  │
│  │      早餐 15%              │  │
│  │      晚餐 10%              │  │
│  │      其他 5%               │  │
│  └───────────────────────────┘  │
│                                 │
│  外卖     ¥2,240   70%          │  ← 二级分类列表
│  早餐     ¥480     15%          │
│  晚餐     ¥320     10%          │
└─────────────────────────────────┘
```

**返回按钮**：
- 位于左上角
- 点击返回一级分类视图

**二级分类显示**：
- 饼图仅显示该一级分类下的二级分类占比
- 下方列表显示各二级分类的金额和占比
- 占比基准为该一级分类的总金额

**场景描述**：
> 小美点击「购物」扇区，进入二级分类。她看到：「服饰 45%」、「数码 30%」、「日用 25%」。原来本月购物主要是买衣服，她想起那件冲动购买的外套。

**提供的信息**：
- 一级分类内的细分结构
- 找出具体哪个子分类花费最多
- 支持返回一级分类总览

**解决的问题**：
- 深入分析某个分类的消费细节
- 精确定位消费问题的根源
- 为精细化预算提供依据

## 6. 交互规范

### 6.1 时间切换

- 切换时间范围后，所有 Tab 内的图表和列表同步刷新
- 保留上次选择的时间范围（记忆）
- 自定义区间使用日期范围选择器

### 6.2 Tab 切换

- Tab 切换时保持当前选择的时间范围
- 每个 Tab 的图表状态独立记忆（如饼图当前是支出还是收入）

### 6.3 图表交互

- 所有图表支持点击查看详情
- 饼图支持一级 → 二级分类下钻
- 趋势图支持点击数据点定位明细

### 6.4 空状态

- 数据不足时显示引导文案
- 建议用户先记录更多数据

---

## 7. 技术约束

- 使用 Swift Charts（iOS 16+）实现标准图表
- 饼图下钻功能需自定义交互逻辑
- 大数据量时需要分页或聚合，避免卡顿
- 图表颜色遵循设计系统（与分类颜色一致）

---

## 8. 成功指标

- 用户在分析页面的停留时长 > 30 秒
- 图表交互率（点击/下钻）> 40%
- 用户反馈「对消费有了新认识」

## 9. 功能优先级

| 优先级 | 功能 | 说明 |
|-------|------|------|
| P0 | 总览 Tab（柱状图 + TOP3） | 核心概览能力 |
| P0 | 明细 Tab（趋势图 + 列表） | 趋势分析能力 |
| P0 | 类别 Tab（饼图 + 下钻） | 分类分析能力 |

| P1 | 趋势图点击定位明细 | 联动交互 |
| P1 | 饼图一级→二级下钻 | 深度分析 |
