# 财务行为洞察系统 — 实施计划

## Context

Holo 目前的 AI 上下文注入是"扁平数据"：只告诉 AI "今天花了 ¥230、最近 5 笔交易"。没有消费趋势、没有模式识别、没有跨周期对比。用户说"我这个月花太多了"，AI 无法判断多了多少、哪个分类异常。

本方案新增 **FinanceInsightEngine**，定期从 FinanceRepository 聚合数据，生成结构化的行为洞察（趋势、异常、偏好、节俭建议），并以精简摘要形式注入 AI 上下文，让 Holo "越来越懂"用户的消费模式。

## Phase 1: 数据模型 — FinanceInsight

### 1.1 FinanceInsight 结构体

新建 `Models/Finance/FinanceInsight.swift`

```swift
struct FinanceInsight: Codable {
    let generatedAt: Date
    let periodType: InsightPeriod        // daily / weekly / monthly
    let periodStart: Date
    let periodEnd: Date

    // 消费趋势
    var totalExpense: Decimal
    var totalIncome: Decimal
    var savingsRate: Double?             // (income - expense) / income
    var expenseChangePercent: Double?    // vs 上一周期
    var incomeChangePercent: Double?

    // 分类偏好 (TOP 5)
    var topExpenseCategories: [CategoryInsight]

    // 消费节奏
    var averageDailyExpense: Decimal
    var maxDailyExpense: Decimal
    var maxDailyExpenseDate: Date?
    var weekdayPattern: [Int: Decimal]   // 1=Mon...7=Sun, 平均日消费

    // 异常检测
    var anomalies: [SpendingAnomaly]

    // 预算表现
    var budgetPerformance: BudgetPerformance?

    // AI 摘要 (自然语言，注入 prompt 用)
    var summary: String
}
```

### 1.2 辅助类型

```swift
struct CategoryInsight: Codable {
    let categoryName: String
    let icon: String
    let amount: Decimal
    let percentage: Double
    let changePercent: Double?   // vs 上一周期
}

struct SpendingAnomaly: Codable {
    let type: AnomalyType        // spike / categoryShift / unusualDay
    let description: String      // "周六消费 ¥800，高于均值 320%"
    let severity: Double         // 0-1，用于排序
}

enum AnomalyType: String, Codable {
    case spike            // 单日消费飙升
    case categoryShift    // 分类占比突变
    case unusualDay       // 非常规日期大额消费
}

struct BudgetPerformance: Codable {
    let totalBudget: Decimal
    let totalSpent: Decimal
    let progressPercent: Double
    let isOnTrack: Bool
    let warningCategories: [String]  // 超预算分类名
}

enum InsightPeriod: String, Codable {
    case daily, weekly, monthly
}
```

### 1.3 存储策略

- **不存 Core Data** — 洞察是可重新计算的派生数据
- 存 `UserDefaults` 或 `Application Support/Holo/FinanceInsights/` JSON 文件
- 保留最近 1 条 daily + 1 条 weekly + 1 条 monthly（共 3 份）
- 每次重新生成时覆盖旧值

## Phase 2: 计算引擎 — FinanceInsightEngine

新建 `Services/Finance/FinanceInsightEngine.swift`

### 2.1 核心接口

```swift
@MainActor final class FinanceInsightEngine {
    static let shared = FinanceInsightEngine()

    // 获取缓存的洞察（注入上下文时调用）
    func getCachedInsight(period: InsightPeriod) -> FinanceInsight?

    // 生成新洞察（后台触发）
    func generateDailyInsight() async throws -> FinanceInsight
    func generateWeeklyInsight() async throws -> FinanceInsight
    func generateMonthlyInsight() async throws -> FinanceInsight

    // AI 摘要文本（注入 prompt 用）
    func insightSummaryForContext() -> String
}
```

### 2.2 计算逻辑（全部从 FinanceRepository 获取数据）

#### 消费趋势

| 指标 | 计算方式 | 数据源 |
|------|----------|--------|
| `totalExpense` / `totalIncome` | 累加当前周期 | `getTransactions(from:to:)` |
| `savingsRate` | `(income - expense) / income` | 同上 |
| `expenseChangePercent` | `(current - previous) / previous * 100` | 对比上一周期 |
| `averageDailyExpense` | `totalExpense / dayCount` | 同上 |

#### 分类偏好

- 调用已有的 `FinanceRepository.getTopLevelCategoryAggregations(from:to:type:)`
- 取 TOP 5 支出分类
- 对比上一周期计算 `changePercent`

#### 消费节奏

- 按星期几分组计算平均日消费 → `weekdayPattern`
- 找最高消费日 → `maxDailyExpense` + `maxDailyExpenseDate`

#### 异常检测（纯规则，不用 ML）

| 异常类型 | 规则 |
|----------|------|
| `spike` | 单日消费 > 过去 30 天日均的 3 倍 |
| `categoryShift` | 某分类占比 vs 上周期变化 > ±15pp |
| `unusualDay` | 工作日消费 > 该周均值的 2 倍 |

#### 预算表现

- 调用已有的 `BudgetRepository.computeGlobalTotalBudgetStatus(period:)`
- 调用 `getWarningCategoryBudgets(period:)`
- 组装 `BudgetPerformance`

### 2.3 AI 摘要生成

`insightSummaryForContext()` 方法将结构化数据转为精简自然语言文本，供 prompt 注入：

```
【消费洞察】本月支出 ¥12,340（较上月 ↑18%）。储蓄率 32%。
TOP3: 餐饮 38%↑5pp、购物 22%↓3pp、交通 12%→。
日均 ¥411，周六消费最高（均值 ¥680）。
⚠️ 4/12 餐饮单日 ¥520（高于均值 320%）。
预算进度 78%，按趋势可持平。
```

控制 **150 字以内**，避免挤占 prompt token 预算。

## Phase 3: 触发机制

### 3.1 何时重新计算

| 洞察类型 | 触发时机 | 实现方式 |
|----------|----------|----------|
| Daily | 每天首次打开 App + 记账后 | `ScenePhase` 从 background → active 时检查日期；`FinanceRepository` 交易变更通知 |
| Weekly | 周一首次打开 App | 同上 + 判断是否新的一周 |
| Monthly | 月首首次打开 App | 同上 + 判断是否新的月份 |

### 3.2 触发入口

在 `HoloApp.swift` 或 `ContentView.swift` 的 `.task` / `.onChange(of: scenePhase)` 中调用：

```swift
// 伪代码
if isNewDay {
    Task { await FinanceInsightEngine.shared.generateDailyInsight() }
}
if isNewWeek {
    Task { await FinanceInsightEngine.shared.generateWeeklyInsight() }
}
if isNewMonth {
    Task { await FinanceInsightEngine.shared.generateMonthlyInsight() }
}
```

## Phase 4: Prompt 注入

### 4.1 修改 UserContextBuilder

在 `Services/AI/UserContextBuilder.swift` 的 `UserContext` 中新增字段：

```swift
struct UserContext {
    // ... 现有字段 ...
    var financeInsight: FinanceInsightContext?  // 新增
}

struct FinanceInsightContext {
    var daily: String?    // 日洞察摘要
    var weekly: String?   // 周洞察摘要
    var monthly: String?  // 月洞察摘要
}
```

在 `buildContext()` 中收集：

```swift
let financeInsight = FinanceInsightContext(
    daily: FinanceInsightEngine.shared.getCachedInsight(period: .daily)?.summary,
    weekly: FinanceInsightEngine.shared.getCachedInsight(period: .weekly)?.summary,
    monthly: FinanceInsightEngine.shared.getCachedInsight(period: .monthly)?.summary
)
```

### 4.2 修改 OpenAICompatibleProvider

在 `buildContextMessage()` 中注入洞察摘要：

```swift
// 在现有上下文消息末尾追加
if let fi = context.financeInsight {
    if let daily = fi.daily { result += "\n- 今日消费洞察: \(daily)" }
    if let weekly = fi.weekly { result += "\n- 本周消费趋势: \(weekly)" }
    if let monthly = fi.monthly { result += "\n- 本月消费分析: \(monthly)" }
}
```

### 4.3 增强 insightGeneration Prompt

将现有的 150 字 prompt 扩展为带结构化输出要求的 prompt：

```
你是 Holo 数据分析助手。基于用户的消费数据和行为洞察，生成个性化分析报告。

当前日期：{{todayDate}}

规则：
- 用中文回复，使用 Markdown 格式
- 数据必须来自提供的洞察，不要编造数字
- 对比要有参照（"较上月↑18%"而非"花了很多"）
- 给出 1-2 条可执行建议
- 语气积极正面，避免说教

输出格式：
## 消费概览
（总额、储蓄率、趋势）

## 分类分析
（TOP3 分类变化）

## ⚠️ 注意事项
（异常消费、预算预警，无异常则省略）

## 💡 建议
（1-2 条可执行建议）
```

## Phase 6: 验证方案

1. **单元测试** — `FinanceInsightEngine` 的各计算方法：
   - 趋势计算准确性（mock 交易数据）
   - 异常检测规则触发/不触发
   - 分类对比计算正确性
   - 摘要文本长度 < 150 字

2. **集成测试** — 端到端验证：
   - 注入上下文后，AI 能正确引用洞察数据回答财务问题
   - "我这个月花了多少" → AI 回答包含趋势对比
   - "我最近消费正常吗" → AI 引用异常检测结果

3. **Token 预算验证** — 确认注入后上下文消息不超过合理范围
   - 当前上下文约 500-800 tokens
   - 新增洞察摘要 ~100-150 tokens (日+周+月)
   - 总增幅控制在 200 tokens 以内
