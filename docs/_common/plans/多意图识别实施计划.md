# 多意图识别实施计划

## Context

当前 AI Chat 是单意图架构：一条消息 → 一个 `ParsedResult` → 一次路由 → 一张卡片。无法处理用户一句话包含多个操作的场景，如：
- 「今天加油200，停车费20」→ 2 笔记账
- 「提醒我今天中午抢票，刚买早饭5块钱」→ 1 个任务 + 1 笔记账

本计划升级为多意图架构，同时完全向后兼容现有单意图消息。

struct MultiParsedResult: Codable {

## Phase 1: Prompt + JSON 解析

> 风险：中高 | LLM 输出不确定性，需三层 JSON 解析回退

### 1.1 更新意图识别 Prompt
**文件**: `HOLO/Holo/Holo APP/Holo/Holo/Services/AI/PromptManager.swift`

修改 `intentRecognition` 模板的 JSON 输出格式：

**旧格式**（单个对象）：
```json
{ "intent": "...", "confidence": 0.9, "extractedData": {...} }
```

**新格式**（数组包装）：
```json
{
  "results": [
    { "intent": "record_expense", "confidence": 0.95, "extractedData": { "amount": "200", "note": "加油", ... }, ... },
    { "intent": "record_expense", "confidence": 0.93, "extractedData": { "amount": "20", "note": "停车费", ... }, ... }
  ]
}
```

关键 prompt 变更：
- 添加「多意图规则」段落：当输入包含多个独立操作时，为每个操作返回一个 results 条目
- 单意图也用数组格式返回（单元素数组）
- 添加 2-3 个多意图示例
- 保留所有现有意图定义、类别表、日期解析规则

### 1.2 更新 JSON 解析逻辑
**文件**: `HOLO/Holo/Holo APP/Holo/Holo/Services/AI/OpenAICompatibleProvider.swift`

新增 `parseMultiResultFromJSON` 方法，三层回退：
1. 尝试解码为 `MultiParsedResult`（`results` 数组格式）
2. 回退：解码为单个 `ParsedResult`，包装成 `MultiParsedResult(single:)`
3. 最终回退：低置信度 unknown 结果（现有行为）

`extractJSON` 辅助方法无需修改。

### 验证
用 PromptTestSheet（设置页现有工具）测试目标场景在 DeepSeek/GLM-4/Qwen 上的 JSON 输出。

## Phase 3: ViewModel 多意图执行管道

> 风险：中 | 核心行为变更，关键路径

### 3.1 重构 sendMessage
**文件**: `HOLO/Holo/Holo APP/Holo/Holo/Views/Chat/ChatViewModel.swift`

`sendMessage()` 中意图识别后的逻辑改为：

```
let multiResult = try await provider.parseUserInput(text, context: userContext)

// 全部高置信度才处理
let allHighConfidence = multiResult.results.allSatisfy { $0.isHighConfidence }

if allHighConfidence && !multiResult.results.isEmpty {
    // 单个 query → 走流式聊天（保持不变）
    if multiResult.results.count == 1 && multiResult.first?.intent == .query {
        // 现有流式聊天路径
    } else {
        // 遍历路由每个意图
        var routeResults: [RouteResult] = []
        var allMergedData: [[String: String]] = []

        for parsedResult in multiResult.results {
            switch parsedResult.intent {
            case .query, .unknown:
                continue  // 多意图中跳过 query/unknown
            default:
                let routeResult = try await IntentRouter.shared.route(parsedResult)
                routeResults.append(routeResult)
                // 构建 mergedData（同现有逻辑：extractedData + entity IDs）
                allMergedData.append(mergedData)
            }
        }

        // 生成组合文本
        let combinedText = routeResults.count > 1
            ? "已为你完成 \(routeResults.count) 件事：\n" + 路由结果列表
            : routeResults.first?.text ?? 默认文本

        // 存储：旧字段存第一个结果（向后兼容）+ 新字段存全部
        chatRepo?.updateMessageMetadata(
            aiMessageId,
            intent: multiResult.first?.intent.rawValue,
            extractedDataJSON: encodeExtractedData(allMergedData.first),
            multiResultsJSON: encodeMultiResults(allMergedData)
        )
    }
} else {
    // 低置信度回退（现有逻辑）
}
```

新增辅助方法：
```swift
private static func encodeMultiResults(_ data: [[String: String]]) -> String?
```

### 关键设计决策
- **部分置信度**：任一结果低置信度 → 整体回退（保守策略）
- **混合 query + 操作**：多意图中跳过 query，仅路由操作意图
- **空结果数组**：视为低置信度 unknown
- **路由失败**：错误文本包含在组合响应中，其他成功路由照常创建实体

### 验证
- 单意图消息行为完全不变
- 多意图消息：验证所有实体创建成功 + 组合文本正确

### 单元测试

| 测试文件 | 测试内容 |
|---------|---------|
| `MultiParsedResultTests.swift` (新) | MultiParsedResult 解码、isMultiIntent、空数组 |

| `ChatCardDataTests.swift` (更新) | multiFrom：混合意图、空数组、数量不匹配 |

### 需覆盖的边缘情况
- 多意图中某个路由失败（如任务不存在）→ 部分成功
- 空结果数组 `{"results": []}` → 视为 unknown
- LLM 返回旧格式单对象 → 自动回退包装
- 多张卡片中某张无 linkedEntityId → 该卡片不可点击
