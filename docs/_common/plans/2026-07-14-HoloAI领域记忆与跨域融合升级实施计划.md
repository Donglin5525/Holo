# HoloAI 领域记忆与跨域融合升级 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** 把现有长期/情景记忆升级为“模块独立萃取、统一存储治理、跨域融合、按问题精准召回”的记忆体系，并让普通用户只感知 Holo 更懂自己、开发者可显性验证完整链路。

**Architecture:** 采用方案 B：各领域拥有独立 Signal Builder/Extractor，但统一输出 `HoloMemoryRecord`；普通记忆写入现有 CloudKit 私有库，健康及包含健康证据的派生记忆进入本机敏感 Store；所有读取强制经过 `HoloMemoryAccessPolicy` 与 `HoloMemoryQueryService`；开放式问题使用 stale-while-revalidate，明确数字问题仍查明细；跨域记忆只能引用已验证的领域记忆与底层证据 lineage。

**Tech Stack:** Swift 6、SwiftUI、Core Data、NSPersistentCloudKitContainer、actor、BackgroundTasks、Node.js/Hono、后端服务端 Prompt、standalone `swiftc` 测试、Node `--test`。
