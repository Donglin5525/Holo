## 三、架构设计要点

### 确定性优先
- 时间、预算、权限、危险动作和可复算指标不交给模型自由决定
- Semantic Frame、Coverage Check、Verifier、Policy 均为纯确定性逻辑，不调用 LLM

### 不重建 Agent
- 所有新组件复用现有 Runtime、Evidence Ledger、Memory Record、Observer 底座
- V1 Verifier 保留，V2 作为升级版按 TaskProfile.requiresVerifier 启用
- AgentPolicyContext 复用 HoloMemoryRecord + InsightPreferenceProfile，不新增存储

### 防护机制
- 简单查数走 fast path，不增加 LLM 轮次
- 工具能力软选择 + 一次扩展回退，不用硬领域门控
- 未授权信号永不 notify（默认 store）
- 效果回看不写因果，只描述前后变化

## 四、后端发版同步

**P0-D 包含 HoloBackend 改动，需要后端发版：**

- **改动内容**：`src/providers/openAICompatibleProvider.js` 的 `completeViaStream()` 方法加固
- **是否需要发版**：✅ **需要**
- **影响范围**：
  - SSE 流结束时 flush decoder 并处理残余 buffer（修复多字节 UTF-8 尾字节丢失）
  - 坏帧计数而非静默跳过（增加可观测性）
  - 内容完整性不确定时返回 `finish_reason: "length"` 或抛 `UPSTREAM_INCOMPLETE`，不再合成 `"stop"` 掩盖异常
  - 增加 `_sseStats` 非敏感元数据（帧数/坏帧/残余 buffer/finish_reason/usage）
- **兼容性**：返回结构与非流式 `complete()` 一致，下游无感；仅在异常时附加 `_sseStats` 字段
- **测试**：12 条 SSE 可靠性测试全部通过

## 六、主流程接入（已完成）

所有新组件已接入生产代码路径，生产 Agent 实际具备新能力：

### 接入点 1：HoloAgentResponseParser → Contract Policy
- **文件**：`HoloAgentResponseParser.swift`
- **改动**：`parse()` 的两个 `return output` 路径现在都经过 `applyContractPolicy()` 校验
- **效果**：空 final_claims、事实 claim 无 evidence、confidence 越界、空 displayText 等关键违规不再静默放过；Debug 构建下任何违规都失败，生产仅关键违规拒绝
- **证明测试**：`HoloAgentIntegrationWiringTests.testParser拒绝空final_claims` + `testParser拒绝无evidence的claim`

### 接入点 2：startAnalysisJob → SemanticFrame + BudgetSelector
- **文件**：`HoloLocalAgentRuntime.swift` L74-82
- **改动**：`startAnalysisJob` 现在调用 `HoloAgentSemanticFrameBuilder.buildFrame()` 构建 Semantic Frame，并用 `HoloAgentBudgetSelector.selectConfig()` 选择预算
- **效果**：预算不再是硬编码 `normalDeep`，而是根据 TaskProfile 动态选择（simpleLookup 用更少 token，crossDomainAnalysis 可用 extendedDeep）
- **证明测试**：`HoloAgentIntegrationWiringTests.testBudgetSelector驱动预算`

### 接入点 3：completeWithClaims → V2 Verifier（按 TaskProfile 路由）
- **文件**：`HoloLocalAgentRuntime.swift` L1211-1284
- **改动**：通过 `classifyTaskProfile(for:)` 判断 `requiresVerifier`，复杂任务启用 `HoloClaimVerifierV2`（十维校验 + 三态 verified/degraded/rejected），简单查数保留 V1 轻量路径
- **效果**：degraded claim 带系统生成的降级披露文案展示，rejected claim 不展示
- **证明测试**：`HoloAgentIntegrationWiringTests.testTaskProfile驱动Verifier选择`

### 接入点 4：runLoop → Clarification + CoverageChecker
- **文件**：`HoloLocalAgentRuntime.swift` L309-327（循环前歧义检测）+ L1285-1320（覆盖补齐）
- **改动**：
  - 进入 LLM 循环前检测高影响歧义（如"帮我看看"），将结构化澄清请求作为系统提示注入对话
  - metric 补齐逻辑现在使用正式的覆盖检查，补齐后仍缺失的指标通过 `capability_boundary` claim 披露，不假装有结论
- **效果**：未完成计划的子问题有可审计状态（answered/unsupported/missing），不再伪装成完整答案
- **证明测试**：`HoloAgentIntegrationWiringTests.testClarification检测高影响歧义` + `testCoverageChecker检测缺失指标`
