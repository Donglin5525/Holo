# HoloAI 记忆模拟器全链路验收设计

## 1. 目标

在 iOS 模拟器中使用固定 mock 数据，一次性跑通领域记忆的生成、持久化、用户反馈、查询路由、回答上下文和忘记阻断链路。验收手段不增加 App 内页面、按钮或普通用户功能，不使用真实用户数据，不连接 iCloud。

## 2. 已确认的产品边界

- 不开发“一键验收”页面或实验室功能。
- 不改变现有记忆管理、Holo 提问和用户反馈交互。
- mock 仅在 `DEBUG + iOS Simulator + 显式环境变量` 三个条件同时满足时启动。
- Release 不编译 mock runner、启动分支、mock 文案和报告生成器。
- 模拟器验收仓库必须使用独立的本地 Core Data Store，不挂载 CloudKit options，不读写 App 正常记忆 Store。

## 3. 方案

### 3.1 启动方式

使用 `xcrun simctl launch` 向 Debug 包注入环境变量：

- `HOLO_MEMORY_SIMULATOR_SCENARIO=full-chain-v1`
- `HOLO_MEMORY_SIMULATOR_RESET=1`

`HoloMemoryRuntime` 仅在此条件下切换到模拟器专用的本地 Store。`RESET=1` 在初始化 Store 前删除上一轮测试目录，保证每次结果可重复。

### 3.2 mock 数据

固定生成以下数据，所有 ID 带 `sim-validation-v1` 命名空间：

1. 财务 active 记忆：近 14 天多次麦当劳消费，具有 aggregate evidence 和 merchant anchor。
2. 健康 active 记忆：近期多日入睡较晚，具有独立 health lineage。
3. 习惯 candidate 记忆：减脂习惯仍在尝试期，用于验收“准确”后转 active。
4. 可纠正记忆：用于验收稳定 ID、版本链和纠正后文本。
5. 可拒绝记忆：用于验收 suppressed 不再被 Query Service 选中。
6. 可忘记记忆：用于验收 tombstone、正文擦除与同义重建阻断。
7. 跨域记忆：使用财务、健康和习惯的独立 evidence，验收共同时间、共同 anchor、跨领域和独立 lineage 四道门。

### 3.3 真实链路

mock runner 不直接拼装最终 UI 结果，而是依次调用现有生产组件：

1. `HoloDomainObservationPackageBuilder`
2. `HoloDomainMemoryOutputValidator`
3. `HoloDomainMemoryObservationApplier`
4. `CoreDataHoloMemoryRepository`
5. `HoloMemoryFeedbackService`
6. `HoloCrossDomainCandidateBuilder` 和现有融合校验链
7. `HoloMemoryQueryService`
8. `HoloMemoryContextEnvelope`

这样既避免真实外部 AI 调用，又保留 Validator、Store、反馈、路由和上下文的真实行为。后端 mock 固定包含 `requestedActions:null`，直接覆盖本次真机暴露的契约问题。

### 3.4 现有 UI 验收

模拟器 App 仍打开现有界面，不增加验收界面：

- `设置 → AI 记忆 → 记忆管理`：检查 mock 记忆分组、正文、时间和来源。
- 记忆详情：使用现有“准确”“不准确”“纠正”“不再使用”按钮。
- `AI 记忆实验室 → 查询模拟器`：检查 holistic/domain/detail 路由、选中 ID、detail fallback 和最终上下文。
- Holo 对话：使用模拟器启动参数将 rollout 设为内部账号，验收记忆上下文进入实际回答链；明细问题仍以 Tool/原始数据为权威。

## 4. 自动断言与报告

runner 生成一份 JSON 报告，存放在模拟器 App 容器的 `Documents/HoloMemoryValidation/full-chain-v1.json`，仅包测试 ID、状态、路由和断言结果，不包含真实用户数据。

必须通过的断言：

- `requestedActions:null` 通过 Validator，非空 requested actions 被拒绝。
- 领域记忆按 active/candidate 预期入库，观察 key 幂等。
- “准确”使 candidate 转 active。
- “纠正”保留 stable ID，版本 +1，新文本成为 active。
- “不准确”变 suppressed，不再被查询选中。
- “不再使用”生成 tombstone，清空正文/证据，同义再写入被拒绝。
- `我最近状态如何` 路由到 holistic memory，可选中跨域/领域 active 记忆。
- `我的财务状态怎么样` 路由到 domain memory，只选中符合条件的 active 记忆。
- `最近14天吃了多少麦当劳` 必须 `detail fallback = true` 且 `authority = backgroundOnly`。
- 记忆辅助关闭时选中 ID 和模型上下文均为空。
- 最终模型上下文只包含实际选中的 active 记忆，不包含 suppressed/forgotten 内容。

任一断言失败时 runner 必须输出具体阶段和预期/实际值，不使用统一的“验收失败”。

## 5. 执行与证据

1. 编译 Debug iOS Simulator 包。
2. 启动模拟器并安装 App。
3. 使用环境变量启动 `full-chain-v1`。
4. 读取 JSON 报告并逐项校验。
5. 在现有记忆管理、查询模拟器和 Holo 对话中完成 UI 与上下文核对。
6. 截图保留关键界面，但不将模型原始输出或敏感正文写入普通 Trace。
7. 编译 Release iPhoneOS 包并扫描 mock 触发字符串、测试文案和 runner 类型，必须全部不存在。
8. 卸载模拟器 App，清理测试 Store 和报告。

## 6. 完成标准

- 不新增用户可见功能或实验室按钮。
- 所有自动断言通过，且报告不含真实用户数据。
- 现有记忆管理能展示和操作 mock 记忆。
- Holo 通用问题能引用符合条件的 active 记忆，明细问题仍走原始数据。
- 关闭记忆辅助后立即停止查询和注入。
- Release 二进制无 mock runner 和验收文案。
- 模拟器测试完成后无残留 Store、报告或后台任务。
