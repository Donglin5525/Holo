# Agent 后台能力生效验证流程

更新时间：2026-06-28

## 结论

当前代码已具备「全局 Agent Runtime + Scheduler 恢复链 + iOS 后台短时续跑」能力，但仍处于灰度开关控制下：

- `agentRuntimeEnabled` 关闭时，线上用户不会触发 Agent Runtime。
- 开启后，Holo AI 对话深度分析与 Observer Tier2 会统一经 `HoloAgentScheduler` 创建/恢复 job。
- 记忆长廊的「深度分析」只读取最近 Agent Result，不直接创建 Agent job。
- 传统 `MemoryInsightService` / `HealthInsightGenerationService` 仍是独立洞察生成链路，不属于本次 Agent Runtime 覆盖范围。

产品口径：Agent 运行时建议用户停留在前台；切后台后 App 会利用 iOS 给的短时间继续尝试，系统回收后台时间或用户杀掉 App 后，未完成 job 会落盘，回到 App 后由 Scheduler 继续。

## 覆盖范围

| 场景 | 当前接入方式 | 验收口径 |
| --- | --- | --- |
| Holo AI 分析类问题 | `ChatViewModel` -> `HoloAgentAnalysisService` -> `HoloAgentScheduler.start` | 新 job 到达终态，结果写入 `agentResultJSON` |
| Observer Tier2 | `HoloMemoryObserverService` -> `HoloAgentAnalysisService` -> `HoloAgentScheduler.start` | 目标信号触发时不绕过总闸 |
| 记忆长廊深度分析 | `MemoryGalleryViewModel` 读取 `HoloLocalAgentRuntime.loadLatestResult()` | 只消费 Agent Result，不创建平行 job |
| App 切后台 | `HoloBackgroundContinuationManager.appDidEnterBackground()` | 不立刻 pause，先保留 iOS 后台任务继续跑 |
| 后台时间到期 | UIBackgroundTask expiration handler | 标记 `waitingForForeground`，等待回前台恢复 |
| App 回前台 | `HoloBackgroundContinuationManager.appWillEnterForeground()` | `resumeAndContinue` 重启 runLoop，而不是只改状态 |

## 手动验证流程

1. 打开设置页，开启 `Agent 深度分析引擎`；需要验证 Observer 时再开启 `后台观察自动深挖`。
2. 在 Holo AI 输入分析类问题，例如「帮我分析最近一周消费和习惯有什么关联」。
3. 看到 Chat 内出现 Agent 深度分析 loading 卡，确认文案提示用户停留前台，并说明切后台风险。
4. 保持前台直到完成，确认卡片可展开，消息落盘后 `agentResultJSON` 有内容。
5. 再发起一次分析，loading 出现后立刻切到后台 10-20 秒，再回前台。
6. 预期：若系统允许后台时间，任务可继续完成；若未完成，回前台后 Scheduler 会继续推进。
7. 再发起一次分析，loading 出现后上划杀掉 App，等待 5-10 秒后重新打开。
8. 预期：未完成 job 不应永久停在 `running`；回前台后 `resumeAndContinue` 会扫描非终态 job 并继续 runLoop。
9. 打开记忆长廊，确认「深度分析」区域展示最近 Agent Result；如果没有 Agent Result，应回退到旧 AI 回放，不应空白或崩溃。
10. 开启 Observer Tier2 后制造目标信号，确认触发链路仍受 `agentRuntimeEnabled` 总闸约束。

## 日志与落盘检查

关键日志：

- `[Agent] 经 Scheduler 启动 runLoop…`
- `[Agent] runLoop 完成 state=... rounds=...`
- `Observer 触发 Tier2 深度分析`

关键落盘状态：

- `running` / `waitingForLLM`：正在推进中。
- `waitingForForeground`：后台时间耗尽或被暂停，等待回前台恢复。
- `completed`：已完成并可展示。
- `failed`：需要查看 `errorSummary`，常见原因是 prompt 加载、网络或后端 `agent_loop` 返回格式。

## 不通过判定

出现以下任一情况，本能力不能判定为正式生效：

- `agentRuntimeEnabled` 开启后，Holo AI 分析类问题未创建 Agent job。
- 回前台后只把 job 状态改成 `running`，但没有重启 `runLoop`。
- 切后台瞬间立刻把所有 running job 改成 `waitingForForeground`。
- App 被杀重启后，非终态 job 永久停留在 `running` / `waitingForForeground`。
- Observer Tier2 绕过 `agentRuntimeEnabled` 总闸。
- UI 文案承诺「后台一定完成」或没有提示切后台失败风险。
