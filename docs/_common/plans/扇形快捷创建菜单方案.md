# 扇形快捷创建菜单 - 实现方案

## Context

首页中央的 `VoiceAssistantButton`（橙色渐变圆形 + mic.fill 图标）目前点击只能打开 ChatView。用户希望增加**向上滑动手势**，在手指拖动过程中以扇形展开 4 个快捷创建选项（观点/Todo/习惯/记账），松手即跳转对应新建页面，取消则无操作。

当前创建流程需要两步导航（先进模块 fullScreenCover → 再点 + 号 sheet），这个手势快捷方式可以一步直达。

---

## 交互设计

**拖动选择模式**（一次连续手势完成选择）：

1. 用户触摸中央按钮并**向上拖动超过 30pt** → 扇形菜单以弹性动画展开
2. 4 个选项以半圆弧形排列在按钮上方（角度 -150° 到 -30°，半径 ~130pt）
3. 手指滑动时，靠近的选项自动高亮 + 触觉反馈（`HapticManager.selection()`）
4. **在高亮选项上松手** → 菜单关闭，弹出对应的创建 sheet
5. **拖回按钮区域或未命中任何选项松手** → 菜单关闭，无操作
6. **短按（移动 < 15pt）** → 保持原有行为，打开 ChatView

## 文件变更

### 1. 新建 `QuickCreateFanMenu.swift`

**路径**: `HOLO/Holo APP/Holo/Holo/Components/QuickCreateFanMenu.swift`

定义 `QuickCreateTarget` 枚举和扇形菜单覆盖层组件：

```swift
enum QuickCreateTarget: Int, CaseIterable {
    case thought, task, habit, transaction
    var title: String  // "观点"/"待办"/"习惯"/"记账"
    var icon: String   // SF Symbol（需验证存在性）
    var color: Color   // 每个选项的品牌色
}
```

`QuickCreateFanMenu` 视图：
- **输入参数**: `progress`（0-1 展开进度）、`fingerLocation`（手指位置）、`center`（按钮中心）、`selectedTarget`（当前高亮项）
- **布局**: 4 个选项以扇形排列，从按钮中心向外弹出
- **视觉**: 毛玻璃圆形背景 + 白色 SF Symbol 图标 + 底部文字标签
- **高亮态**: 选中项 scale 1.2 + 发光阴影，其余项 scale 0.85 + 透明度 0.5
- **动画**: `spring(response: 0.35, dampingFraction: 0.75)` 展开，交错 30ms

### 2. 修改 `VoiceAssistantButton.swift`

**路径**: `HOLO/Holo APP/Holo/Holo/Components/VoiceAssistantButton.swift`

核心变更：
- 移除 `Button` 包装，改用 `ZStack` + 手势识别器
- 新增 `onTap` 和 `onDragCreate: (QuickCreateTarget) -> Void` 两个回调
- 新增状态变量：`@GestureState isDragging`、`@State fanProgress`、`@State selectedTarget`、`@State fingerOffset`
- 手势逻辑：
  - `DragGesture(minimumDistance: 30)` 追踪拖动
  - 当 `translation.height < -30`（向上）时激活扇形菜单
  - 实时计算手指位置到各选项的距离，判断高亮项
  - 松手时判断：有选中项 → 调用 `onDragCreate`；无选中项 → 取消
  - `TapGesture().simultaneousGesture` 处理短按
- 扇形菜单作为 overlay 叠加在按钮上方
- 拖动时暂停呼吸动画，按钮缩小到 0.92

### 3. 修改 `HomeView.swift`

**路径**: `HOLO/Holo APP/Holo/Holo/Views/HomeView.swift`

变更：
- 新增 4 个 `@State` 变量：
  ```swift
  @State private var showQuickAddTask = false
  @State private var showQuickAddTransaction = false
  @State private var showQuickAddHabit = false
  @State private var showQuickAddThought = false
  ```
- 更新 `VoiceAssistantButton` 调用：
  ```swift
  VoiceAssistantButton(
      onTap: { showChatView = true },
      onDragCreate: { target in handleQuickCreate(target) }
  )
  ```
- 新增 `handleQuickCreate(_ target:)` 方法，根据 target 设置对应 `@State`
- 新增 4 个 `.sheet` 修饰符：
  ```swift
  .sheet(isPresented: $showQuickAddThought) {
      ThoughtEditorView()
  }
  .sheet(isPresented: $showQuickAddTask) {
      AddTaskSheet(repository: TodoRepository.shared, existingTask: nil)
  }
  .sheet(isPresented: $showQuickAddHabit) {
      AddHabitSheet()
  }
  .sheet(isPresented: $showQuickAddTransaction) {
      AddTransactionSheet(editingTransaction: nil, presetDate: nil) { }
  }
  ```

## 触觉反馈

| 时机 | 调用 |
|------|------|
| 扇形展开（拖动超 30pt） | `HapticManager.medium()` |
| 手指滑入新选项区域 | `HapticManager.selection()` |
| 在选项上松手确认 | `HapticManager.success()` |
| 取消关闭 | 无反馈 |

---

## 边界情况处理

- **点击 vs 拖动**: `minimumDistance: 30` 区分，< 15pt 视为点击
- **与五角形图标拖拽冲突**: VoiceAssistantButton zIndex 更高且在 ZStack 上层，手势不会穿透
- **小屏幕适配**: 用 `GeometryReader` 读取按钮实际位置，距顶部 < 200pt 时扇形向下展开
- **场景切换/来电**: `@GestureState` 自动重置，扇形关闭
- **CLAUDE.md 合规**: 不涉及 ScrollView，SwiftUI DragGesture 可用；SF Symbol 名称实现前需验证存在性

## 验证方式

1. **短按按钮** → ChatView 正常弹出（原有行为不受影响）
2. **向上拖动** → 4 个选项扇形展开，动画流畅
3. **滑到选项上松手** → 对应创建 sheet 弹出
4. **拖回取消** → 菜单平滑收回，无任何操作
5. **五角形图标** → 长按拖拽排序不受影响
6. **不同屏幕尺寸** → 扇形方向自适应
