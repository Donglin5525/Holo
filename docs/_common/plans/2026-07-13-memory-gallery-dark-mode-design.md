# 记忆长廊 Dark Mode 适配方案

## 产品结论

记忆长廊当前并非整体没有适配 Dark Mode，而是少数日历与热力图组件绕过了设计系统，直接使用固定浅色。结果是深色界面中出现大面积亮白控件或浅色方格，破坏阅读连续性，也让切换器看起来像未完成状态。

本次采用中等范围修复：覆盖记忆长廊完整访问链路中的固定浅色问题，不扩展到全 App，也不改变已有布局、交互、数据逻辑和品牌色。

## 适配范围

### 1. 日历切换器

- 周历 / 月历切换器的容器背景改用支持 Dark Mode 的语义色。
- 网格视图 / 列表视图切换器同步处理。
- 选中态继续使用 Holo 品牌橙色，未选中文字继续使用语义次级文字色。

### 2. 月历热力格

- 保留现有 0 到 4 级活跃度映射和冷静蓝色视觉方向。
- 为每一级增加独立的深色模式色阶，避免深色背景中出现浅蓝白块。
- 空白日期与非本月日期使用适合当前界面的动态背景，不再依赖固定浅色十六进制值。
- 今天、选中态、模块底条的含义和交互保持不变。

### 3. 记忆长廊热力图

- 修复热力图中同类固定浅色底格。
- 保留当前活跃度层级与橙色系表达，只补齐深色模式的背景层级。

## 明确保留

- 品牌橙色按钮上的白色文字属于正确的高对比设计，不修改。
- 彩色头图、模块色条、状态色和品牌装饰色不因 Dark Mode 改变语义。
- 不修改周历动态时间轴、洞察卡片业务逻辑和当前工作区内其他未提交改动。
- 不进行全 App Dark Mode 重构。

## 实现边界

动态颜色优先收口到现有设计系统或对应颜色模型，视图只消费语义颜色，避免在多个组件内重复判断 `colorScheme`。已有 Asset Catalog 动态颜色可直接复用；需要多级色阶时，由颜色模型集中返回 Light/Dark 对应颜色。

## 验收标准

- Light Mode 的现有视觉层级基本不变。
- Dark Mode 下切换器不再出现大面积亮白背景。
- Dark Mode 下月历和热力图的空格、低活跃度和高活跃度仍能逐级区分。
- 日期文字、选中框、模块底条在两种模式下均保持清晰。
- iOS 工程编译成功。
- Git diff 仅包含本次方案与适配相关文件，不夹带工作区内其他改动。

## 风险控制

- 动态色阶只改变视觉表现，不改变事件数量到等级的映射，避免影响现有逻辑测试。
- 当前工作区已有记忆长廊和周历相关未提交改动，实施与提交时必须逐文件核对并 scoped staging。
- 本次不涉及 `HoloBackend/`，无需后端部署。
