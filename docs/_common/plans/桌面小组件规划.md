## 📋 前置要求

### 必需条件
| 项目 | 说明 |
|------|------|
| 开发者账号 | 付费 Apple Developer Program（$99/年） |
| App Groups | 主 App 与 Widget 共享数据的唯一方式 |
| 最低 iOS 版本 | iOS 17+（支持 AppIntent 直接交互） |

### 为什么需要开发者账号
- Widget 和主 App 运行在**独立的沙盒**中
- App Groups 是 Apple 付费功能，需要在后台注册
- 没有它，Widget 无法读取主 App 的 Core Data 数据

## 🔄 刷新频率

| 频率 | 优点 | 缺点 |
|------|------|------|
| 高 (5-15分钟) | 数据更实时 | 耗电增加、系统可能限制 |
| 低 (30-60分钟) | 省电、稳定 | 数据可能过时 |

**建议**：15-30 分钟
- 用户点击 +1 打卡后，Widget 会**立即刷新**
- 刷新频率主要影响"被动展示"的准确性

## 📝 实现步骤

### 阶段 1：配置 App Groups（需要开发者账号）
1. Apple Developer 后台创建 App Group
2. Xcode 启用 App Groups 能力
3. 修改 Core Data 存储位置到共享容器

### 阶段 2：创建 Widget Extension
1. Xcode 添加 Widget Extension Target
2. 配置共享代码和数据模型
3. 实现 TimelineProvider

### 阶段 3：实现 Widget UI
1. 小尺寸 Widget（优先）
2. 中尺寸 Widget
3. 大尺寸 Widget

### 阶段 4：实现交互
1. AppIntent 打卡功能
2. Link 跳转功能
3. 数据刷新逻辑
