# HoloAI 领域记忆上线验收清单

> 目标：确保领域记忆和跨域融合在不增加用户理解负担、不泄露敏感正文、不拖慢普通聊天的前提下分阶段上线。

## 1. 发布原则

- 用户只看到自然语言结论、时间范围、来源和反馈操作，不展示置信度、新鲜度、相关度等内部参数。
- 普通问答优先读取已整理的 active 记忆；明确明细问题仍查询原始业务数据。
- shadow 阶段只生成和评估，不把新记忆注入回答。
- 灰度顺序固定为：内部账号 → 小流量桶 → 全量。
- extraction、fusion、answer injection 可独立关闭；关闭能力不删除已有记录。
- 后端 Prompt 或 purpose 变更必须完成 ECS 发版与公网版本验证后才算上线。

## 2. 功能正确性

- [ ] 财务、观点、健康、习惯、任务、目标、对话、个人八个领域可独立形成记忆。
- [ ] 跨域融合必须同时通过共同时间、共同 anchor、跨领域、独立底层 evidence 四道门。
- [ ] 第一次未确认的跨域候选只做 transient preview；重复周期或用户确认后才持久化。
- [ ] Query Service 只向回答提供 `active` 且未被拒绝、忘记或标记无关的记忆。
- [ ] “最近 14 天吃了多少麦当劳”等明细问题路由到原始数据，记忆只能作为背景。
- [ ] Chat、分析、洞察、规划、五个能力入口、Agent、Tool 全部经过统一权限和查询入口。
- [ ] 关闭“记忆辅助回答”后，所有消费端返回空记忆且不读取仓库、不触发 SWR。

## 3. 用户控制与隐私

- [ ] 用户可以执行“准确”“不准确”“纠正”“不再使用”“查看来源”。
- [ ] 纠正生成用户确认的新版本，保留稳定 ID、版本链和来源证据。
- [ ] “不再使用”先写 semantic tombstone，再擦除正文；同义候选不能重新生成。
- [ ] 用户界面不出现 confidence、freshness、relevance 数字或内部评分术语。
- [ ] 来源失效、记录冲突或部分来源不可用时明确显示降级状态。
- [ ] 普通 Trace 只保存计数、路由、选中 ID、刷新决策和不可逆问题指纹。
- [ ] Debug 敏感摘要只能在主动开启的临时内存视图中显示，退出即释放。
- [ ] Release 二进制不包含 AI 记忆实验室入口、类型和内部文案。

## 4. 性能与容量门槛

- [ ] 本地筛选和 top-K 召回 p95 ≤ 100ms。
- [ ] 普通回答最多注入 8 条记忆，默认预算不超过约 2,000 tokens。
- [ ] 普通聊天关键路径新增串行网络往返数为 0。
- [ ] 后台同时运行的记忆 AI job 不超过 1。
- [ ] 每个领域最多保留 50 条 active current-state/phase/durable 自动记忆。
- [ ] active 跨域记忆最多保留 100 条。
- [ ] compaction 不归档用户确认/纠正事实，不删除 permanent fact 或 tombstone。
- [ ] 历史记录真实 JSON 编码体积初始上限为 8 MiB；evidence metadata 独立上限为 4 MiB。
- [ ] Debug Trace 最多 500 条并只保留 7 天。

## 5. 可观测性与开发者验收

Debug 路径：`AI 设置 → AI 记忆实验室`。

- [ ] 总览能看到各领域 last run、dirty、记录数、重试时间和当日预算。
- [ ] 领域页能按 signal → package → AI raw result → validator → planned mutation 检查 dry-run。
- [ ] 跨域页能展示候选、四道门、lineage 和首个未通过原因。
- [ ] 查询模拟器能展示 route、选中 ID、detail fallback、SWR 和临时最终上下文。
- [ ] 单条检查能展示版本、anchor、evidence、counter-evidence、tombstone 命中和评分明细。
- [ ] 指标只含计数、时延、命中率、拒绝率、纠正率，不含问题或记忆正文。

## 6. 灰度与熔断

| 阶段 | extraction | fusion | answer injection | 晋级条件 |
|---|---:|---:|---:|---|
| off | 关 | 关 | 关 | 仅紧急停用 |
| shadow | 开 | 开 | 关 | 完成数据质量、隐私和性能验收 |
| internalAccounts | 开 | 开 | 仅内部账号 | 内部问答无严重误用，纠正/拒绝率可接受 |
| limited | 开 | 开 | 内部账号 + 小流量桶 | 指标稳定且无 P0/P1 隐私或错误结论 |
| full | 开 | 开 | 开 | 产品负责人确认全量 |

- [ ] extraction kill switch 独立验证。
- [ ] fusion kill switch 独立验证。
- [ ] answer injection kill switch 独立验证。
- [ ] 从任意阶段回退到 shadow/off 不破坏已有数据。

## 8. Go / No-Go

以下任一情况必须 No-Go：

- 忘记内容可通过 Query、Tool、Trace 或重建候选再次出现。
- 关闭 Holo AI / 记忆辅助后仍有静默读取或外部 AI 请求。
- 明细问题由记忆替代真实数据回答。
- Release 暴露实验室、Prompt、敏感日志或 evidence 正文。
- 测试命令退出 0 但实际执行 0 个测试。
- 后端本地已改但未完成生产部署与公网契约验证。
