# ADR：记忆默认采用与敏感确认边界

- 状态：Accepted
- 日期：2026-07-16
- 决策范围：iOS 本地记忆策略、迁移、召回与用户界面

## 背景

逐条审批让记忆能力变成用户负担，也让 HoloAI 无法及时使用已经通过安全校验的普通记忆。但所有内容无条件自动采用会把健康、身份、永久事实和推测性结论带入回答，风险不可接受。

## 决策

采用“默认学习 + 风险分层确认”：

- 普通且通过 Validator 的领域记忆自动生效。
- 健康、身份、人生事件、永久事实、假设和首次跨域结论等待确认。
- 待确认记忆既不召回，也不能作为跨域融合上游。
- 历史候选统一解除确认门槛，但保留原时间并立即应用衰减。
- 非永久记忆按 14/60/150 天半衰期治理，低于门槛退出召回并归档。
- 反馈覆盖自然衰减：不准确立即抑制，可恢复；不再使用只删除当前记录。
- 普通界面不展示内部评分，Debug 实验室保留完整解释链。
- 继续使用现有“自动形成记忆”和“记忆辅助回答”开关，不新增设置。

## 结果

正向结果：用户不需要维护审批队列，HoloAI 能更快利用精炼记忆；高影响内容仍由用户掌握；过时结论会自然退出；开发者可以解释每次采用和排除。

代价：需要维护采用策略版本、回执去重和迁移 journal；历史候选会在一次迁移中发生状态变化；Debug 与正式用户界面必须严格隔离。

## 被否决方案

- 所有记忆继续逐条审批：交互成本过高，记忆长期停留在候选态。
- 所有记忆无条件自动采用：敏感与高影响错误会直接污染回答。
- 删除写入永久阻断规则：与“新证据可以重新形成同类记忆”的产品语义冲突。
- 向普通用户展示置信度/新鲜度：增加理解负担，且容易把内部排序误解为事实概率。

## 变更边界

本 ADR 不修改后端 Prompt，不需要发布 HoloBackend。后续如改变抽取 Prompt 或服务端策略，必须单独立项并同步双端版本。
