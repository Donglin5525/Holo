# 个人页记忆入口迁移 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** 将用户记忆控制和 Debug 记忆实验室从设置体系迁移到个人页，同时保持现有数据与服务逻辑不变。

**Architecture:** 新增个人记忆二级页复用 `HoloMemorySettings` 与 `HoloMemoryCenterView`；个人页只展示一个用户入口和一个 Debug 开发入口。设置页及 `AISettingsView` 删除重复 UI，不改 Repository、CloudKit、反馈或查询链路。

**Tech Stack:** SwiftUI, Swift, iOS Simulator, xcodebuild.

---

### Task 1: 新增个人记忆二级页

**Files:**
- Create: `Holo/Holo APP/Holo/Holo/Views/Personal/PersonalMemorySettingsView.swift`
- Modify: `Holo/Holo APP/Holo/Holo/Views/Personal/PersonalView.swift`

**Steps:**

1. 新建二级页，复用 `HoloMemorySettings.shared` 展示“自动形成记忆”和“记忆辅助回答”。
2. 添加“记忆管理”入口，继续打开现有 `HoloMemoryCenterView`。
3. 在个人页新增“我的记忆”单卡入口，副标题只显示用户可理解的开关状态。
4. 在 `#if DEBUG` 下新增“开发者工具 → AI 记忆实验室”，直接进入 `AIMemoryLabView`。
5. 使用源码结构断言确认普通入口与 Debug 门控存在。

### Task 2: 清理设置体系重复入口

**Files:**
- Modify: `Holo/Holo APP/Holo/Holo/Views/Settings/SettingsView.swift`
- Modify: `Holo/Holo APP/Holo/Holo/Views/Settings/AISettingsView.swift`

**Steps:**

1. 从 `SettingsView` 删除整块 `aiMemorySection` 及其状态依赖。
2. 从 `SettingsView` 删除重复的“个人档案”入口及其 sheet 状态。
3. 从 `AISettingsView` 删除 `memorySection`、手动观察状态与方法。
4. 保留 Provider、授权、模型、Prompt、学习映射、Agent 灰度和危险操作。
5. 使用 `rg` 验证设置页不再包含用户记忆入口，个人页成为唯一用户入口。

### Task 3: 构建与模拟器验收

**Files:**
- Preserve: existing simulator App data and `full-chain-v1` Store/report.

**Steps:**

1. 构建 Debug 指定模拟器，要求 `BUILD SUCCEEDED`。
2. 构建 Release generic simulator，要求 `BUILD SUCCEEDED`。
3. 扫描 Release 主二进制，实验室入口文案必须 0 命中。
4. 覆盖安装 Debug App，不 uninstall、不 erase simulator。
5. 使用无 reset 的 `full-chain-v1` 参数启动，确认之前的 Mock Store/report 仍存在。
6. 在模拟器检查：设置页已收口；个人页有“我的记忆”；二级页可操作开关和记忆管理；Debug 可进入实验室。
7. 停留在“个人”或“我的记忆”页面供东林验收，不清理模拟器数据。
