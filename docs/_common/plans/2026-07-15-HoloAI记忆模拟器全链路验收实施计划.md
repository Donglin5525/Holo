# HoloAI 记忆模拟器全链路验收 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** 在 Debug iOS 模拟器中通过启动环境变量注入隔离 mock 记忆，使用现有 UI 和生产服务完成领域生成、用户反馈、跨域融合、查询路由和回答上下文的全链路验收。

**Architecture:** 使用 `#if DEBUG && targetEnvironment(simulator)` 限制的环境解析器切换 `HoloMemoryRuntime` 到独立本地 SQLite Store，由无 UI 的 scenario runner 将固定模型 JSON 依次送入 Validator、Applier、Repository、Feedback、Fusion、Query 和 Context Envelope。runner 输出机读 JSON 报告；现有记忆管理、查询模拟器和 Holo 对话共享同一隔离 Repository。

**Tech Stack:** Swift 5, SwiftUI, Core Data, iOS Simulator, `xcodebuild`, `xcrun simctl`, standalone `swiftc` tests.

### Task 3: 实现无 UI 的全链路 Scenario Runner

**Files:**
- Create: `Holo/Holo APP/Holo/Holo/Services/Diagnostics/HoloMemorySimulatorValidationScenario.swift`
- Create: `Holo/Holo APP/Holo/HoloTests/Services/AI/HoloMemorySimulatorValidationScenarioTests.swift`

**Step 1: Write failing fixture/contract tests**

测试固定场景必须产生：

- finance active、health active、habit candidate、correct/reject/forget 专用记忆。
- 至少一组跨域候选通过四道门。
- 所有 ID/lineage 都以 `sim-validation-v1` 稳定命名。
- 模型 JSON 包含 `requestedActions:null`。
- 报告结构必须可 Codable，每个 assertion 包含 name/status/expected/actual。

**Step 2: Run tests to verify failure**

Expected: FAIL because fixture and runner are absent.

**Step 3: Implement scenario in phases**

1. 构建 domain signal/package 和固定模型 JSON。
2. 调用 `HoloDomainMemoryOutputValidator.decodeAndValidate` 和 `HoloDomainMemoryObservationApplier.apply`。
3. 验证 observation key 幂等。
4. 构建跨域 candidate，调用 `HoloCrossDomainFusionService.evaluate`，第二次 occurrence 持久化。
5. 使用不同专用记忆分别调用 accurate/correct/inaccurate/noLongerUse。
6. 对 forgotten identity 重新 upsert，断言 `rejectedByTombstone`。
7. 以 answering allowed/disabled 两种模式调用 `HoloMemoryQueryService`。
8. 生成 holistic/domain/detail 三类 context，用 `HoloMemoryContextEnvelope.render` 检查最终正文。
9. 将脱敏断言和结论编码到 report URL。

runner 需幂等：相同 scenario/store 已生成通过报告时，非 reset 启动不重放破坏性反馈。

**Step 4: Run scenario tests**

Expected: every named assertion passes and no report field contains non-mock data.

**Step 6: Preserve simulator state**

- Do not delete the scenario Store.
