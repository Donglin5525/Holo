# iOS 全局右滑返回手势实现规划

## Context

用户希望为 iOS 应用全局添加右滑返回上一页的手势功能。

**现状分析**：
- FinanceView 和 HabitsView 已有右滑返回实现，但代码重复
- 其他 Sheet/FullScreenCover 页面没有右滑返回
- 没有统一的可复用组件

**目标**：创建可复用的 ViewModifier，统一应用到所有需要返回的页面。

---

## 实现方案

### 1. 创建 SwipeBackModifier

**新建文件**: `Holo/Holo APP/Holo/Holo/Modifiers/SwipeBackModifier.swift`

```swift
struct SwipeBackModifier: ViewModifier {
    let edgeWidth: CGFloat      // 触发区域宽度（默认 40pt）
    let triggerDistance: CGFloat // 触发滑动距离（默认 120pt）
    let onDismiss: () -> Void
    @State private var offset: CGFloat = 0
}

extension View {
    func swipeBackToDismiss(onDismiss: @escaping () -> Void) -> some View
}
```

**关键参数**（与现有实现保持一致）：
- 触发区域：左侧 40pt
- 触发距离：滑动 120pt
- 完成动画：`.easeOut(duration: 0.25)`
- 回弹动画：`.spring(response: 0.3)`

### 3. 实现步骤

1. **创建 SwipeBackModifier.swift**
   - 定义 ViewModifier 结构体
   - 实现 DragGesture 手势逻辑
   - 添加 View 扩展方法

2. **重构现有实现**
   - FinanceView: 删除 `@State private var swipeBackOffset`，替换手势代码
   - HabitsView: 同上

3. **应用到其他页面**
   - 在 body 最外层视图添加 `.swipeBackToDismiss { dismiss() }`

---

### 4. 修改示例

**FinanceView.swift 修改前 (71-90行)**:
```swift
@State private var swipeBackOffset: CGFloat = 0
// ...
.offset(x: swipeBackOffset)
.gesture(DragGesture()...)
```

**修改后**:
```swift
.swipeBackToDismiss { dismiss() }
```

## 验证方案

### 模拟器测试

1. 在 Xcode 中构建并运行
2. 测试每个修改的页面：
   - 从左边缘 40pt 内右滑 120pt+ → 页面关闭
   - 从左边缘右滑不足 120pt → 回弹
   - 从屏幕中央右滑 → 不响应
