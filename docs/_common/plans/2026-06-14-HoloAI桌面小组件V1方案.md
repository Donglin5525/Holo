# HoloAI 桌面小组件 V1 方案

> 日期：2026-06-14
> 平台：iPhone / iPad 主屏幕 WidgetKit
> 状态：方案定稿，待工程拆解

---

## 1. 目标

HoloAI 桌面小组件 V1 的目标不是把完整 App 搬到桌面，而是让用户在主屏幕上更快进入 Holo 的四个高频场景：

1. 直接唤起 HoloAI 语音输入。
2. 快速执行常用动作。
3. 看见今日收支和本月预算节奏。
4. 偶然回到自己写过的想法。

首发版优先解决“可用、稳定、像 Holo”。所有复杂数据处理由主 App 完成，Widget 只读取轻量快照并负责展示和跳转。

---

## 2. 产品范围

### 2.1 首发组件

| 组件 | 推荐尺寸 | 核心价值 | 首发行为 |
|---|---|---|---|
| HoloAI 语音启动 | 小尺寸优先，中尺寸可选 | 少一步进入 HoloAI 语音输入 | 点击打开 AI 对话，并自动弹出语音输入面板 |
| Holo 快捷控制台 | 中尺寸 | 桌面 2x2 快捷入口 | 问 Holo / 记一笔 / 写想法 / 加待办 |
| 今日收支卡片 | 中尺寸，小尺寸简化 | 今日收入、支出和本月预算节奏 | 展示今日收入、今日支出、预算进度、时间进度 |
| 想法随机漫步 | 中尺寸 | 从想法网络里偶然回忆一条内容 | 默认隐私保护，可开启原文摘录 |

### 2.2 非首发范围

- 不做锁屏 Widget。
- 不做 macOS 桌面 Widget。
- 不在 Widget 内直接调用 AI。
- 不在 Widget 内直接读写完整 Core Data。
- 不做复杂动画；Widget 刷新和动画能力受系统限制。
- 不做“点击后自动开始录音”，首发只自动弹出语音输入面板。

## 3. 统一视觉方向

视觉方向采用“现有 Holo 深色克制风格 + HoloAI 入口的轻微召唤感”。

### 3.1 共同原则

- 深色底，减少桌面噪音。
- 信息层级清晰，数字和动作优先。
- 高光只用于引导注意，不做大面积炫光。
- 组件之间统一圆角、内边距、字体层级和图标语义。
- 财务和想法组件不依赖重图片，主要靠排版、标签和进度条建立质感。

### 3.2 HoloAI 语音波纹核心

快速启动组件使用抽象核心：不出现 Holo / H 标识，只用中心光点、细线波纹和轻微紫蓝高光表达“AI 正在待命”。

## 4. 组件设计

### 4.1 HoloAI 语音启动组件

### 用户体验

用户点击主屏幕小组件后：

1. 打开 Holo。
2. 进入 HoloAI 对话页。
3. 自动弹出语音输入面板。
4. 用户确认后开始说话。

首发不自动开始录音，避免踩 iOS 隐私和权限边界。

### 小尺寸布局

```text
┌────────────────────┐
│                    │
│      ◎ )))         │
│                    │
│      问 Holo       │
│    语音输入        │
└────────────────────┘
```

### 中尺寸布局（可选）

左侧是抽象语音波纹核心，右侧是短提示：

- 主文案：`问 Holo`
- 副文案：`打开语音输入`

### 技术行为

- Widget 发起 DeepLink：`holo://ai?voiceInput=true`
- App 解析后打开 ChatView。
- ChatView 根据参数把 `activeSheet` 切到 `.voiceInput`。
- 若用户未授权语音权限，在 App 内走现有权限引导。

---

### 4.2 Holo 快捷控制台组件

### 用户体验

中尺寸组件提供 2x2 快捷入口：

| 入口 | 行为 |
|---|---|
| 问 Holo | 打开 HoloAI 对话页 |
| 记一笔 | 打开快速记账 |
| 写想法 | 打开想法编辑 |
| 加待办 | 打开新建待办 |

### 布局

```text
┌──────────────────────────────────┐
│ Holo 快捷                         │
│ ┌────────┐ ┌────────┐             │
│ │ 问Holo │ │ 记一笔 │             │
│ └────────┘ └────────┘             │
│ ┌────────┐ ┌────────┐             │
│ │ 写想法 │ │ 加待办 │             │
│ └────────┘ └────────┘             │
└──────────────────────────────────┘
```

### 设计取舍

首发不放习惯打卡。习惯打卡通常需要选择具体习惯，单击直达价值不如前四个入口稳定。后续可以做“今日习惯”专用组件。

### 技术行为

- iOS 17+ 优先使用 AppIntent 或 Widget 内多个 Link。
- 每个入口独立 DeepLink，不把整个 Widget 做成单一点击区域。
- 动作语义复用 `HoloOneAction`，但 Widget 不强制跟随 HoloOne 设置。

### 4.3 今日收支卡片组件

### 用户体验

中尺寸卡片展示：

- 今日支出。
- 今日收入。
- 本月预算使用进度。
- 当前月份时间进度。

用户一眼能判断：今天收支如何，以及这个月消费节奏是否跑得太快。

### 中尺寸布局

```text
┌──────────────────────────────────┐
│ 今日收支                    本地 │
│                                  │
│ 支出 ¥128.50      收入 ¥0.00      │
│                                  │
│ 本月预算 62% · 时间过了 47%       │
│ ███████░░░░░░                    │
└──────────────────────────────────┘
```

### 小尺寸布局

小尺寸只展示今日支出和预算进度，避免信息拥挤。

### 预算进度口径

- 预算进度：`本月支出 / 本月总预算`
- 时间进度：`当前日期 / 当月总天数`
- 状态判断：
  - 预算进度 <= 时间进度 + 10%：正常
  - 预算进度明显快于时间进度：提醒
  - 预算进度 >= 100%：超预算

### 空状态

如果用户没有设置月预算：

- 不显示空进度条。
- 底部替换为：`本月支出 ¥xxx`
- 设置预算后自动恢复预算进度展示。

### 4.4 想法随机漫步组件

### 用户体验

组件每天从用户的想法网络里“漫游”出一条内容。它不是完全随机，而是沿着标签、引用关系、时间相近或主题相近做关联选择。

### 隐私策略

默认隐私保护：

- 不直接展示原文。
- 展示日期、标签和弱提示。
- 用户可在设置中开启“显示原文摘录”。

开启原文摘录后：

- 显示 2-3 行内容。
- 点击进入对应想法详情。

默认保护状态下：

- 点击进入想法模块，并定位到这条想法或打开详情前在 App 内查看。

### 默认布局

```text
┌──────────────────────────────────┐
│ 今天想起一条想法                   │
│                                  │
│ 2026.06.08                       │
│ #产品灵感  #自我观察               │
│ 来自一次夜间记录                   │
└──────────────────────────────────┘
```

### 随机漫步规则

首发可落地版本：

1. 每天选一个起点想法。
2. 起点优先来自有标签、有引用、内容长度适中的想法。
3. 如果有引用关系，优先走引用关系。
4. 如果没有引用关系，按标签或时间窗口找关联想法。
5. 最终选择一条适合展示的内容写入 Widget 快照。

后续增强：

- 支持下一次刷新沿着同一标签或引用继续走。
- 显示“从 A 想到了 B”的轻提示。
- 接入 AI，把多条关联想法压缩成一句回忆。

## 5. 数据架构

### 5.1 总原则

主 App 负责计算，Widget 负责展示。

Widget 与主 App 独立沙盒运行，首发采用 App Group 共享轻量 JSON 快照：

```text
主 App Core Data / Settings
        ↓
WidgetSnapshotWriter
        ↓
App Group JSON
        ↓
Widget TimelineProvider
        ↓
SwiftUI Widget View
```

### 5.2 快照文件建议

| 文件 | 内容 |
|---|---|
| `widget_quick_actions.json` | 快捷控制台动作列表和展示文案 |
| `widget_finance_snapshot.json` | 今日收入、今日支出、预算进度、时间进度、状态 |
| `widget_thought_memory.json` | 想法回忆展示数据、隐私状态、目标 thoughtId |

### 5.3 数据刷新触发

主 App 在以下场景写快照：

- App 启动完成后。
- 财务新增、编辑、删除后。
- 想法新增、编辑、删除、归档后。
- 用户修改 Widget 隐私设置后。
- 用户修改预算后。

写入完成后调用 `WidgetCenter.shared.reloadAllTimelines()` 或按 kind 刷新。

### 5.4 为什么不让 Widget 直接查 Core Data

- Widget 生命周期短，不适合重查询。
- Core Data 共享容器迁移风险更高。
- 财务和想法都可能包含隐私数据，快照更容易做裁剪和脱敏。
- 后续若数据结构变化，只要快照协议稳定，Widget UI 不必跟着大改。

## 6. DeepLink 与 AppIntent

### 6.1 DeepLink 草案

| 入口 | DeepLink |
|---|---|
| HoloAI 语音输入 | `holo://ai?voiceInput=true` |
| 问 Holo | `holo://ai` |
| 记一笔 | `holo://finance/add` |
| 写想法 | `holo://thoughts/new` |
| 加待办 | `holo://tasks/new` |
| 打开想法详情 | `holo://thoughts/detail?id=<uuid>` |

### 6.2 需要扩展的 App 内目标

现有 DeepLink 已有财务、任务、记忆长廊等目标，但首发 Widget 需要补齐：

- AI 对话页。
- AI 对话页 + 语音输入面板。
- 快速记账 Sheet。
- 想法编辑 Sheet。
- 新建待办 Sheet。
- 想法详情。

建议新增统一的 `WidgetDeepLinkTarget` 或扩展现有 `DeepLinkTarget`，避免每个页面各自解析 URL。

## 7. 设置项

### 7.1 Widget 设置入口

建议在设置页新增 `桌面小组件` 设置项，包含：

- 想法回忆是否显示原文摘录。
- 财务卡片是否展示预算提醒色。
- 快捷控制台入口排序（后续）。

### 7.2 首发最小设置

首发只必须有一个设置：

- `想法回忆：显示原文摘录`，默认关闭。

其他设置可先写死，降低首发复杂度。

## 8. 工程实施分期

### Phase 1：Widget 基础设施

- 新增 Widget Extension Target。
- 配置 App Groups。
- 建立共享快照目录和读写工具。
- 增加 Widget kind 和基础 TimelineProvider。

### Phase 2：DeepLink 接线

- App 支持 `holo://` URL scheme。
- 扩展 DeepLinkTarget。
- 接通 AI 语音输入、快速记账、想法编辑、新建待办。
- 实机验证冷启动和热启动。

### Phase 3：四个组件 UI

- HoloAI 语音启动小组件。
- Holo 快捷控制台中组件。
- 今日收支中/小组件。
- 想法随机漫步中组件。

### Phase 4：数据快照

- 财务快照写入。
- 想法随机漫步快照写入。
- 隐私设置接线。
- Widget 刷新策略接线。

### Phase 5：验证与收尾

- iPhone 小/中尺寸截图验证。
- iPad 主屏幕截图验证。
- 无预算、无想法、无财务记录空状态验证。
- 冷启动 DeepLink 验证。
- 权限未授权时的语音输入路径验证。

## 9. 验收标准

### 9.1 用户可见验收

- 用户可以在 iPhone/iPad 主屏幕添加四类小组件。
- 点击语音启动组件后，进入 HoloAI 并看到语音输入面板。
- 快捷控制台四个按钮分别打开正确页面或弹窗。
- 财务卡片展示今日收入、今日支出、预算进度和时间进度。
- 想法回忆默认不暴露原文；开启设置后展示摘录。

### 9.2 工程验收

- Widget 不直接调用后端 AI。
- Widget 不直接执行复杂 Core Data 查询。
- App Group 快照字段有默认值和版本号。
- 空数据状态不崩溃、不显示脏数据。
- DeepLink 冷启动、热启动都能处理。

## 10. 风险与取舍

| 风险 | 影响 | 首发策略 |
|---|---|---|
| App Groups 需要开发者账号和 entitlement | 无法真机完整验证共享数据 | 先完成工程结构，真机配置后验证 |
| 自动录音受系统限制 | 体验可能不如“一点即说” | 首发只弹语音输入面板，不自动录音 |
| Widget 刷新不实时 | 财务数据可能延迟 | App 内数据变更后主动 reload timeline |
| 想法内容隐私风险 | 桌面泄露私人内容 | 默认隐私保护，原文摘录需用户开启 |
| 多入口 DeepLink 变复杂 | 页面状态容易冲突 | 统一目标枚举和 URL 解析层 |

---

## 11. 后续增强

- 锁屏 Widget：只放 HoloAI 语音入口和今日支出。
- 今日习惯 Widget：支持一键打卡或打开具体习惯。
- 财务 Widget 增加分类 Top 1 或异常提醒。
- 想法随机漫步接入 AI，生成“今天 Holo 想起了...”摘要。
- 小组件 Gallery 预览图，用于 App 内设置页展示。
