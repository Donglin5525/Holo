# 「今日状态」上下文构建 修复方案

> 走查日期：2026-06-11
> 状态：待实施

## 问题全景

AI 在回答"帮我看看今天的整体状态"时，看到的上下文数据存在多处与实际不符的问题，导致回复内容错误。

共发现 9 个问题，按严重程度分 3 级：

| 级别 | 数量 | 说明 |
|------|------|------|
| P0 | 1 | 数据完全错误，AI 基于错误数据给出错误结论 |
| P1 | 2 | 关键数据缺失或语义反转 |
| P2 | 3 | 信息缺失或计算浪费 |
| P3 | 3 | 命名和细节问题 |

## P0：任务统计无时间过滤

### 现象

AI 显示「今日任务：99 个（已完成 93），逾期 0 个」。

实际含义：用户历史创建的所有未删除、未归档任务共 99 个，其中 93 个已完成。**和今天无关。**

### 根因

`UserContextBuilder.swift` 第 231-253 行 `buildTaskSummary()` 方法：

```swift
let stats = repo.getTaskStatistics()  // ← 返回全部活跃任务
...
todayTotal: stats.total,        // ← activeTasks.count（全部活跃）
todayCompleted: stats.completed // ← 全部活跃中已完成数
```

`getTaskStatistics()` 的统计条件是 `deletedFlag == NO AND archived == NO`，没有日期过滤。

### 修复方案

**改什么**：`buildTaskSummary()` 改用「今日到期 + 今日完成」的统计口径。

**不改什么**：`getTaskStatistics()` 本身不改，其他模块可能在使用它的全量统计语义。

#### 步骤 1：在 `TodoRepository+Stats.swift` 新增方法

```swift
/// 今日任务统计（今日到期 + 今日已完成）
func getTodayTaskStats() -> (dueToday: Int, completedToday: Int, overdue: Int) {
    let calendar = Calendar.current
    let today = calendar.startOfDay(for: Date())
    guard let tomorrow = calendar.date(byAdding: .day, value: 1, to: today) else {
        return (0, 0, 0)
    }

    let base = "deletedFlag == NO AND archived == NO"

    // 今日到期（不管是否已完成）
    let dueToday = countTasks(
        predicate: "\(base) AND dueDate >= %@ AND dueDate < %@",
        today as NSDate, tomorrow as NSDate
    )

    // 今日完成（completedAt 在今天的，不限 dueDate）
    let completedToday = countTasks(
        predicate: "\(base) AND completed == YES AND completedAt >= %@ AND completedAt < %@",
        today as NSDate, tomorrow as NSDate
    )

    // 逾期（截止日已过且未完成）
    let overdue = countTasks(
        predicate: "\(base) AND completed == NO AND dueDate < %@",
        today as NSDate
    )

    return (dueToday, completedToday, overdue)
}
```

**设计说明**：
- `dueToday`：今日到期的任务总数（含已完成和未完成），代表"今天需要处理的事"
- `completedToday`：今天实际完成的任务数（不限截止日期），代表"今天的产出"
- `overdue`：截止日已过且未完成的任务，代表"积压"

这两个数字的含义不同，AI 可以分别解读：
- dueToday = 3, completedToday = 5 → "今天到期 3 个任务，实际完成了 5 个（含清历史积压）"
- dueToday = 0, completedToday = 2 → "今天没有到期任务，但主动完成了 2 个积压"

```swift
private func buildTaskSummary() -> TaskSummary {
    let repo = TodoRepository.shared
    let stats = repo.getTodayTaskStats()  // ← 改用新方法
    let todayTasks = repo.getTodayTasks()

    var recentList: [String] = []
    for task in todayTasks.prefix(5) {
        let status = task.completed ? "✓" : "○"
        recentList.append("\(status) \(task.title)")
    }

    let activeTasks = repo.activeTasks.filter { !$0.completed && !$0.deletedFlag }
    let activeTaskSummaries = activeTasks.prefix(10).map { "○ \($0.title)" }

    return TaskSummary(
        dueToday: stats.dueToday,
        completedToday: stats.completedToday,
        overdueCount: stats.overdue,
        recentTasks: recentList,
        activeTaskSummaries: activeTaskSummaries
    )
}
```

```swift
/// 构建今日习惯打卡摘要，格式如："戒烟: 未打卡" / "健身: 30分钟" / "俯卧撑: 15个（超标）"
```

#### 步骤 2：在 HabitRepository 新增获取今日值的方法

```swift
/// 获取某个习惯今日的记录值（打卡型返回是否有记录，数值型返回具体数值）
func getTodayValue(for habit: Habit) -> (hasRecord: Bool, value: Double?, isCompleted: Bool) {
    let calendar = Calendar.current
    let today = calendar.startOfDay(for: Date())
    guard let tomorrow = calendar.date(byAdding: .day, value: 1, to: today) else {
        return (false, nil, false)
    }

    let request = HabitRecord.fetchRequest()
    request.predicate = NSPredicate(
        format: "habitId == %@ AND date >= %@ AND date < %@",
        habit.id as CVarArg,
        today as NSDate,
        tomorrow as NSDate
    )
    request.fetchLimit = 1

    guard let record = try? context.fetch(request).first else {
        return (false, nil, false)
    }

    return (true, record.value, record.isCompleted)
}
```

#### 步骤 3：格式化输出

```swift
private func formatHabitLine(habit: Habit, todayValue: (hasRecord: Bool, value: Double?, isCompleted: Bool)) -> String {
    let name = habit.name

    if !todayValue.hasRecord {
        return "\(name): 未记录"
    }

    if habit.isCheckInType {
        // 打卡型：显示是否打卡
        if habit.isBadHabit {
            return "\(name): 已发生"  // 负向习惯用"已发生"而非"已完成"
        } else {
            return "\(name): 已打卡"
        }
    }

    // 数值型：显示具体数值
    if let value = todayValue.value {
        let unit = habit.unit ?? ""
        let overLimit = habit.isBadHabit && habit.dailyLimit > 0 && value > habit.dailyLimit
        let suffix = overLimit ? "（超标）" : ""
        return "\(name): \(Int(value))\(unit)\(suffix)"
    }

    return "\(name): 已记录"
}
```

## P1-2：负向习惯在「今日完成」中语义反转

### 现象

负向习惯（戒烟/抽烟）的打卡被计为"已完成"，AI 读到的数字和实际含义矛盾。

### 修复方案

**核心思路**：不改 `getTodayCheckInProgress()` 的计数逻辑（它已经正确区分了打卡型），改为在 AI 上下文中**分开展示正向和负向**，避免用一个笼统的"完成"掩盖语义差异。

#### 步骤 1：扩展 HabitSummary 模型

```swift
// 改前

    let totalActive: Int
    let todayCompleted: Int
    let todayTotal: Int
    ...
}

// 改后

    let totalActive: Int
    let todayCompleted: Int     // 今日正向习惯打卡数
    let todayTotal: Int         // 正向打卡型习惯总数
    let todayNegativeChecked: Int  // 今日负向习惯发生数
    let todayNegativeTotal: Int    // 负向打卡型习惯总数
    ...
}
```

#### 步骤 2：修改 `getTodayCheckInProgress()` 或新增方法

在 `HabitRepository` 新增：

```swift
/// 分别统计正向和负向打卡型习惯的今日进度
func getTodayCheckInSplit() -> (positive: (completed: Int, total: Int), negative: (checked: Int, total: Int)) {
    let positiveHabits = activeHabits.filter { $0.isCheckInType && !$0.isBadHabit }
    let negativeHabits = activeHabits.filter { $0.isCheckInType && $0.isBadHabit }

    let calendar = Calendar.current
    let today = calendar.startOfDay(for: Date())
    guard let tomorrow = calendar.date(byAdding: .day, value: 1, to: today) else {
        return ((0, 0), (0, 0))
    }

    let positiveIds = positiveHabits.map(\.id) as NSArray
    let negativeIds = negativeHabits.map(\.id) as NSArray

    // 查正向
    let positiveCompleted = countCompletedHabits(ids: positiveIds, from: today, to: tomorrow)
    // 查负向
    let negativeChecked = countCompletedHabits(ids: negativeIds, from: today, to: tomorrow)

    return (
        (positiveCompleted, positiveHabits.count),
        (negativeChecked, negativeHabits.count)
    )
}

private func countCompletedHabits(ids: NSArray, from: Date, to: Date) -> Int {
    let request = NSFetchRequest<NSDictionary>(entityName: "HabitRecord")
    request.resultType = .dictionaryResultType
    request.propertiesToFetch = ["habitId"]
    request.returnsDistinctResults = true
    request.predicate = NSPredicate(
        format: "habitId IN %@ AND date >= %@ AND date < %@ AND isCompleted == YES",
        ids, from as NSDate, to as NSDate
    )
    return (try? context.fetch(request).count) ?? 0
}
```

#### 步骤 3：更新 AI 上下文渲染

```
// 改前（一个笼统的数字）
- 活跃习惯：5 个，今日完成 0/1

// 改后（分开语义）
```

## P2-2：近期任务可能为空

### 现象

`recentTasks` 来自 `getTodayTasks().prefix(5)`，只取今日到期且未完成的任务。如果今天没有到期任务，列表为空。

### 修复方案

改为混合策略：优先显示今日到期任务，不足时补充最近未完成任务：

```swift
private func buildRecentTaskList(todayTasks: [TodoTask], repo: TodoRepository) -> [String] {
    var lines: [String] = []

    // 第一优先：今日到期任务
    for task in todayTasks.prefix(5) {
        let status = task.completed ? "✓" : "○"
        lines.append("\(status) \(task.title)")
    }

    // 补充：如果不足 3 条，加入最近的未完成任务
    if lines.count < 3 {
        let activeTasks = repo.activeTasks
            .filter { !$0.completed && !$0.deletedFlag }
            .sorted { ($0.dueDate ?? .distantFuture) < ($1.dueDate ?? .distantFuture) }

        let existingTitles = Set(todayTasks.map(\.title))
        for task in activeTasks {
            if lines.count >= 5 { break }
            if existingTitles.contains(task.title) { continue }
            let dueLabel = formatDueLabel(task.dueDate)  // "明天"、"后天"、"6/15"
            lines.append("○ \(task.title)（\(dueLabel)）")
        }
    }

    return lines
}

private func formatDueLabel(_ date: Date?) -> String {
    guard let date = date else { return "无截止日" }
    let calendar = Calendar.current
    let today = calendar.startOfDay(for: Date())
    let target = calendar.startOfDay(for: date)

    switch calendar.dateComponents([.day], from: today, to: target).day {
    case 0: return "今天"
    case 1: return "明天"
    case 2: return "后天"
    case let d where d > 2: return "\(calendar.component(.month, from: date))/\(calendar.component(.day, from: date))"
    case let d where d < 0: return "逾期\(-d!)天"
    default: return "无截止日"
    }
}
```

```
当用户问"今天状态"时，只使用上下文中的数字，不要自行推算或四舍五入。
```
