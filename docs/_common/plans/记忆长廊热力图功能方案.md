# 记忆长廊热力图功能方案

## Context

在记忆长廊页面顶部新增两个组件：

1. `统计概览`：展示全量累计数据
2. `活动热力图`：展示最近 13 周的主动活跃情况

当前页面结构：`导航栏 -> 筛选器 -> 时间线列表`。新组件插入在筛选器下方、时间线 section 上方，并与时间线共用同一个垂直 `ScrollView`，随时间线一起滚动。

### 已确认口径

| 维度 | 口径 | 说明 |
|------|------|------|
| 统计口径 | 全量累计 | 不受最近 13 周限制，不受 fetchLimit 截断 |
| 热力图口径 | 最近 13 周 + 主动活跃 | 只统计用户主动完成的动作 |
| 点击行为 | 滚动到对应周 | 不弹 Tooltip、不切周视图、不改筛选器 |
| 周定义 | 周一 ~ 周日 | 复用现有 `Date.startOfWeek` 口径 |
| 观点模块 | 本次接入 | 观点也算记忆，补齐接入链路 |

---

## 一、需求摘要

### 功能目标

- 给记忆长廊补一个总览层：先看长期累计，再看近期活跃，再进入逐日时间线
- 点击热力图某一天后，把时间线滚动到该日期所属周
- 把 `Thought` 模块接入记忆长廊，使其参与统计、热力图和时间线

### 非目标

- 不引入独立周视图
- 不新增 Tooltip / Popover
- 不改造现有高亮、里程碑机制
- 不让热力图跟随筛选器变化
- 不接入 AI 洞察真实数据，洞察字段仍固定为 `0`

---

## 二、数据口径定义

### 三套口径，独立计算

| 口径 | 服务对象 | 时间范围 | 任务过滤 | 数据来源 |
|------|----------|----------|----------|----------|
| 全量统计 | 顶部三字段 | 全量累计 | 已完成 + 逾期未完成 | 独立聚合查询 |
| 热力图 | 13 周色块 | 最近 13 周 | 仅已完成 | 独立范围查询 |
| 时间线 | 列表渲染 | 分页加载（7 天/页） | 已完成 + 逾期未完成 | 现有 `cachedItems` |

原因：

- 现有 `cachedItems` 有 fetchLimit，不能拿它做全量统计
- 热力图只看“主动活跃”，不能直接复用时间线全部数据
- 时间线继续保留现在的分页与缓存逻辑，避免把总览层和列表渲染耦死

### 各数据类型计入规则

| 数据类型 | 全量统计 | 热力图 | 时间线 |
|----------|----------|--------|--------|
| 交易 | 是 | 是 | 是 |
| 习惯记录 | 是，但仅限未归档习惯 | 是，但仅限未归档习惯 | 是 |
| 已完成任务 | 是 | 是，日期取 `completedAt` | 是 |
| 逾期未完成任务 | 是 | 否 | 是 |
| 观点 | 是 | 是 | 是 |

补充说明：

- `Transaction` 当前模型没有 `deletedFlag`，删除是硬删除，因此交易查询不加“未删除”谓词
- 习惯记录必须和当前时间线语义保持一致，只统计“所属 Habit 未归档”的记录，不能直接全表扫 `HabitRecord`

## 四、顺带修复：fetchTasks() 谓词 bug

当前 `fetchTasks()` 的谓词会把“已删除但已完成”的任务也算进来：

```swift
(completed == YES) OR (deletedFlag == NO AND archived == NO AND dueDate < %@)
```

本次一并修为：

```swift
deletedFlag == NO AND archived == NO AND (completed == YES OR dueDate < %@)
```

这样时间线、统计、热力图在任务侧的基础口径才一致。

## 六、数据层实现

### 1. 日期工具

仓库里已经有现成的 [CalendarHelpers.swift](/Users/tangyuxuan/Desktop/Claude/HOLO/Holo/Holo%20APP/Holo/Holo/Utils/CalendarHelpers.swift)，其中：

### 2. 全量统计查询

`totalMemoryCount` 用聚合查询，不走 `fetchLimit`，也不构造完整 `MemoryItem`。

推荐写法：

```swift
private func computeAggregateStats() {
    let context = CoreDataStack.shared.viewContext

    let transactionRequest = Transaction.fetchRequest()
    let txCount = (try? context.count(for: transactionRequest)) ?? 0

    let activeHabitIds = fetchActiveHabitIds()
    let habitRecordCount = countHabitRecords(for: activeHabitIds)

    let taskRequest = TodoTask.fetchRequest()
    taskRequest.predicate = NSPredicate(
        format: "deletedFlag == NO AND archived == NO AND (completed == YES OR dueDate < %@)",
        Date() as NSDate
    )
    let taskCount = (try? context.count(for: taskRequest)) ?? 0

    let thoughtRequest = Thought.fetchRequest()
    thoughtRequest.predicate = NSPredicate(format: "isSoftDeleted == NO AND isArchived == NO")
    let thoughtCount = (try? context.count(for: thoughtRequest)) ?? 0

    totalMemoryCount = txCount + habitRecordCount + taskCount + thoughtCount

    var allDates = Set<Date>()
    allDates.formUnion(fetchUniqueDates(entityName: "Transaction", key: "date", predicate: nil))
    allDates.formUnion(fetchHabitRecordDates(for: activeHabitIds))
    allDates.formUnion(fetchUniqueDates(
        entityName: "TodoTask",
        key: "completedAt",
        predicate: NSPredicate(format: "deletedFlag == NO AND archived == NO AND completed == YES")
    ))
    allDates.formUnion(fetchUniqueDates(
        entityName: "TodoTask",
        key: "dueDate",
        predicate: NSPredicate(format: "deletedFlag == NO AND archived == NO AND completed == NO AND dueDate < %@", Date() as NSDate)
    ))
    allDates.formUnion(fetchUniqueDates(
        entityName: "Thought",
        key: "createdAt",
        predicate: NSPredicate(format: "isSoftDeleted == NO AND isArchived == NO")
    ))

    totalRecordedDays = allDates.count
    totalInsights = 0
}
```

关键点：

- 交易不加 `deletedFlag`
- 习惯记录不能直接 `count(HabitRecord.fetchRequest())`
- 习惯记录必须先对齐未归档 Habit 集合
- 文档中不使用不存在的 `.then {}` 辅助写法

### 3. 热力图查询

热力图只保留一套最终实现：按天分组计数。

不要保留“先 `fetchUniqueDates` 再 `+1`”那套错误版本，因为它会把同一天多条记录压成 1。

推荐写法：

```swift
private func computeHeatmapData() {
    let today = Date().startOfDay
    let currentWeekStart = today.startOfWeek
    let windowStart = currentWeekStart.addingWeeks(-12)
    let windowEnd = currentWeekStart.addingDays(7)

    let activeHabitIds = fetchActiveHabitIds()

    var counts: [Date: Int] = [:]
    mergeDayCounts(into: &counts, fetchDayCounts(
        entityName: "Transaction",
        key: "date",
        predicate: NSPredicate(format: "date >= %@ AND date < %@", windowStart as NSDate, windowEnd as NSDate)
    ))
    mergeDayCounts(into: &counts, fetchHabitRecordDayCounts(
        for: activeHabitIds,
        start: windowStart,
        end: windowEnd
    ))
    mergeDayCounts(into: &counts, fetchDayCounts(
        entityName: "TodoTask",
        key: "completedAt",
        predicate: NSPredicate(format: "deletedFlag == NO AND archived == NO AND completed == YES AND completedAt >= %@ AND completedAt < %@", windowStart as NSDate, windowEnd as NSDate)
    ))
    mergeDayCounts(into: &counts, fetchDayCounts(
        entityName: "Thought",
        key: "createdAt",
        predicate: NSPredicate(format: "isSoftDeleted == NO AND isArchived == NO AND createdAt >= %@ AND createdAt < %@", windowStart as NSDate, windowEnd as NSDate)
    ))

    heatmapData = counts
}
```

### 4. 通用轻量查询能力

建议补 7 个小工具方法：

```swift
private func fetchActiveHabitIds() -> [UUID]
private func countHabitRecords(for activeHabitIds: [UUID]) -> Int
private func fetchHabitRecordDates(for activeHabitIds: [UUID]) -> Set<Date>
private func fetchHabitRecordDayCounts(for activeHabitIds: [UUID], start: Date, end: Date) -> [Date: Int]
private func fetchUniqueDates(entityName: String, key: String, predicate: NSPredicate?) -> Set<Date>
private func fetchDayCounts(entityName: String, key: String, predicate: NSPredicate?) -> [Date: Int]
private func mergeDayCounts(into target: inout [Date: Int], _ source: [Date: Int])
```

其中：

- `fetchUniqueDates` 只服务“记录天数”
- `fetchDayCounts` 只服务热力图
- 习惯记录走专用方法，确保和未归档 Habit 集合绑定
- 通用 Core Data 查询优先保持稳定：可以 fetch 指定日期字段后在 Swift 里按 `Date.startOfDay` 去重或分组，避免不同持久化存储对日期分组表达式支持不一致

## 七、时间线联动

### 核心思路

复用现有 `TimelineSection.id == Date` 作为滚动目标，不额外引入隐藏锚点。

### 点击热力图后的流程

```text
点击某天
-> 计算目标周 [weekStart, weekEnd)
-> 在当前 timelineSections 中找该周第一条已加载 section
-> 若已存在，直接 scrollTo(section.id)
-> 若不存在，持续 loadMore，直到：
   1. 找到该周 section；或
   2. 已经加载到比该周更早的位置
-> 若当前筛选器下该周仍无内容，则不滚动，保留选中态
```

### ViewModel 方法

```swift
func findSectionInWeek(of date: Date) -> Date?
func ensureWeekLoaded(_ date: Date) async
```

### 交互边界

- 不自动切换筛选器
- 如果热力图有活跃，但当前筛选器下该周没有 section，这是允许的
- 这种情况下不视为错误，只是不滚动

## 八、刷新策略

| 触发场景 | 时间线 | 统计 | 热力图 |
|----------|--------|------|--------|
| 首次进入 `refresh()` | 重算 | 重算 | 重算 |
| 数据变更通知 | 重算 | 重算 | 重算 |
| 筛选器切换 | 重算 | 不重算 | 不重算 |
| `loadMore()` | 追加 | 不重算 | 不重算 |
| 热力图点击 | 追加 | 不重算 | 不重算 |

原则：

- 统计和热力图只受“数据变化”影响
- 筛选器只影响时间线展示，不影响全局总量和 13 周活跃图

## 十一、最终结论

这次功能不是单纯“加两个卡片”，而是给记忆长廊补一层总览能力，所以实现上最重要的是守住 4 条边界：

1. 顶部统计必须是全量累计，不能被 fetchLimit 截断
2. 热力图必须只统计最近 13 周主动活跃，不能把逾期未完成任务算进去
3. 习惯记录的统计和热力图必须与当前时间线保持同一口径，只统计未归档习惯
4. 观点接入不能只停在 `MemoryItemType`，还要把通知、fetch、筛选和日摘要一起补齐

按这版方案实现，文档和现有代码结构就是闭合的。
