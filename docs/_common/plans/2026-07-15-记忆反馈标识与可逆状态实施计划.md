# 记忆反馈标识与可逆状态 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** 在记忆列表外层显示准确性反馈，允许用户反复修改，并将“不再使用”改为不写 tombstone 的单条物理删除。

**Architecture:** 保留现有 `confirmed/rejected/corrected` 决策模型，只改用户管理列表的可见性和呈现。在反馈存储契约中增加业务级单条删除方法，使 `noLongerUse` 不再调用永久忘记服务。

**Tech Stack:** Swift, SwiftUI, Core Data, standalone Swift tests, xcodebuild

---

### Task 2: 实现不写 tombstone 的物理删除

**Files:**
- Modify: `Holo/Holo APP/Holo/Holo/Services/AI/HoloMemoryFeedbackService.swift`
- Modify: `Holo/Holo APP/Holo/Holo/Services/AI/MemoryRepository/CoreDataHoloMemoryRepository.swift`

**Steps:**
1. 在 `HoloMemoryFeedbackStore` 增加 `deleteRecord(id:) async throws -> Bool`。
2. Core Data 实现在 mutation lock 内查找存储位置，删除主记录及其 evidence 对象。
3. 将 `.noLongerUse` 改为直接调用 `deleteRecord`，不调用 `HoloMemoryForgettingService.forget`。
4. 重跑 Task 1 命令，预期所有断言通过。

**Steps:**
1. 建立 `confirmed/rejected/corrected` 到“准确/不准确/已纠正”的唯一展示映射。
2. 在卡片摘要右侧显示小标识，`none` 不显示。

5. “准确”、“不准确”和“纠正”任意切换后都反映当前最新状态。
