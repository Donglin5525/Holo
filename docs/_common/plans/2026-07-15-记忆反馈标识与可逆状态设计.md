# 记忆反馈标识与可逆状态设计

## 产品目标

用户无需进入详情，即可在记忆管理列表识别自己已经反馈过的记忆；同时所有准确性反馈都可以修改，“不再使用”只删除当前记忆，不影响未来重新形成同类记忆。

## 交互规则

- 记忆卡片外层根据当前用户决策显示小标识：
  - `confirmed` 显示绿色“准确”。
  - `rejected` 显示橙色“不准确”。
  - `corrected` 显示主题色“已纠正”。
  - `none` 不显示标识。
- 选择“准确”或“不准确”后，详情页不关闭，列表状态立即更新。
- 用户可以反复修改准确性状态；“不准确”可改回“准确”，“已纠正”也可再重新评价。
- 标记为“不准确”的记忆保留在记忆管理列表，但不参与 HoloAI 回答。
- 选择“不再使用”并确认后，物理删除当前记忆及其证据记录，并从列表消失。

## 数据语义

- “不准确”继续使用 `userDecision = rejected` 与 `state = suppressed`，用来阻止该版本进入问答召回，但用户管理列表会显示它。
- 用户后续改为“准确”时，覆盖当前决策为 `confirmed`，状态恢复为 `active`。
- “不再使用”不调用忘记服务，不写入 tombstone，不保留空正文占位记录。
- 删除后，后台观察可以根据新证据重新生成相同稳定身份的记忆。

## 实现边界

- 在用户反馈服务中增加单条记忆物理删除能力，不复用只面向迁移的接口。
- 删除覆盖普通与敏感记忆存储，确保主记录与证据记录一起删除。
- 列表的可见规则单独允许 `rejected/suppressed`，不改变 HoloAI 查询只读取可用记忆的规则。
- 不增加新页面、新设置项或额外历史记录。

## 异常与验收

- 准确性反馈或删除失败时，保留当前卡片和原状态，显示失败提示。
- 单元验证覆盖：准确 → 不准确 → 准确可逆；纠正状态持久化；物理删除后记忆和证据不存在；删除不产生 tombstone。
- UI 验收覆盖：三种标识在外层可见；修改反馈后无需退出页面即时更新；删除后该卡片立即消失。
