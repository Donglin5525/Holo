## 0. 一句话定义

把首页顶部提醒胶囊升级为统一入口，承载 HoloAI「本周观察」的唤起与新用户洞察养成。**观察产物复用现有 `MemoryInsight(periodType:.weekly)`，不新建平行系统**；Push 整体 defer 到 V1.1。

核心路径：

```text
首页提醒胶囊（统一入口）
  -> HoloAI
  -> 本周观察卡片（MemoryInsight 升级形态）
  -> 状态判断 + 证据 + 建议 + 下一步动作
```

## 1. 范围

### 1.1 V1 做

| 能力 | 说明 |
| --- | --- |
| 首页胶囊统一入口 | 承载任务提醒 + 本周观察唤起 + 新用户养成进度 + AI 授权兜底，按优先级展示一条 |
| 本周观察产物 | 复用 `MemoryInsight(.weekly)`，分 3 天轻量版（`light3d`）与 7 天完整版（`full7d`） |
| 有效记录日 | 净新增 builder，V1 支持 Finance / Todo / Habit / Thought 四类 |
| consent 安全 | `MemoryInsightService.generateInsight` 加最终防线，修存量 8 入口裸奔 bug |
| HoloAI 卡片 | 基于现有 `MemoryInsightPayload` 升级展示结构（不强切 Agent） |
| AI 授权兜底 | 未授权时不生成、展示授权引导卡 |
| Core Data 扩展 | `MemoryInsight` 加 4 字段（自动迁移） |
| 合规 | PrivacyInfo + App Store 隐私问卷 + Review Notes |

### 1.2 V1 不做（defer V1.1）

- Push 全套（新手 2 次 + 常规每周 + 通知权限兜底 + 凌晨静默 + 幂等限频存储）
- 卡片级 DeepLink（`DeepLinkTarget.aiMessage` + ChatView 滚动）
- Agent 内核增强生成（feature flag 灰度，V1.0b / V1.1）
- AI 设置「本周观察提醒」开关
- 系统通知权限首次请求引导

> V1 点击胶囊/Push 的落地深度：只到 HoloAI 根页（`.ai`），不做卡片级定位。

## 2. 产品体验最终态

### 2.1 首页胶囊文案

| 状态 | 短文案 | 副语义（空间足够时） |
| --- | --- | --- |
| 养成期 0 天 | Holo 正在认识你 | 开始记录后，Holo 会逐渐发现你的生活模式 |
| 养成期 1-2 天 | 再记录 N 天生成观察 | — |
| 轻量观察已就绪（light3d） | 第一条观察已准备好 | 基于最近 3 个有效记录日的初步观察 |
| 完整观察已就绪（full7d） | 本周观察已准备好 | 基于最近 7 个有效记录日 |
| 强洞察待查看 | Holo 发现了一个新模式 | — |
| AI 授权未开启 | 开启 HoloAI 生成本周观察 | — |
| 生成失败 | 本周观察待重试 | — |
| 数据不足 | Holo 还需要更多记录 | — |

口径规则：天数够但累计模块不足 2 个 → `再记录一种内容，观察会更准`；模块够但天数不足 → `再记录 1 天生成观察`。

### 2.2 点击行为

| 胶囊类型 | 点击后 |
| --- | --- |
| 任务提醒 | 保持现有逻辑（跳任务详情/任务页） |
| 养成进度 | 跳 HoloAI，展示本周观察进度卡 |
| AI 授权未开启 | 跳 HoloAI 或 AI 设置授权引导卡 |
| 本周观察已生成 | 跳 HoloAI（V1 落到根页，不做卡片级滚动） |
| 观察失败兜底 | 跳 HoloAI，展示可重试卡片 |

### 2.3 HoloAI 本周观察卡片结构

| 区块 | 内容 |
| --- | --- |
| 标题 | 本周观察（full7d）/ 初步观察（light3d） |
| 状态判断 | Holo 观察到的一个主要生活模式 |
| 证据 | 2-4 条来自健康/任务/记账/习惯/想法的证据 |
| 建议 | 只给一个主建议 |
| 下一步 | 查看观察 / 生成下周轻量计划 / 继续观察一周 / 稍后提醒 / 今天不再显示 |

承诺文案：

- light3d：`这是基于最近 3 个有效记录日生成的初步观察，持续记录到 7 天后会更完整。` 用词限「初步 / 可能 / 线索」，禁「结论 / 证明 / 导致」。
- full7d：`这是基于最近 7 个有效记录日生成的本周观察。`

### 2.4 胶囊优先级（P0 最高）

| 优先级 | 类型 | 说明 |
| --- | --- | --- |
| P0 | 新观察已生成且未读 | 24 小时保护期 |
| P1 | 逾期任务 | 长期优先，保护任务主线 |
| P2 | 第一条观察养成进度 | 仅新手期，不压过逾期任务 |
| P3 | 今日到期 / 大量今日任务 | 任务主线 |
| P4 | 普通任务数量 | 现有「有 N 个任务待完成」 |
| P5 | 普通本周观察提醒 | 无更高优内容时 |

> 关键：现有 insight 通道 urgency 硬编码 `.pending`（最低，`HomeScheduleService.swift:245`），导致有任何未完成任务时 insight 永不展示。本周观察必须给 **P0 独立业务优先级**，不复用 `.pending`，否则加完等于没加。

## 3. 数据模型最终态

### 3.1 MemoryInsight 字段扩展

新增 4 个 optional 字段：

| 字段 | 类型 | 用途 |
| --- | --- | --- |
| `readAt` | `Date?` | 已读时间，回写后不再作「新观察」顶置 |
| `snoozedUntil` | `Date?` | 稍后提醒，延后展示 |
| `hiddenUntil` | `Date?` | 今天不再显示 |
| `observationStage` | `String?` | `light3d` / `full7d`，旧数据 nil 视为 `full7d/legacy` |

迁移：**自动轻量迁移，无需写迁移任务**。`CoreDataStack.swift:78-79` 已开启 `NSMigratePersistentStoresAutomaticallyOption` + `NSInferMappingModelAutomaticallyOption`；程序化模型加 optional 字段（无新关系、无 unique 约束）属最安全场景，有 `importBatchId` 先例。唯一验证：用一份旧 store 跑一次升级确认自动迁移通过。

### 3.2 observationStage 语义

```swift
enum MemoryInsightObservationStage: String {
    case light3d   // 3 个有效记录日触发的轻量版
    case full7d    // 7 个有效记录日触发的完整版
}
```

核心约束：**full7d 绝不能被 light3d 挡住或复用**（见 §4.2 的去重陷阱）。

### 3.3 有效记录日定义

满足以下任一条件的日期计 1 个有效记录日（同一天多次记录只计 1 个）：

- 新增记账记录（Finance）
- 新增或完成待办（Todo）
- 完成习惯打卡（Habit）
- 新增想法/观点记录（Thought）
- （V1.1 增强）健康数据同步 + 已授权；Chat 记录型交互

### 3.4 触发标准

| 阶段 | 条件 | 产物 |
| --- | --- | --- |
| 养成期 | 0-2 个有效记录日 | 胶囊提示「Holo 正在认识你」 |
| 轻量观察 | 累计 3 个有效记录日 **且累计覆盖 ≥2 个模块** | 生成 `MemoryInsight(.weekly, light3d)` |
| 完整观察 | 累计 7 个有效记录日 | 生成 `MemoryInsight(.weekly, full7d)` |

跨模块门槛是**窗口级**（累计覆盖 ≥2 模块），不是每天 ≥2 模块——避免误伤前几周只记账或只打卡的真实新用户。连续 7 个自然日但有效记录不足 → 数据不足兜底，不伪造洞察。

## 4. 关键落点（实施者照这里改代码）

> 所有 file:line 基于三轮审查的代码核实，实施前请用编辑器跳转复核一次（行号可能随改动漂移）。

### 4.1 [P0 · Day-1] consent 最终防线

**问题**：`MemoryInsightService.generateInsight` 的 **8 个调用点当前全部无 consent 检查**，未授权时靠底层 `HoloBackendAIProvider.ensureDataProcessingConsent()`（`:254`）抛 `APIError.serverError` 兜底，UI 显示 `.failed` / 「请稍后重试」。这是上周 commit `0c7f539`（修手动发送静默失败）的同类未修实例。

**8 个调用点**（3 文件）：

| 入口 | 位置 | 类型 |
| --- | --- | --- |
| `MemoryGalleryViewModel.generateInsight` | `MemoryGalleryViewModel.swift:819` | 手动 |
| `IntentRouter.handleGenerateMemoryInsight` | `IntentRouter.swift:949` | Chat 意图 |
| `handleBackgroundTask` 日/周/月 | `MemoryInsightBackgroundService.swift:94/118/143` | 后台 |
| `checkForegroundCompensation` 日/周/月 | `:190/215/240` | 前台补偿 |

**解法（service 一处覆盖全部 8 入口）**：

1. `MemoryInsightService.generateInsight`（`:102` 的 `guard !isGenerating` **之前**）加：

   ```swift
   guard HoloAIFeatureFlags.aiDataProcessingConsentGranted else {
       throw MemoryInsightError.aiDataProcessingConsentRequired
   }
   ```

2. 新增 `MemoryInsightError.aiDataProcessingConsentRequired` case（`MemoryInsightModels.swift:212`，现有 5 case 旁）。

3. 入口友好兜底：
   - `MemoryGalleryViewModel.swift:851` switch 加 `.aiDataProcessingConsentRequired → .needConsent`（跳过 `.failed`，参考现有 `.aiNotConfigured → .notConfigured`）。
   - `IntentRouter.swift:963` 改 `catch let error as MemoryInsightError` + 专属文案「开启 AI 数据处理授权后可生成本周观察」（当前「请稍后重试」语义错误）。

4. 后台/前台补偿路径（`:101/126/197/223`）**无需改**——它们 `do/catch + logger.error` 静默跳过，正是未授权时想要的行为。

> 不采用「在 N 个入口各加 guard」——已被证明会漏（首轮 4 处、二轮补到 6 处，实际 8 处），且未来新增入口还会漏。service 最终防线是唯一能覆盖全部入口且防未来漏的方案。

### 4.2 [P0 · Day-1] observationStage 接入去重链路

**问题**：`checkSimilarityWithRecent`（`MemoryInsightService.swift:381-404`）通过 `fetchRecentReadyInsights` 取**同 periodType+promptVersion 最近 3 条 ready，不限周期**，Jaccard > 0.85 即复用。3 天 light3d 刚生成（ready/weekly）必在 7 天 full7d 的 recent 列表里，且前 3 天数据是后 7 天子集 → 相似度高 → **full7d 被判定相似、复用 light3d、不生成**。`snapshotHash` 救不了（它只进缓存闸 `:138-150`，不进 checkSimilarityWithRecent）。

**解法**：加 `observationStage` 字段并接入 3 处查询谓词（缺一不可）：

| # | 位置 | 修订 |
| --- | --- | --- |
| 1 | `MemoryInsightRepository.swift:266-291` `fetchRecentReadyInsights` | 谓词加 `observationStage == %@`（**最关键**，修复跨 stage 误杀） |
| 2 | `MemoryInsightRepository.swift:28-39` 主 `fetchInsight` | 谓词加 stage，避免取错条 |
| 3 | `MemoryInsightRepository.swift:158-173` `markStaleIfNeeded` | 谓词加 stage，避免漏标/错标 |

实体定义：`CoreDataStack+MemoryInsightEntity.swift:14` `createMemoryInsightEntity()` 加 `observationStage` 的 `NSAttributeDescription(isOptional: true)`；`MemoryInsight+CoreDataProperties.swift` 加 NSManaged 属性 + factory 默认值。`checkSimilarityWithRecent` 调用方透传当前 stage。

### 4.3 [P0] 有效记录日 builder

**新建统一 builder**，禁用 `MemoryInsightContextBuilder.effectivePeriodRange(minDays:)`（`MemoryInsightContextBuilder.swift:337-352`）——其语义是日历跨度天数，非有效记录日，直接复用会把「周一到周三的 3 天日历跨度」误判成「3 个有效记录日」。

要求：

- 跨 Finance / Todo / Habit / Thought（V1 四类）按天聚合 + `Set<startOfDay>` 去重。
- **自带缓存层**（仿 `HoloWidgetSnapshotService` 快照模式），避免每次首页 `onAppear`（`HomeView.swift:291/461`）全表扫描 4 张 Core Data 表。
- 累计窗口覆盖 ≥2 模块才算达标（窗口级，非每天级）。
- V1 数据源：`FinanceRepository+Aggregation.swift` / `TodoRepository+Stats.swift` / `HabitRepository.swift` / `ThoughtRepository.swift`。

### 4.4 [P0] HomeScheduleService 排序层升级

现状是单行无状态纯比较（`HomeScheduleService.swift:142` `sorted{ $0.urgency > $1.urgency }.first`），无 tiebreaker、无保护期、无曝光记录。需升级为带状态 Ranker：

| 增强 | 要求 |
| --- | --- |
| 业务优先级 | 引入 §2.4 的 P0-P5 维度，weeklyObservation 给 P0，不复用 `.pending` |
| 24h 保护期 | 新观察生成后 24h 内优先展示；保护期内 P1 逾期任务仍可抢占 |
| 曝光持久化 | 「每天最多 N 次」需持久化（UserDefaults 轻量够用）；**口径定义为 `currentState` 实际变化次数**，非 `refresh()` 调用次数（后者被 5min 定时器 + 回前台 + todo 变更三路刷爆） |
| 稳定 tiebreaker | 业务优先级 → module → ID 字典序，避免 urgency 相等时胶囊文案抖动 |
| 候选构建器 | 新增 `buildWeeklyObservationReminderState()`，读 `MemoryInsight(.weekly)` 状态（notEnoughData/generating/ready/failed/read/unread） |

注意：`ScheduleReminderState`（`:41-50`）当前全 `let` 不可变、无稳定 ID。加 ID / 展示状态时，若展示状态需运行期可变，拆成「不可变候选 + 可变展示记录」两个模型，避免与 `@Published` 流冲突。

### 4.5 [P1] HoloAI 卡片渲染

V1 基于现有 `MemoryInsightPayload` / `cardsJSON` 升级展示结构（§2.3），承载「主观察 + 证据 + 建议 + 下一步」。**不强切 Agent**——Agent 多轮生成作为 V1.0b feature flag 灰度或 V1.1，避免把首页胶囊和多轮 LLM 成本/失败形态绑死在同一首发。

## 6. 约束与禁区

### 6.1 实施约束（来自审查）

- **G5**：Agent 内核不首发强切，V1 用现有 `MemoryInsightPayload`。
- **G6**：跨模块门槛窗口级，非每天级。
- **G7**：提审前补 PrivacyInfo + 隐私问卷 + Review Notes。
- **M4**：实施前补成本测算——单次 weekly 观察生成 3-5 次后端 LLM 调用（`runLoop` 每轮 1 次 + 解析重试），养成期一周可能触发 2 次。作为后端网关限流/预算依据。

### 6.2 编码约定（项目 CLAUDE.md 摘要）

- 禁止 `!`（用 `if let` / `guard let`）；禁止 `print()`（用 `Logger`）；错误必须 `try-catch`。
- Core Data：关系必须有反向；禁止 `refreshAllObjects()`；`Repository.init()` 必须零 I/O；数据变更后 `await` 刷新，禁止 fire-and-forget；线程安全用 `NSManagedObjectID` 跨线程传递。
- 不可变模式：新建对象而非原地修改。
- 数据变更后页面不更新 → 查开发规范第 13 节 Checklist。

### 6.3 非目标

- 不做埋点（无承接系统）。
- 不新增站内信中心。
- 不把首页作为洞察详情页。
- 不做心理/医疗诊断或因果结论。
- 不做随机轮播式首页提醒。
- 未授权 AI 数据处理时不生成 AI 洞察。
- 不让 3 天轻量观察冒充完整周报。

## 7. 验收标准

### 7.1 consent 与安全

- [ ] 未授权时 8 个 generateInsight 入口都不进入 AI 生成（service 最终防线）
- [ ] 手动入口（MemoryGallery / Chat 意图）未授权时显示授权引导，非 `.failed`
- [ ] 后台/前台补偿未授权时静默跳过，不写 failed、不反复重试

### 7.2 observationStage 与去重

- [ ] light3d 与 full7d 独立生成，互不挡/不复用
- [ ] checkSimilarityWithRecent 按 stage 分组比较
- [ ] 旧用户升级自动迁移通过，4 字段可持久化

### 7.3 有效记录日

- [ ] 3 天 + ≥2 模块触发 light3d；7 天触发 full7d
- [ ] 同天多次记录只计 1
- [ ] 首页刷新不全表扫（缓存命中）

### 7.4 首页胶囊

- [ ] 新观察未读可优先展示（P0）
- [ ] 逾期任务长期优先，不被养成进度压掉
- [ ] 24h 保护期后任务可重新优先
- [ ] 「今天不再显示」当天生效
- [ ] 杀进程重启曝光计数不丢、文案不抖动

### 7.5 HoloAI 卡片

- [ ] 点击胶囊进入 HoloAI
- [ ] 卡片含状态判断/证据/建议/下一步
- [ ] light3d 标注「初步观察」，承诺文案合规
- [ ] 已读后首页不再把同条观察当新观察顶置

### 7.6 兜底

- [ ] AI 失败有重试/稍后再试，不空白
- [ ] 数据不足不伪造洞察
- [ ] 未授权点击能看到授权说明 + 开启按钮
