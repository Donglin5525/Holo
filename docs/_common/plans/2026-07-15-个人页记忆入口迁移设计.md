# 个人页记忆入口迁移设计

## 目标

减少设置页的信息密度，把“用户希望 Holo 如何记住自己”归入个人数据资产；同时让开发验收入口与普通用户入口保持隔离。

## 信息架构

### 个人页

按以下顺序展示：

1. 个人档案
2. 我的目标
3. 我的记忆
4. 开发者工具（仅 Debug）

“我的记忆”在个人页只展示一张入口卡片，不直接平铺多个开关。卡片副标题根据两个记忆开关显示简短状态，点击后进入二级页。

### 我的记忆二级页

二级页集中承载：

- 自动形成记忆
- 记忆辅助回答
- 记忆管理
- 普通记忆与健康相关记忆的存储说明

两个开关继续复用 `HoloMemorySettings.shared`，记忆列表继续复用 `HoloMemoryCenterView`，不改变现有存储、同步、反馈或查询逻辑。

### 开发者工具

个人页底部在 `#if DEBUG` 中增加“AI 记忆实验室”，直接进入 `AIMemoryLabView`。Release 不编译入口，也不展示任何实验室文案。

实验室内部继续承载手动观察、领域状态、跨域融合、查询模拟器和记录检查，不把这些内部概念放入普通个人页。

## 设置页收口

从设置页移除：

- 整块“AI 记忆”（两个开关、记忆管理）
- 重复的“个人档案”入口

从 Debug `AISettingsView` 移除重复的“记忆管理”Section，包括：

- AI 记忆实验室入口
- 记忆管理入口
- 两个记忆开关
- 手动触发观察

`AISettingsView` 继续保留 Provider、数据处理授权、API Key、模型、连接测试、Prompt、分类学习、Agent 灰度和危险操作等开发配置。

## 交互与状态

- 个人页“我的记忆”副标题只使用用户语言，不展示置信度、调用次数、dirty、Validator 等内部指标。
- 开关关闭只改变后续形成或回答读取行为，不删除已有记忆。
- “记忆管理”仍以现有页面展示来源及“准确 / 不准确 / 纠正 / 不再使用”。
- 页面迁移不触发任何网络请求、数据迁移或仓库重建。

## 验收标准

- 设置页不再出现“AI 记忆”和“个人档案”。
- 个人页出现“我的记忆”，点击后可操作两个开关并进入记忆管理。
- Debug 个人页出现“AI 记忆实验室”，可以进入现有实验室。
- Release 个人页不出现开发者工具或实验室入口。
- 原有记忆 Store、开关值、记忆记录和反馈结果保持不变。
- Debug 与 Release 构建均成功；Release 二进制不新增实验室入口文案。
