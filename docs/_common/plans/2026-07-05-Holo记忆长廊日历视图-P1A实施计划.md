## 二、关键技术决策（基于代码核实，纠正 spec 一处过严）

1. **`CalendarEvent` = 单条事件（1:1 实体）**，非聚合。月历的色阶/色条由渲染层从 `[CalendarEvent]` 计算（P1B）。**不复用 `MemoryItem`**——它是去模块化的展示 struct（`subtitle` 已把完成态/时间烘成字符串），日历需要保留模块身份 + 原始实体引用。

2. **区间统一半开 `[start, end)`**（最危险坑，4 模块现状不一致）：
   | 模块 | 现状 | P1A 处理 |
   |------|------|---------|
   | Finance `getTransactions(from:to:)` | 半开 `>= AND <`（`FinanceRepository+Aggregation.swift:50`） | **直接复用**，不新增 |
   | Habit `getRecords(for:in:)` | 闭区间 `>= AND <=`（`HabitRepository.swift:344`） | **新增**半开方法 |
   | Todo `getCompletionTrend` | 闭区间（`TodoRepository+Stats.swift:79`）；同文件 `getCompletionStats` 半开（`:107`） | **新增**半开方法（避开不一致） |
   | Thought `fetchAll` | 无 range（`ThoughtRepository.swift:48`） | **新增**半开方法 |
   > **纠正 spec §4.3**：spec 写「新增 `fetchTransactions(in:)`」，实际 Finance 已可复用，P1A 财务零新增。

3. **软删/归档字段各模块独立写 predicate**：
   - Thought：`isSoftDeleted == NO AND isArchived == NO`
   - Todo：`deletedFlag == NO AND archived == NO`
   - Habit：record 无软删；用 `isArchived == NO` 建习惯 `habitMap`
   - Transaction：无软删过滤

4. **同步/async 混合**：Finance 是 `async throws`，Habit/Todo/Thought 是同步（`throws` 或吞错）。Provider 内统一调度，同步方法直接调、不强行包 `async`。

5. **习惯反查用 `habitMap`**（不依赖 `record.habit` 关系），复刻 `MemoryGalleryViewModel.fetchHabitRecords()` `:298-322` 模式。

6. **测试栈**（项目已有稳定范式，TDD 可行）：
   - Repository 区间查询测试 → 抄 `HoloTests/Services/AI/TopicRepositoryTests.swift:15-39`（in-memory `NSPersistentContainer` + `NSInMemoryStoreType` + `Repo(context: ctx)` 注入）
   - 聚合纯函数测试 → 抄 `HoloTests/Views/MemoryGallery/MemoryGalleryViewModelConstellationTests.swift`（不碰 Core Data，`makeDate(year:month:day:)` helper 造确定性日期）

## 三、任务拆解（TDD，按依赖顺序）

### T1 · 模型层（纯值类型，零 I/O）
**新增**
- `Models/Calendar/CalendarModule.swift`：`enum CalendarModule: CaseIterable { finance/habit/todo/thought/health }`，`color: Color`、`iconName: String`（SF Symbol 新增前 `NSImage(systemSymbolName:)` 验证）
- `Models/Calendar/CalendarEvent.swift`：`struct CalendarEvent: Identifiable { id, module, date, title, detail?, originID: NSManagedObjectID }`
- `Models/Calendar/CalendarEventsResult.swift`：`struct CalendarEventsResult { events: [CalendarEvent], moduleStates: [CalendarModule: CalendarModuleLoadState] }` + `enum CalendarModuleLoadState { loaded/empty/failed(message) }`

**验收**：编译过；纯值类型，无 Core Data 依赖。

### T2 · `CalendarRangeBuilder`（区间生成，纯函数，TDD 先行）
**新增** `Services/Calendar/CalendarRangeBuilder.swift`
- `static func weekRange(around date: Date) -> DateInterval`：周一首（`Calendar(identifier: .gregorian)` + `firstWeekday = 2`），`[weekStart, nextWeekStart)` 半开
- `static func dayRange(_ date: Date) -> DateInterval`：`[startOfDay, nextStartOfDay)`
- locale 用 `zh_CN`（与项目约定一致）

**TDD**（抄 `makeDate` helper）：测跨月/月末/跨年/周一首对齐边界。

**验收**：`weekRange`/`dayRange` 单测绿。

### T3 · 3 个事件级 fetch 契约（Repository 层，TDD）
**HabitRepository**（`Models/HabitRepository.swift` 或 `+Stats.swift`）
- 新增 `func getRecords(from start: Date, to end: Date) -> [HabitRecord]`
- predicate：`date >= %@ AND date < %@`（半开，**无 habitId 过滤**）
- Provider 侧另用 `habitMap`（`isArchived == NO` 的 habits）反查 `record.habitId`

**TodoRepository**（`Models/TodoRepository+Stats.swift`）
- 新增 `func getTasks(completedFrom start: Date, completedTo end: Date) -> [TodoTask]`
- predicate：`completedAt >= %@ AND completedAt < %@ AND deletedFlag == NO AND archived == NO`

**ThoughtRepository**（`Data/Repositories/ThoughtRepository.swift`）
- 新增 `func fetchThoughts(from start: Date, to end: Date) throws -> [Thought]`
- predicate：`createdAt >= %@ AND createdAt < %@ AND isSoftDeleted == NO AND isArchived == NO`

**Finance**：直接复用 `getTransactions(from:to:)`，不新增。

**TDD**（抄 in-memory container 范式）：每个新方法测 ① 边界（`月末 23:59:59` 在内、`次日 00:00:00` 不在内）② 软删/归档被过滤 ③ 空区间返回 `[]`。

**验收**：3 个新方法测试全绿（`test_sim`）。

### T4 · `CalendarEventProvider`（聚合 + 失败态，TDD）
**新增** `Services/Calendar/CalendarEventProvider.swift`
```swift
struct CalendarEventProvider {
    let financeRepo: FinanceRepository
    let habitRepo: HabitRepository
    let todoRepo: TodoRepository
    let thoughtRepo: ThoughtRepository

    func fetchEvents(in range: DateInterval) async -> CalendarEventsResult {
        // 4 模块各自 do-catch：
        //   Finance: await getTransactions(from:to:)  → map to CalendarEvent
        //   Habit:   getRecords(from:to:) + habitMap 反查 → map
        //   Todo:    getTasks(completedFrom:completedTo:) → map（completedAt 为 nil 的跳过）
        //   Thought: try fetchThoughts(from:to:) → map
        // 失败 → moduleStates[.x] = .failed(message)，记 Logger，不阻塞其他
        // 成功 → events 合并按 date 升序；moduleStates[.x] = events.isEmpty ? .empty : .loaded
    }

    func fetchDaySummary(_ date: Date) async -> CalendarEventsResult {
        fetchEvents(in: CalendarRangeBuilder.dayRange(date))
    }
}
```
- `init` 只注入 repo，**零 I/O**（CLAUDE.md 约定）
- 事件 `title/detail` 按模块字段契约填充（spec §4.1：记账金额/分类、习惯名/值、待办标题/时间、想法摘要/心情）

**TDD**：① 4 模块聚合 + 排序 ② mock 单 repo throw → `moduleStates[.todo] == .failed`、其他正常 ③ 全失败 → events 空、全 failed ④ 空区间 → events 空、全 `.empty`

**验收**：Provider 单测绿（含失败态）。

### T5 · `CalendarEventDetailSheet`（只读详情）
