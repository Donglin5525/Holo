# HoloAI 记忆默认采用与时间衰减设计

## 产品目标

记忆形成从“用户逐条审批”改为“系统默认学习、用户事后管理”。HoloAI 优先使用已经提炼好的领域与跨域记忆回答状态类问题；精确金额、次数和明细仍回到业务数据源查询。

普通用户只看到“新记住”“待确认”“过去记忆”和准确/不准确/纠正/不再使用操作。置信度、新鲜度、相关度、采用原因码和召回排除原因只在 Debug 记忆实验室展示。

## 状态与采用策略

| 内容 | 初始状态 | 是否参与回答 |
| --- | --- | --- |
| 普通财务、观点、习惯、任务、目标、对话记忆 | `active + none` | 是 |
| 健康、Profile/身份、人生事件、永久事实、假设 | `candidate + none` | 否 |
| 首次普通跨域推断 | `candidate + none` | 否 |
| 获得重复独立证据的普通跨域推断 | `active + none` | 是 |
| 用户确认或纠正 | `active + confirmed/corrected` | 是 |
| 用户标记不准确 | `suppressed + rejected` | 否，可恢复 |
| 已过时 | `archived` | 否，进入过去记忆 |

统一由 `HoloMemoryActivationPolicy` 决定自动采用、等待确认或丢弃。Validator 的证据、锚点、越权和安全校验仍是前置硬门槛，默认采用不能绕过校验。

“不再使用”只删除当前记录，不创建阻止同类记忆重建的墓碑。新证据未来仍可形成新记录。

## 时间生命周期

- 近期状态半衰期 14 天。
- 阶段记忆半衰期 60 天。
- 稳定模式半衰期 150 天。
- 永久事实不自然衰减，但首次形成仍需确认。
- 有效新鲜度低于 `0.35` 进入后台复查。
- 过期或有效新鲜度低于 `0.20` 退出召回。
- 综合召回分低于 `0.08` 不注入回答。
- 后台压缩把持续低于门槛的非永久记录转为 `archived`。
- 用户确认的阶段状态仍会衰减；用户删除或标记不准确立即生效。

同一稳定记忆获得新证据时沿用稳定 ID 合并证据，并刷新 `lastSupportedAt`，因此会自然回到更高的召回优先级，不产生重复提醒。

## 迁移与唯一写入链

迁移版本升级到 v4：

1. 统一仓库以及旧长期/情景记忆库中的 `candidate + none` 全部转为 `active + none`。
2. 保留 `createdAt`、`lastSupportedAt`、证据和旧 JSON，不把历史记忆伪装成今天生成。
3. 按迁移当天重新计算新鲜度；很旧的记录可立即退出召回，随后进入过去记忆。
4. 迁移有 journal、版本门槛和回滚能力，重复启动幂等。
5. 停止启动 `HoloLongTermMemoryCandidateObserver`，统一领域记忆链成为唯一新写入入口。
6. 历史迁移只产生一条汇总回执。

## 汇总提醒与管理界面

- 个人页“我的记忆”展示“新记住 N 件 / 待确认 N 件”。
- 个人页或 HoloAI 最多每 24 小时显示一次非模态胶囊。
- 关闭胶囊只标记已读，不修改记忆状态。
- 记忆中心分开呈现“想和你确认的”“当前记忆”“过去记忆”。
- 自动采用只显示短暂“新”，不冒充用户确认。
- 回执按观察批次、采用类型和记忆 ID 去重，兼容旧 JSON 缺失字段。

## Debug 验证面

AI 记忆实验室显示：

- 自动采用、待确认、已归档数量；
- 策略版本、采用方式与内部原因码；
- 最后支持时间、当前有效新鲜度；
- 当前是否允许召回；
- 汇总回执未读与迁移去重状态。

模拟器 `full-chain-v2` 使用隔离 Store，覆盖普通自动采用、健康待确认、首次跨域确认、历史迁移、衰减归档、召回、反馈恢复和删除后重建。`full-chain-v1` 的旧 Mock 数据目录不删除。

## 验收口径

- 普通记忆无需点击即可召回；健康、身份、假设、永久事实和首次跨域确认前不可召回。
- 待确认记忆不得作为跨域融合的上游来源。
- 历史候选解除门槛但时间不刷新；低于门槛的历史记录不进入回答。
- 不准确立即退出召回并可改回准确；不再使用后当前记录消失且未来可重新生成。
- 精确明细问题继续走业务数据；状态类问题走领域与跨域记忆。
- Debug、Release 均可构建；相关 standalone 测试实际执行；签名模拟器运行后报告全部通过且 Mock 数据保留。
