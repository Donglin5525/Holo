# 用户自定义里程碑设置 — 功能设计文档

## Context

记忆长廊的里程碑检测完全硬编码在 `MilestoneDetector.swift` 中（连续打卡 [30,365]、累计记账 [100,500]、习惯掌握 30 天），用户无法自定义。本功能将里程碑配置开放给用户，支持开关预设、调整阈值、创建自定义里程碑。

**入口**：设置 → 通用 →「里程碑设置」

---

## 一、数据模型

### 存储方案：UserDefaults + Codable

复用 `DarkModeManager` 的 UserDefaults + `@Published` 单例模式。配置量小（<20 条），无需 Core Data。

### 核心结构

```swift
struct MilestoneTemplate: Codable, Identifiable, Equatable {
    let id: UUID
    var type: MilestoneType
    var threshold: Int          // 含义随 type 变化（天数/笔数/金额分值）
    var isEnabled: Bool
    var isPreset: Bool          // true=预设（不可删除，可开关和调阈值）
    var customTitle: String?    // nil 时自动生成
    var customIcon: String?     // nil 时用类型默认图标
    var sortOrder: Int
}
```

### MilestoneType 扩展

`MemoryTimelineNode.swift` 中新增两个 case（现有 3 个保持）：

| MilestoneType | 说明 | 默认图标 | 新增？ |
|---|---|---|---|
| `.streakDays` | 连续打卡 N 天 | flame.fill | 现有 |
| `.cumulativeCount` | 累计记账 N 笔 | trophy.fill | 现有 |
| `.habitMastery` | 习惯掌握（单习惯连续 N 天） | star.fill | 现有 |
| `.taskCompletion` | 累计完成 N 个任务 | checkmark.circle.fill | **新增** |
| `.totalExpense` | 累计支出达 N 元 | creditcard.fill | **新增** |

新增类型的数据来源均已存在：`TodoTask.completed`、`Transaction.amount + type`。

#### habitMastery 与 streakDays 的关系

`habitMastery` 的检测逻辑与 `streakDays` 本质相同（都是单习惯连续 N 天），区别仅在于展示语义：streakDays 关注"坚持某件事 N 天"，habitMastery 关注"习惯已融入生活"。用户可通过自定义 streakDays + 自定义标题实现同样效果，因此**自定义里程碑的类型 Picker 中不包含 `.habitMastery`**。

habitMastery 的边界处理：检测时应排除所有已启用的 streakDays 阈值，即某个习惯的 streak **不等于**任何一个已启用的 streakDays 阈值时，才触发 habitMastery。这取代了当前的 `< 365` 硬编码上界。

### 默认预设（向后兼容）

升级时写入与当前硬编码完全一致的 5 条预设，全部启用。新增类型的预设**默认不启用**。

### 管理器：MilestoneConfigManager

单例 `ObservableObject`，提供 CRUD 方法：
- `toggleTemplate(id:)` — 开关
- `updateThreshold(id:, threshold:)` — 改阈值
- `addCustomTemplate(...)` — 新增自定义
- `deleteTemplate(id:)` — 删除（仅 isPreset == false）

配置变更时发送 `.milestoneConfigDidChange` 通知，`MemoryGalleryViewModel` 监听后调用 `invalidateCache()` 刷新。

**参考模式**：`Utils/DarkModeManager.swift`

## 二、UI 设计

### 导航方式

`SettingsView` 以 Sheet 弹出，内部包裹 `NavigationView`，但现有代码中所有行均使用 `Button` + 回调模式，未使用 `NavigationLink`。

里程碑设置页沿用相同模式：在 `SettingsView` 中添加 `@State private var showMilestoneSettings = false`，通过 `.sheet(isPresented:)` 弹出 `MilestoneSettingsView`。不使用 NavigationLink push。

SettingsView（设置）
  └─ 通用 section

### 预设行交互

预设行同时包含行点击和 Toggle 两个交互，需**分离手势区域**：
- Toggle 区域：点击仅切换开关，不触发行点击
- 行其余区域：点击弹出 Sheet，内有 Stepper 可调整阈值

实现方式：将 Toggle 放在行的 trailing 位置，使用 `.contentShape(Rectangle())` + `Button` 包裹行内容，Toggle 使用独立的 tap 处理。

### AddCustomMilestoneSheet

半屏 Sheet，字段：

| 字段 | 组件 | 说明 |
|---|---|---|
| 类型 | Picker | 连续打卡天数 / 累计记账笔数 / 累计完成任务 / 累计支出金额（不含 habitMastery） |
| 阈值 | Stepper | 根据类型输入数字 |
| 自定义标题 | TextField（可选） | 留空自动生成 |
| 自定义图标 | 图标选择器（复用现有图标） | 留空用默认 |

## 三、检测逻辑变更

### MilestoneDetector 改造

- 删除 3 个硬编码常量（`streakDaysThresholds`、`cumulativeCountThresholds`、`habitMasteryThreshold`）
- 从 `MilestoneConfigManager.shared.enabledTemplates` 读取配置
- 每个检测方法接收 `[MilestoneTemplate]` 参数，仅在该类型有启用模板时执行

### habitMastery 边界处理

```swift
// 获取所有已启用的 streakDays 阈值
let streakThresholds = enabledTemplates
    .filter { $0.type == .streakDays }
    .map { $0.threshold }

for template in masteryTemplates {
    for habit in habits {
        let streak = HabitRepository.shared.calculateStreak(for: habit)
        guard streak >= template.threshold else { continue }
        // 排除已被 streakDays 里程碑覆盖的阈值
        guard !streakThresholds.contains(streak) else { continue }
        // 生成 MilestoneData...
    }
}
```

### 新增检测方法

- `detectTaskCompletion` — `countResultType` 查询 `TodoTask where completed == YES`，取最大命中阈值
- `detectTotalExpense` — SUM 聚合 `Transaction where type == expense`，取最大命中阈值

### 去重规则

- 连续打卡类：精确匹配（`streak == threshold`）
- 累计类（记账/任务/支出）：取所有命中的最大阈值，只生成一条

## 四、缓存联动

配置变更需及时反映到记忆长廊：

```
MilestoneConfigManager 配置变更
  → 发送 NotificationCenter.default.post(name: .milestoneConfigDidChange)
  → MemoryGalleryViewModel 监听
  → invalidateCache() + refresh()
```

在 `MemoryGalleryViewModel.setupNotifications()` 中新增对 `.milestoneConfigDidChange` 的监听，与现有的 `.financeDataDidChange` 等通知处理方式一致。

## 六、实现阶段

### Phase 3: 新类型 + 打磨
8. 实现 `detectTaskCompletion` 和 `detectTotalExpense`
9. 预设行点击调整阈值交互
10. 更新 CHANGELOG

### 验证方式
- Phase 1 完成后：修改 UserDefaults 中某个阈值为 1，确认记忆长廊能正确检测到
- Phase 2 完成后：在设置页面开关里程碑，返回记忆长廊确认变更生效
- Phase 3 完成后：创建自定义任务完成里程碑，完成任务后确认触发
