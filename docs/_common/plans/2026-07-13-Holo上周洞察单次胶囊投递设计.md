# Holo 上周洞察单次胶囊投递设计

> 日期：2026-07-13  
> 状态：已确认，待实施  
> 范围：首页胶囊、周洞察生成、失败恢复、记忆长廊精准跳转、后端响应治理

## 1. 产品结论

将现有「本周观察」改为「上周洞察」：每次只分析上一完整自然周（周一至周日），生成成功后通过首页胶囊投递一次。用户点击胶囊后立即消费该投递，并直达记忆长廊、自动展开对应洞察。

HoloAI 页面不再展示周观察卡片；养成、授权、生成中和失败状态也不再占用首页胶囊。失败由系统自动恢复，上一条成功洞察始终可查看。

## 2. 目标与非目标

### 2.1 目标

1. 统一周洞察的周期语义，资格判断和生成数据都使用上一完整自然周。
2. 首页胶囊只投递成功结果，单次有效，点击后失效。
3. 点击胶囊后按洞察 UUID 直达并展开记忆长廊的指定洞察。
4. 空响应、超时、非法 JSON 和内容不完整可区分、可诊断、可恢复。
5. 生成失败不覆盖上一条成功洞察，不向用户展示阻塞式失败卡。
6. App 前台启动负责可靠补偿，BGTask 仅作辅助。

### 2.2 非目标

- 不新建第二套周洞察内容实体，继续复用 `MemoryInsight(periodType: .weekly)`。
- 不把周洞察迁移为服务端定时任务；用户数据仍由 iOS 本地聚合后按授权发送。
- 不新增 Push 通知样式；复用现有「上周记忆 / 上周回放」内容语义。
- 不调整月度、季度、自定义洞察的产品入口。
- 不在本次重做记忆长廊整体信息架构。

## 3. 用户体验

### 3.1 生成时机

- App 每周首次进入前台时，检查上一完整自然周是否已有 ready 洞察。
- 已有 ready 洞察则不重复生成。
- 没有 ready 洞察且满足数据门槛时自动生成。
- BGTask 唤醒时执行同一检查，作为前台补偿之外的 best-effort 增强。
- 生成中的同周期请求必须幂等，不能并发重复调用模型。

### 3.2 数据门槛

上一完整自然周内满足以下条件才生成：

- 至少 3 个有效记录日；
- 至少覆盖 2 个业务模块；
- 有效模块沿用 Finance、Todo、Habit、Thought，健康数据只进入洞察上下文，不单独满足模块门槛。

数据不足属于「本周没有足够新变化」，系统静默跳过，不创建 failed 记录，不展示胶囊。

`light3d` 仅保留历史兼容，不再作为每周投递产物；新生成的上一周洞察统一为 `full7d`，这里的 `full7d` 表示完整自然周窗口，而不是要求每天都有记录。

### 3.3 首页胶囊

仅在存在尚未消费的 ready/stale weekly insight 时创建候选：

- 文案优先使用 `上周洞察：{洞察标题}`；空间不足时降级为 `上周洞察已准备好`。
- 业务优先级为 P0，允许覆盖其他首页提醒一次。
- 每条洞察只投递一次，不再使用「每日最多曝光 3 次」规则。
- 胶囊的消费标识绑定 insight UUID，而不是绑定日期或固定字符串。
- 用户点击时先原子写入 `readAt`，再发起 Deep Link；即使导航中断，也不重复投递。
- 用户没有点击时，胶囊可跨前后台继续显示，直到被点击或被更新的周洞察替代。

养成、授权、生成中、失败状态不创建周洞察胶囊候选。

### 3.4 HoloAI

- 删除 ChatView 顶部 `WeeklyObservationCard` 入口和相关本地交互状态。
- HoloAI 聊天、快捷操作和正常消息流保持不变。
- 用户通过自然语言要求生成周洞察时，也生成上一完整自然周；成功后结果进入记忆长廊并等待首页胶囊投递。

### 3.5 记忆长廊精准打开

Deep Link 新增：

```swift
case memoryInsight(insightId: UUID)
```

消费流程：

1. HomeView 打开记忆长廊模块。
2. MemoryGalleryView 接收仍未消费的 `memoryInsight` 目标。
3. ViewModel 按 UUID 加载洞察，并切换 `selectedInsightPeriod = .weekly`。
4. 记忆长廊切到洞察内容区域，不停留在默认日历。
5. Hero 卡切换为完整展开状态并滚动到洞察锚点。
6. 展开完成后清除 `pendingTarget`，避免重复触发。

如果指定 UUID 已被清理或不可用，则回退到最新一条 ready weekly insight；仍无结果时进入记忆长廊洞察空态，不回退日历页。

## 4. 周期与生成架构

### 4.1 单一周期定义

新增纯逻辑周期解析器，作为所有周洞察入口的单一真相源：

```swift
struct WeeklyObservationPeriod: Equatable {
    let start: Date   // 上周周一 00:00
    let end: Date     // 上周周日 00:00，业务查询按需要 +1 天形成半开区间
}
```

后台生成、前台补偿、手动重试、Chat 意图、记忆长廊手动生成和首页查询全部使用同一个解析器。禁止周洞察入口继续调用「当前周到今天」的 `periodRange(.weekly, Date())`。

### 4.2 成功资产与生成尝试分离

Repository 明确提供两类查询：

- `fetchLatestAvailableInsight`：只返回 ready/stale，用于用户内容展示。
- `fetchLatestGenerationAttempt`：返回 generating/failed，用于恢复与诊断。

现有名称误导的 `fetchLatestReadyInsight` 必须改为严格只返回 ready/stale。failed 记录不能覆盖 ready 内容，也不能参与首页胶囊和记忆长廊默认展示。

生成新洞察时可以保留同周期 attempt 记录，但只能在成功后成为可展示资产。失败尝试按清理策略回收，不改变上一条 ready 洞察。

### 4.3 幂等与恢复

幂等键使用：

```text
weekly:{periodStart ISO date}:{sourceSnapshotHash}:full7d
```

恢复策略：

1. 同周期已有 generating 且未超出任务租约时不重复生成。
2. App 进入前台时检查 failed/stale generating attempt。
3. 可恢复错误使用退避：首次失败后 1 分钟、第二次 10 分钟；最多自动重试 2 次。
4. 空响应由后端同请求内额外重试一次，不计入 iOS 的跨请求重试次数。
5. 非可恢复错误（未授权、数据不足、Schema 永久不兼容）不自动重试。
6. 下一周开始后停止恢复更早周的 failed attempt，但保留上一条成功内容。

## 5. AI 响应稳定性

### 5.1 请求契约

- `insight` 使用服务端 `memory_insight_generation` Prompt。
- DeepSeek 洞察请求不再强传 `response_format: json_object`，避免推理模型兼容性问题；仍由 Prompt 强约束纯 JSON。
- 洞察路由输出 token 上限调整为 4096，避免推理内容占满 2048 后最终正文为空。
- iOS 超时必须大于后端上游超时。统一为：后端 60 秒，iOS 70 秒；外层 ViewModel 不再叠加更短的第二层超时。

### 5.2 后端结果校验

后端在向 iOS 返回 200 前检查：

- `choices[0].message.content` 存在且 trim 后非空；
- 若存在 `finish_reason == length`，按截断错误处理；
- 洞察正文至少包含可提取的 JSON 对象；
- 记录 content 长度、reasoning_content 长度、finish_reason 和 requestId，不记录正式用户正文。

第一次结果为空或被截断时，后端使用同一 Prompt、去掉 `response_format` 自动重试一次。第二次仍失败则返回稳定错误码：

- `EMPTY_MODEL_RESPONSE`
- `TRUNCATED_MODEL_RESPONSE`
- `INVALID_INSIGHT_JSON`

后端不负责完整业务 Schema 解码；最终 Schema 校验仍由 iOS `MemoryInsightResponseParser` 完成。

### 5.3 iOS 错误分类与诊断

`MemoryInsightGenerationResult` 增加 requestId；`MemoryInsight` 生成尝试保存 requestId 和错误类别。用户界面不展示技术细节，但内部日志可以按 requestId 定位后端记录。

错误分为：

- 数据不足：正常跳过；
- 未授权：等待授权，不创建 failed；
- 网络、超时、空响应：可恢复；
- 非法 JSON、Schema 不完整：后端已重试后进入可恢复队列；
- 持久化失败：本地错误，不重复调用模型。

## 6. 首页与 Push 的关系

现有 Push 文案已经采用「上周记忆已准备好」「生成一份上周回放」语义，本次复用其周期口径，但不把 Push 作为生成任务的唯一触发器。

- Push 继续按用户设置的周几和时间提醒。
- 如果洞察已经自动生成，Push 点击应直达对应洞察。
- 如果尚未生成，Push 点击进入记忆长廊洞察区域，并触发同一前台补偿检查。
- 首页胶囊与 Push 共用 insight UUID、消费和 Deep Link 语义，避免两个入口打开不同内容。

## 7. 数据迁移与兼容

- 历史 weekly ready/stale 洞察继续可在记忆长廊查看。
- 历史 failed 记录不再成为首页或 HoloAI 展示候选。
- 历史 `readAt == nil` 的周洞察不全部重新投递；只允许目标上一完整周的最新 ready 洞察进入新胶囊逻辑。
- `light3d` 历史记录保持原样，不参与新的每周自动投递。
- 新增 optional Core Data 字段时使用现有轻量迁移机制，并用旧 store 验证升级。

## 8. 测试与验收

### 8.1 纯逻辑测试

- 周一、周中、跨月、跨年时都正确返回上一完整周。
- 上周有效记录不足时不生成。
- ready/stale 查询不会返回 failed。
- 同一 UUID 胶囊点击一次后永久不再成为候选。
- 新一周的新 UUID 可以再次投递。
- Deep Link 指定 UUID、UUID 缺失回退和重复目标均正确处理。
- 重试次数、退避时间和下一周停止恢复符合契约。

### 8.2 后端测试

- insight 不向 DeepSeek 发送 `json_object`。
- 空 content 自动重试一次。
- 第二次空 content 返回 `EMPTY_MODEL_RESPONSE`。
- `finish_reason == length` 返回截断错误。
- 日志只记录元数据，不记录用户正文。
- 既有 chat、intent、agent、health insight 路由不回归。

### 8.3 iOS 集成验收

1. 准备上一周满足门槛的数据，启动 App 后只发起一次 weekly 请求。
2. 成功后 HoloAI 不出现周观察卡，首页出现 P0 胶囊。
3. 点击胶囊后立即消失，重新进入首页不再出现。
4. 记忆长廊自动显示并展开指定上周洞察，不停留在日历。
5. 模拟空响应、超时和非法 JSON，上一条成功洞察仍可查看，首页不显示失败胶囊。
6. 后端全量测试通过，iOS standalone 专项测试实际执行，模拟器 Debug build 通过。

## 9. 发布边界

本方案包含 `HoloBackend/` 路由校验、token 配置和日志元数据改动。iOS 与后端代码完成并推送后，生产仍不会自动生效；必须单独执行 ECS 后端发版，并验证：

- `/v1/health` 正常；
- 生产 insight 路由使用目标模型和 4096 token；
- 空响应返回新错误码而非 200 空正文；
- 真机生成上一周洞察成功；
- 首页胶囊单次投递和记忆长廊精准展开闭环通过。
