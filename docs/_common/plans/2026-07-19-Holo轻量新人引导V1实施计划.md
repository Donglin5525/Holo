# Holo 轻量新人引导 V1 实施计划

> 日期：2026-07-19
>
> 状态：待实施
>
> 产品方案：[2026-07-18-Holo新人引导与首次价值激活方案.md](./2026-07-18-Holo新人引导与首次价值激活方案.md)
>
> 目标：用 3 个轻量页面和 1 次首页入口提示，让新用户知道 Holo 是什么、能做什么、从哪里开始，以及 AI 为什么需要授权。

## 一、实施结论

V1 只实现：

```text
欢迎与昵称
  →
功能与使用方式
  →
AI 数据处理授权
  →
首页 HoloAI 入口提示
```

不在引导内调用 AI、不创建业务数据、不等待用户完成真实操作。

## 二、图片与首页展示决策

### 2.1 首页不放介绍图片

首页是高频工作台，不增加轮播图、Banner、功能海报或多张介绍卡片。

原因：

- 介绍内容只在第一次有价值，长期占据首页会浪费空间。
- 首页已有问候、日程、今日看板、五角形功能入口和底部导航，继续加图片会使首屏更拥挤。
- 用户完成 onboarding 后，首页应该马上进入正常使用状态。
- 图片中的界面截图容易随着功能更新过时，也需要额外维护深色模式和本地化版本。

首页只增加一次性气泡：

> 从这里告诉 Holo 一件事

气泡指向底部导航栏中央的 HoloAI 按钮，关闭后不再出现。

### 2.2 V1 不新增 PNG、JPG 或插画资源

三页视觉全部由 SwiftUI、现有语义色和 SF Symbols 构成。

| 页面 | 视觉表达 | 是否新增图片资源 |
|---|---|---|
| 欢迎页 | 橙色渐变光晕 + `sparkles` 图标 + 简洁文字 | 否 |
| 功能页 | 3 张能力卡 + SF Symbols + 示例胶囊 | 否 |
| 授权页 | `checkmark.shield` / `lock.shield` 图标 + 说明卡 | 否 |
| 首页 | 指向 HoloAI 按钮的气泡 | 否 |

### 2.3 为什么不用产品截图

- 截图信息密度高，新人需要额外理解截图里的 UI。
- 浅色和深色模式需要两套截图。
- 文案或布局变化后截图会失效。
- 截图容易把 onboarding 做成宣传页，而不是使用说明。

如果未来需要更强品牌感，只考虑在欢迎页增加一张统一的 Holo 品牌插画，不在首页增加多张介绍图片；该事项不进入 V1。

## 三、页面结构与视觉规格

### 3.1 通用容器

使用 `HoloLightweightOnboardingView` 管理三页：

```swift
@State private var currentPage = 0
@State private var nicknameDraft = ""
@FocusState private var nicknameFocused: Bool
```

建议采用显式“继续”按钮切换页面，不需要 Coordinator。

通用布局：

```text
┌────────────────────────────┐
│                    跳过    │
│                            │
│        页面视觉区           │
│                            │

│                            │

│                            │
│            ● ○ ○           │
```

通用规格：

## 四、页面一：欢迎与昵称

### 4.1 页面目标

解释 Holo 的定位，同时替换现有首次昵称 Alert。

### 4.2 视觉区

使用 SwiftUI 绘制一个 112pt 的 Holo 标识：

```text
      ╭────────────╮
      │   ✦  ✧     │
      │     ✨      │
      │  橙色光晕   │
      ╰────────────╯
```

实现建议：

- 外层 `Circle` 使用 `RadialGradient`。
- 中心使用 `Image(systemName: "sparkles")`。
- 使用 `Color.holoPrimary` 与 `Color.holoPurple` 的低透明度光晕。
- 动画只做轻微呼吸缩放；Reduce Motion 开启时完全静止。
- 不读取 `StartupSplashArtwork`，避免启动图和 onboarding 重复表达。

### 4.3 文案

标题：`你好，我是 Holo`

说明：

> 你的个人数据助理。帮你记录生活、安排事情，并慢慢看见自己的变化。

昵称字段：`希望我怎么称呼你？（选填）`

按钮：`继续`

### 4.4 状态

- 昵称为空：合法，按钮保持可用。
- 昵称有效：进入下一页时只保留在本地 draft，完成整个 onboarding 时再保存。
- 键盘弹出：页面切换为可滚动，按钮不能被键盘遮挡。
- 点击背景或继续：释放 `nicknameFocused`。

## 五、页面二：功能与使用方式

### 5.1 页面目标

让用户形成三个最小使用心智，不解释全部模块。

### 5.2 三张能力卡

#### 一句话记录

- 图标：`square.and.pencil`
- 标题：`一句话记录`
- 说明：`不用先找入口，直接告诉 Holo 发生了什么。`
- 示例：`“午饭花了 35 元”`

#### 直接问自己的数据

- 图标：`bubble.left.and.text.bubble.right`
- 标题：`直接问自己的数据`
- 说明：`想知道最近的情况，直接用自然语言提问。`
- 示例：`“这个月餐饮花了多少？”`

#### 回看生活变化

- 图标：`book.closed`
- 标题：`回看生活变化`
- 说明：`记忆长廊会把不同模块的记录串成时间线。`
- 示例：`财务、待办、习惯和想法会逐渐形成生活轨迹。`

### 5.3 卡片规格

```text
┌──────────────────────────┐
│  [图标]  一句话记录       │
│          说明文字         │
│  “午饭花了 35 元”         │
└──────────────────────────┘
```

- 卡片使用 `Color.holoCardBackground`。
- 图标底使用主色 10% 透明度圆角方块，尺寸 40pt。
- 示例使用浅色胶囊，不做聊天气泡截图。
- 三张卡纵向排列；小屏和大字体下允许滚动。
- 页面没有加载、错误或空状态。

## 六、页面三：AI 数据处理授权

### 6.1 页面目标

在用户进入 HoloAI 前，用清晰、非技术化的方式说明授权。

### 6.2 视觉区

- 72pt 圆形浅色底。
- 中心图标：`checkmark.shield`。
- 已授权时使用 `holoSuccess`；未授权时使用 `holoPrimary`。
- 不使用人物插画、服务器示意图或第三方 Logo。

### 6.3 文案与操作

标题：`开始前，确认一项授权`

正文：

> 当你使用 HoloAI、AI 洞察或语音转文字时，完成这项功能所需的问题和数据摘要，会经 Holo 后端发送给第三方 AI 或语音服务处理。

说明项：

- 只在使用相关 AI 功能时处理必要数据。
- 不授权仍可使用本地记账、待办、习惯和观点。
- 可以在 HoloAI 数据授权中随时开启或撤回。

操作：

- 主按钮：`同意并开始使用`
- 次级按钮：`暂不授权，先进入 Holo`
- 文本链接：`查看隐私政策`

### 6.4 行为

同意：

```text
grant AI consent
  → 保存有效昵称
  → 标记 onboarding 完成
  → 关闭全屏页
  → 显示首页 AI 入口提示
```

暂不授权：

```text
不修改 consent
  → 保存有效昵称
  → 标记 onboarding 完成
  → 关闭全屏页
  → 显示首页 AI 入口提示
```

## 七、首页一次性入口提示

### 7.1 首页不新增图片区域

不修改首页主内容结构，不增加 Banner，不改变五角形按钮和今日看板的布局。

仅在底部 HoloAI 按钮上方叠加一个气泡：

```text
          ┌───────────────────────┐
          │ 从这里告诉 Holo 一件事 │
          │ 例如：午饭花了 35 元   │
          └──────────┬────────────┘
                     ▼
          个人       ✨        记忆长廊
```

### 7.2 交互

- 气泡最大宽度 260pt。
- 背景使用 `.ultraThinMaterial`，增加 1pt `holoBorder`。
- 点击气泡：关闭提示并打开 `ChatView`。
- 点击 HoloAI 按钮：关闭提示并打开 `ChatView`。
- 点击其他区域：只关闭提示。
- 气泡只由本次 onboarding 完成或跳过回调触发，不增加第二个持久化 key。
- 旧用户迁移不显示气泡。

## 八、状态模型

新增：

```swift
struct LightweightOnboardingSettings {
    static let completedKey = "holo_onboarding_lightweight_v1_completed"

    private let defaults: UserDefaults

    var isCompleted: Bool { defaults.bool(forKey: Self.completedKey) }
    func markCompleted() { defaults.set(true, forKey: Self.completedKey) }
}
```

只保存一个 Bool：

- 不保存当前页。
- 不保存昵称草稿。
- 不保存授权页停留状态。
- 不保存首页提示状态。

中途杀掉 App 后从第一页重新开始，属于 V1 可接受行为。

## 九、新老用户与 Deep Link

### 9.1 判断顺序

```text
截图 / 模拟器验收模式？ → 跳过
新 completed key 为 true？ → 跳过
旧昵称 onboarding 已完成？ → 迁移并跳过
本次带 Deep Link？ → 延后，不写 completed
否则 → 展示三页 onboarding
```

### 9.2 HomeView 接线

在 `HomeView.onAppear` 中：

1. 先记录本次进入时是否存在 `deepLinkState.pendingTarget`。
2. 调用现有 `handleDeepLink()`。
3. 如果本次携带 Deep Link，设置当前会话 `didDeferOnboardingForDeepLink = true`。
4. 只有没有 Deep Link、没有其他 cover、且满足新用户条件时才展示 onboarding。

不在 `handleDeepLink()` 执行后重新读取 `pendingTarget`，因为目标可能已被消费。

## 十一、实施任务

### Task 1：最小状态封装

- 新增 completed key。
- 支持注入 `UserDefaults` 便于测试。
- 增加旧昵称状态迁移判断。
- 写 standalone tests。

完成标准：新用户、已完成用户和旧用户迁移判断正确。

### Task 2：三页 onboarding UI

- 实现通用容器、页面指示器和跳过按钮。
- 实现欢迎页和昵称 draft。
- 实现三张能力卡。
- 实现 AI 授权页和隐私政策 Sheet。
- 接入深色模式、Dynamic Type、VoiceOver 和 Reduce Motion。

完成标准：三页无需网络即可完整走通。

### Task 3：HomeView 接线

- 移除旧昵称 Alert。
- 接入 fullScreenCover。
- 实现统一完成、暂不授权和跳过回调。
- 实现 Deep Link 优先级。
- 显式跳过截图和模拟器验收场景。

完成标准：新用户展示，老用户不展示，Deep Link 不被拦截。

### Task 4：首页 AI 入口提示

- 在底部导航栏上方增加轻量气泡。
- 点击气泡复用现有 `showChatView = true`。
- 点击外部关闭。
- 确保不遮挡个人和记忆长廊按钮。

完成标准：只在本次 onboarding 完成后出现一次，不改变首页布局。

### Task 5：验证

- standalone 状态测试。
- Swift parse / build。
- Debug 与 Release simulator build。
- 新用户、老用户、跳过、同意、拒绝和 Deep Link 人工流程。

## 十二、测试矩阵

| 场景 | 预期 |
|---|---|
| 全新安装 | 展示三页 onboarding |

## 十三、验收标准

- [ ] 首页没有新增介绍图片、Banner 或轮播图。
- [ ] `Assets.xcassets` 没有新增 onboarding 图片资源。
- [ ] 新用户看到且只看到 3 页引导。
- [ ] 每页都可以跳过。
- [ ] 昵称可选，不阻塞流程。
- [ ] 三类核心用法能够在一页内理解。
- [ ] AI 授权同意与暂不授权都能正常进入首页。
- [ ] 麦克风、通知、HealthKit 没有在 onboarding 中申请。
- [ ] 旧昵称 Alert 被移除。
- [ ] 首页只增加一次 HoloAI 入口气泡。
- [ ] 不修改 Chat、Repository、后端和 Prompt。
- [ ] standalone tests、Debug/Release build 和人工流程通过。

## 十四、风险控制

| 风险 | 控制 |
|---|---|
| 三张能力卡在小屏过长 | 使用 ScrollView，按钮保持可到达 |
| 授权文案过多 | 主文案控制在一段，详细内容放隐私政策 |
| 首页气泡遮挡操作 | 最大宽度 260pt，点击外部立即关闭 |
| 旧用户误展示 | 复用旧昵称完成 key 迁移 |
| Deep Link 被欢迎页打断 | 首次进入时先记录并处理 Deep Link |
| 动画影响可访问性 | Reduce Motion 下关闭呼吸动画 |

## 十五、发布边界

本方案只有 iOS UI 和 UserDefaults 改动：

- 不修改后端。
- 不修改 Prompt。
- 不新增网络请求。
- 不需要后端部署。

完成定义：实现、状态测试、Debug/Release build、模拟器完整流程与旧用户迁移验证全部通过。
