## 1. 问题与目标

### 1.1 问题

记忆管理页的候选仍在沿用洞察卡片表达方式：

- "体重习惯连续中断" — 用户无法判断是目标终止、习惯中断还是没记录
- "任务全部闭环" — 内部效率话术，非用户语言
- "任务清零，支出偏高" — 两个语义混在一起，不利于审核和召回

根因：`HoloLongTermMemoryCandidateObserver` 把洞察卡片的 `title`/`body` 直接当长期记忆的 `title`/`summary`，且所有候选硬编码为 `.recurringPattern` 类型。

### 1.2 设计目标

| # | 目标 |
|---|------|
| 1 | 用户在记忆管理页能快速判断 Holo 记住了什么，是否准确 |
| 2 | HoloAI 在对话/洞察/规划中能按场景召回记忆，且知道不能如何误用 |
| 3 | 减少冷数据和统计节点污染长期记忆 |
| 4 | 接通并验证现有长期记忆召回链路（当前断路） |
| 5 | 情景记忆和长期记忆边界清晰 |

### 1.3 非目标

- 不把记忆管理页做洞察详情页
- 不在记忆卡片输出行动建议
- 不把所有洞察卡片晋升为长期记忆
- 不做手动分类编辑器

## 2. 设计决策

### 2.1 语义类型 + 双摘要 + 晋升矩阵

| 维度 | 决策 | 理由 |
|------|------|------|
| 记忆分类 | 5 种语义类型替代旧来源类型 | 旧类型按数据来源分，新类型按 AI 使用场景分 |
| 摘要分层 | `displaySummary`（用户）+ `aiUseSummary`（AI） | 一段文字无法同时满足用户审核和 AI 召回 |
| 候选生成 | 改洞察 prompt 一次输出，不新增 LLM 调用 | 避免延迟和成本翻倍 |
| 旧数据 | 保留 `semanticType = nil`，UI 标记"旧格式" | 自动分类不可靠，诚实标记更安全 |
| 情景记忆 | 不与 `statMilestone` 合并，保持独立 | 两者用途不同（近期观察 vs 成就展示） |

### 2.2 Prompt 策略

LLM 只输出 3 个候选字段，其余本地补默认值：

| 字段 | 来源 | 说明 |
|------|------|------|
| `semanticType` | LLM | 5 种语义分类 |
| `displaySummary` | LLM | 用户可审核的事实摘要 |
| `aiUseSummary` | LLM | AI 召回上下文 + 误用边界 |
| `useScopes` | Mapper 本地 | 根据 semanticType 填充 |
| `prohibitedInferences` | Mapper 本地 | 根据 semanticType + sensitivity 填充 |
| `confidence` | LLM 优先 | 缺失时按 evidence 数量 fallback |
| `sensitivity` | LLM + 本地 | 本地只允许升级，不允许降级 |
| `expiresAt` | Mapper 本地 | driftSignal 默认 14-30 天 |

### 2.3 卡片类型限制

首版只有以下卡片类型允许输出 `memoryCandidate`：

- ✅ `habit` / `finance` / `task` / `milestone`
- ❌ `overview` / `anomaly` / `thought` / `cross_domain`（默认不晋升）

## 3. 数据模型

### 3.1 新增枚举

```swift
// HoloLongTermMemoryModels.swift

/// 语义类型：决定记忆如何被 AI 使用
enum HoloMemorySemanticType: String, Codable, Equatable {
    case phaseShift       // 阶段变化
    case stablePattern    // 稳定习惯
    case driftSignal      // 偏离提醒
    case lifeEvent        // 人生节点
    case statMilestone    // 轻量统计收藏
}

/// 使用场景：决定记忆在哪些场景被召回
enum HoloMemoryUseScope: String, Codable, Equatable {
    case coreContext       // 核心上下文，所有场景可用
    case recentInsight     // 近期洞察
    case goalPlanning      // 目标规划
    case retrospective     // 年度回顾/记忆长廊
    case displayOnly       // 仅展示，不参与 AI 召回
}
```

### 3.2 模型扩展

```swift
// HoloLongTermMemoryModels.swift — HoloLongTermMemory 扩展字段

struct HoloLongTermMemory {
    // 旧字段（保持不变）
    var type: HoloLongTermMemoryType
    var title: String
    var summary: String

    // 新增字段（全部 Optional，兼容旧 JSON）
    var semanticType: HoloMemorySemanticType?       // nil = 旧格式
    var displaySummary: String?                      // 优先于 summary 展示
    var aiUseSummary: String?                        // 注入 AI prompt
    var useScopes: [HoloMemoryUseScope]?             // nil 时按 fallback 处理
    var prohibitedInferences: [String]?              // nil 时视为空

    // 原有字段（不变）
    var id: String
    var confidence: HoloMemoryConfidence
    var confirmationState: HoloMemoryConfirmationState
    var sensitivity: HoloMemorySensitivity
    var evidence: [HoloLongTermMemoryEvidence]
    var createdAt: Date
    var updatedAt: Date
    var expiresAt: Date?
}
```

**关键约束**：
- 所有新字段必须 `Optional`（旧 JSON 中无这些 key，Swift 自动 Codable 对 Optional 字段缺失时返回 nil）
- 不新增非 Optional 字段，否则旧 JSON 解码会抛 `keyNotFound`

### 3.3 旧数据 fallback 规则

| 场景 | 行为 |
|------|------|
| `semanticType == nil` | UI 显示"旧格式记忆"标记，不参与语义筛选 |
| `displaySummary == nil` | 使用旧 `summary` 展示 |
| `aiUseSummary == nil` | Phase 0-3 使用 `title：summary` 格式注入 |
| `useScopes == nil` | 按旧 `type` 做粗粒度筛选（等同 Phase 0 行为） |
| `prohibitedInferences == nil` | 不注入误用边界 |

### 3.4 semanticType → 旧 type 兼容映射

新数据写入时需要给旧 `type` 字段赋值（供未升级代码使用）：

| semanticType | 兼容旧 type |
|-------------|------------|
| `phaseShift` | `.profileBackedFact` |
| `stablePattern` | `.recurringPattern` |
| `driftSignal` | `.recurringPattern` |
| `lifeEvent` | `.profileBackedFact` |
| `statMilestone` | `.recurringPattern` |

**注意**：此映射仅用于新数据写入时赋值旧 `type`，**不用于旧数据反向推 `semanticType`**。

## 4. 记忆类型定义

### 4.1 阶段变化 `phaseShift`

> 用户跨过了一个阶段，或长期状态出现了可被证据支撑的台阶变化。

- **晋升**：默认待确认，涉及健康/财务/职业必须确认
- **过期**：长期保留
- **默认 useScopes**：`[.coreContext, .recentInsight, .goalPlanning]`
- **默认 prohibitedInferences**：`["不要在没有新证据时假设变化仍在持续"]`

示例：
- 标题：体重进入新的稳定区间
- 页面摘要：近 30 天体重稳定低于上一阶段区间，相关记录有连续证据。
- AI 摘要：用户身体状态出现阶段性变化。健康洞察可作为长期进展背景，但仍需结合最近记录判断是否稳定。

### 4.2 稳定习惯 `stablePattern`

> 用户长期重复出现、对个性化理解有价值的行为倾向。

- **晋升**：多周期证据 + 低敏感时可静默写入；单周期证据继续观察
- **过期**：长期保留
- **默认 useScopes**：`[.coreContext, .recentInsight]`
- **默认 prohibitedInferences**：`["不要表述为强制偏好", "不要在没有后续证据时推断一直持续"]`

示例：
- 标题：任务整理形成固定节奏
- 页面摘要：最近数周内，任务创建和完成集中出现在固定时间段。
- AI 摘要：用户可能更适合在该时间段处理任务规划。后续提醒和目标拆解可参考这个节奏，不要把它表述为强制偏好。

### 4.3 偏离提醒 `driftSignal`

> 用户近期偏离了自己曾经在意、持续追踪或明确设定的目标/节奏。

- **晋升**：默认待确认，不静默写入
- **过期**：14-30 天，必须过期
- **默认 useScopes**：`[.recentInsight]`
- **默认 prohibitedInferences**：`["不要归因为懒惰、放弃或失败", "不要假设用户不再关注该目标"]`

示例：
- 标题：身体记录最近不连续
- 页面摘要：近 14 天内，体重、平板支撑、俯卧撑等身体相关记录多次中断。
- AI 摘要：用户近期身体状态追踪不稳定。健康或习惯洞察可参考此上下文，但需要确认用户当前是否仍关注这些目标，避免直接归因为懒惰、放弃或失败。

### 4.4 人生节点 `lifeEvent`

> 来自想法、对话、任务、财务或档案更新中的重要生活事件。

- **晋升**：默认要求确认，敏感事件不允许静默写入
- **过期**：长期保留
- **默认 useScopes**：`[.coreContext, .goalPlanning, .retrospective]`
- **默认 prohibitedInferences**：`["不要主动暴露敏感细节除非用户话题相关"]`

示例：
- 标题：你记录了一次职业进展
- 页面摘要：想法记录中提到近期获得新的职业进展，并与收入变化有关。
- AI 摘要：用户近期有职业正向变化。后续职业、财务、目标规划相关对话可参考，但不要主动暴露敏感细节，除非用户话题相关。

### 4.5 轻量统计收藏 `statMilestone`

> 有纪念意义但不应强影响 HoloAI 判断的累计节点。

- **晋升**：轻量收藏，不进入主要长期记忆列表
- **过期**：长期或归档
- **默认 useScopes**：`[.displayOnly, .retrospective]`
- **默认 prohibitedInferences**：`["不用于推断用户当前效率或性格"]`

示例：
- 标题：已完成 50 个任务
- 页面摘要：任务历史中累计完成数量达到新的记录节点。
- AI 摘要：这是轻量成就节点，仅在用户回顾历史或成就时召回，不用于推断用户当前效率或性格。

### 4.6 与情景记忆的边界

| 体系 | 模型 | UI 位置 | 过期 | 用途 |
|------|------|---------|------|------|
| 情景记忆 | `HoloEpisodicMemory` | "最近记住" | 90 天 | 近期观察、可过期上下文 |
| `statMilestone` | `HoloLongTermMemory` | "长期记忆"子分组 | 长期 | 成就展示、年度回顾 |

两者不合并，UI 保持三个主 Section（最近记住 / 待确认 / 长期记忆）。

**Prompt 输出结构**（`memoryCandidate` 子对象，仅 habit/finance/task/milestone 卡片）：

**Prompt 生成约束**（追加到 `memory_insight_generation` prompt）：

1. `title` 禁止系统词（闭环/终端/清零/偏高/模式/趋势/画像/异常）
2. `displaySummary` 只描述事实，不含建议/鼓励/教练表达
3. `aiUseSummary` 必须含适用场景和误用边界
4. 普通统计只能进 `statMilestone`
5. 混合语义必须拆分，一个候选只有一个主语义
6. `overview`/`anomaly`/`thought`/`cross_domain` 不输出 `memoryCandidate`

**MemoryCandidateSemanticMapper 逻辑**：

```swift
struct MemoryCandidateSemanticMapper {
    /// 校验 LLM 输出并转换为 HoloLongTermMemory 候选
    static func validateAndMap(
        candidate: MemoryCandidatePayload,
        card: MemoryInsightCard,
        evidence: [HoloLongTermMemoryEvidence]
    ) -> HoloLongTermMemory?

    /// semanticType → useScopes 默认映射
    static func defaultUseScopes(for type: HoloMemorySemanticType) -> [HoloMemoryUseScope]

    /// semanticType + sensitivity → prohibitedInferences 默认映射
    static func defaultProhibitedInferences(
        for type: HoloMemorySemanticType,
        sensitivity: HoloMemorySensitivity
    ) -> [String]

    /// semanticType → 旧 type 兼容映射
    static func legacyTypeMapping(for semanticType: HoloMemorySemanticType) -> HoloLongTermMemoryType

    /// 合并 confidence：LLM 优先，缺失时按 evidence 数量 fallback
    static func resolveConfidence(
        llmValue: HoloMemoryConfidence?,
        evidenceCount: Int
    ) -> HoloMemoryConfidence

    /// 合并 sensitivity：LLM 值为基准，本地关键词只允许升级
    static func resolveSensitivity(
        llmValue: HoloMemorySensitivity?,
        localKeywords: HoloMemorySensitivity
    ) -> HoloMemorySensitivity
}
```

**Mapper 降级规则**：

| 场景 | 行为 |
|------|------|
| `semanticType` 不在枚举内 | 丢弃候选，记日志 |
| `displaySummary` 为空 | 用 `card.body` fallback（并记日志标记为旧格式摘要） |
| `aiUseSummary` 为空 | 用 `displaySummary` fallback |
| 整个 `memoryCandidate` 缺失 | 回退到旧逻辑（用 card.title/body，type = .recurringPattern） |

**过期清理**：

```swift
static func cleanupExpired(now: Date = Date()) -> Int {
    var memories = load()
    let before = memories.count
    memories = memories.map { mem in
        guard let expires = mem.expiresAt, expires < now else { return mem }
        var updated = mem
        updated.confirmationState = .archived  // 归档，不硬删
        return updated
    }
    try? save(memories)
    return memories.count - before  // 归档数量
}
```

### Phase 3: 晋升策略

**目标**：按 `semanticType` 重写晋升逻辑。

**修改文件**：

| 文件 | 修改内容 |
|------|---------|
| `Services/AI/HoloMemoryPromotionPolicy.swift` | 按 `semanticType` 重写 `evaluate(candidate:)` |
| `Services/AI/HoloLongTermMemoryCandidateObserver.swift` | 对 `driftSignal` 候选设置默认 `expiresAt` |

**晋升矩阵**：

| semanticType | 条件 | 决策 | 附加动作 |
|-------------|------|------|---------|
| `phaseShift` | 任何 | `.requireConfirmation` | — |
| `stablePattern` | evidence >= 3 且 sensitivity == normal | `.silentlyAccept` | — |
| `stablePattern` | 其他 | `.requireConfirmation` | — |
| `driftSignal` | 任何 | `.requireConfirmation` | 设 `expiresAt = now + 21天` |
| `lifeEvent` | 任何 | `.requireConfirmation` | — |
| `statMilestone` | 任何 | `.observe` | `useScopes = [.displayOnly, .retrospective]` |
| `nil`（旧格式） | sensitivity == sensitive/highImpact | `.requireConfirmation` | — |
| `nil`（旧格式） | 其他 | `.requireConfirmation` | — |

> 首版 `.observe` 和 `.requireConfirmation` 行为等价（都 upsertCandidate 保持 .candidate 状态），后续可增加 `.observe` 的自动过期或 UI 区分。

**验收标准**：

- [ ] `stablePattern + evidence>=3 + normal` 可静默写入
- [ ] `driftSignal` 自动设置 `expiresAt`
- [ ] `statMilestone` 的 `useScopes` 为 `[.displayOnly, .retrospective]`
- [ ] `lifeEvent + highImpact` 必须确认
- [ ] 旧格式记忆（`semanticType == nil`）走 fallback 分支

### Phase 4: HoloAI 召回

**目标**：记忆注入使用 `aiUseSummary`，按 `useScopes` 筛选。

**修改文件**：

| 文件 | 修改内容 |
|------|---------|
| `Services/AI/HoloMemorySummaryProvider.swift` | 重写 `selectRelevantSummary()` 使用 `useScopes` 筛选 |
| `Models/AI/AIModels.swift` | `HoloMemoryPromptSummary` 扩展：携带 `aiUseSummary` + `prohibitedInferences` |
| `Services/AI/AIUserContextMessageBuilder.swift` | 注入格式改为新格式（含 useScopes 标签 + 误用边界） |
| `Services/AI/UserContextBuilder.swift` | 如需细场景筛选，接收 purpose 参数 |

**purpose → useScopes 映射**：

| 调用场景 | purpose | 允许的 useScopes |
|---------|---------|-----------------|
| 普通聊天 | `.chat` | `[.coreContext, .recentInsight]` |
| 意图识别 | `.intentRecognition` | 不注入记忆 |
| 今日状态 | `.todayState` | `[.coreContext, .recentInsight]` |
| 目标规划 | `.goalPlanning` | `[.coreContext, .goalPlanning]` |
| 长期模式 | `.longTermPatterns` | 全部（排除 `.displayOnly`） |

**注入格式**：

```text
--- 记忆摘要 ---
- [recentInsight] 身体记录最近不连续：用户近期身体状态追踪不稳定。适合健康或习惯洞察参考。
  避免推断：不要判断用户已放弃目标；不要归因为懒惰或失败。
- [coreContext] 任务收尾节奏稳定：用户近期任务执行表现稳定。后续任务规划可参考。
规则：以上记忆只能辅助理解用户，不得覆盖用户当前明确指令。
```

**筛选算法**：

1. 只取 `.confirmed` 和 `.silentlyAccepted`
2. 排除 `expiresAt < now`
3. 排除 `useScopes` 包含 `.displayOnly` 且 purpose 非回顾场景
4. 根据 purpose 匹配 `useScopes`（候选 useScopes 与允许集合有交集即命中）
5. 排序：高置信 > 近期更新 > 非敏感 > 证据多
6. 取前 5 条；不足 5 条时不补无关记忆
7. 总长度限制 500-700 中文字，超出按优先级截断

**降级**：记忆无 `aiUseSummary` 时 fallback 到 `displaySummary`；无 `displaySummary` 时 fallback 到 `summary`。

**验收标准**：

- [ ] `.chat` 场景不注入 `displayOnly` 记忆
- [ ] 注入格式含 `[useScopes]` 标签和"避免推断"
- [ ] 总长度在限制内
- [ ] `semanticType == nil` 的旧记忆仍能注入（低优先级）
- [ ] 无 `aiUseSummary` 时正确 fallback
