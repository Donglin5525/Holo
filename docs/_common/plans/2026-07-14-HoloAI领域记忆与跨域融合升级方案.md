# HoloAI 领域记忆与跨域融合升级方案

> 日期：2026-07-14
>
> 状态：产品设计已确认，待实施计划
>
> 范围：领域记忆、跨域记忆、统一召回、静默更新、用户控制、开发者验证
> 前置基础：现有严格语义长期记忆、情景记忆、证据引用、记忆回执和 Agent/洞察召回链路

## 1. 产品结论

Holo 的下一阶段记忆能力，不是继续增加一批通用 AI 摘要，而是建立一套“各模块分别理解、跨模块共同理解、按问题精准使用”的记忆体系。

最终用户应感受到：

> Holo 不仅记得我记录过什么，还能理解我在财务、健康、习惯、目标和观点上的阶段变化，并在合适的问题里把这些理解连接起来。

系统采用“稳定结论持久化、即时状态动态组合”的混合模式：

- 财务、观点、健康、习惯、任务、目标等模块分别形成领域记忆。
- 两个及以上领域共同支持的稳定结论，可以形成阶段性跨域记忆。
- “我最近状态如何”这类即时判断默认动态组合，不自动永久保存。
- 同一跨域结论连续多个周期成立，或经用户确认后，才允许持久化。
- 明确数字问题仍查询原始明细，记忆不能替代事实计算。

## 2. 已确认的产品决策

1. 采用“独立模块萃取器 + 统一记忆底座 + 跨域融合层”。
2. 模块之间逻辑隔离，但不为每个模块复制一套 Store、状态机和用户反馈系统。
3. 用户看到结论、来源和控制操作；内部置信度、新鲜度、相关度和排序分数不向普通用户展示。
4. 跨模块可以发现关联和冲突，但不得在证据不足时写成确定因果。
5. Profile 是用户明确填写或确认的高优先级事实，不由后台 AI 静默改写。
6. 用户当前明确输入优先于 Profile 和任何历史记忆。
7. “自动形成记忆”和“记忆辅助回答”分开控制。
8. 静默萃取失败不得影响记账、任务、习惯等主流程。
9. 用户选择“忘记”后，系统不得换个说法重新生成同一结论。
10. Debug 必须提供“AI 记忆实验室”，显性展示从数据变化到回答使用的真实链路。

## 3. 当前实现基础与真实缺口

### 3.1 已有基础

当前工程已经具备：

- 严格语义格式的 `HoloLongTermMemory`。
- `semanticType`、`useScopes`、`aiUseSummary` 和 `prohibitedInferences`。
- 情景记忆 `HoloEpisodicMemory` 及过期、拒绝和抑制逻辑。
- 稳定记忆身份和跨周期证据合并。
- `HoloMemorySummaryProvider` 与相关性召回。
- 普通聊天、洞察和 Agent 的部分记忆接入。
- 记忆写入回执和使用回执。
- HoloAI 数据处理授权总闸。

### 3.2 主要缺口

1. `HoloObservationPackage` 和 `HoloMemoryObserverService` 目前实际只覆盖习惯和目标。
2. 财务、观点、健康、任务和对话没有稳定的领域记忆萃取器。
3. 现有记忆以长期/情景分类，没有形成统一的“领域/跨域”语义底座。
4. 当前召回把置信度、证据数和更新时间混合排序，没有独立表达“成立程度”和“当前相关性”。
5. 没有跨域融合服务，也没有共同目标、主题和时间阶段索引。
6. “情景记忆观察”只在 Debug AI 设置中可见，且当前只有手动触发入口。
7. Release 设置只有“长期记忆”和“记忆辅助对话”，尚不能完整管理静默模块萃取。
8. 长期记忆和情景记忆仍是本机 JSON Store，没有跨设备版本与删除冲突策略。
9. 当前 30 天 suppression 不足以实现用户明确要求的永久“忘记”。

## 4. 方案比较

### 方案 A：每个模块独立记忆系统

分别建立 Finance/Thought/Health 等 Store 和状态机。

- 优点：模块物理隔离。
- 缺点：重复实现存储、过期、置信度、用户反馈和召回；跨域融合还要重新统一数据。
- 结论：不采用。

### 方案 B：统一记忆底座，按领域逻辑隔离

每个模块拥有独立萃取器和时间窗口，但输出统一记忆契约；跨域融合只读取通过校验的领域记忆。

- 优点：领域语义清晰，用户控制和召回统一，能够复用现有记忆基础。
- 风险：统一 Schema 必须同时容纳近期状态、稳定模式和人生事实。
- 结论：采用。

### 方案 C：不保存模块记忆，查询时动态总结

- 优点：初期改动少。
- 缺点：每次重复读取明细、成本高、结果漂移，无法形成持续理解。
- 结论：不采用。

第一版不引入向量数据库。结构化领域、主题键、时间范围、记忆类型和关键词足以支撑初期召回；未来只有在真实规模证明结构化召回不足时，才增加 embedding 作为辅助排序。

## 5. 总体分层

```text
L0 原始数据
财务明细、观点原文、健康读数、习惯记录、任务、目标、对话
        ↓
L1 模块信号
确定性统计生成的高密度事实，不直接作为记忆
        ↓
L2 领域记忆
财务记忆、观点记忆、健康记忆、习惯记忆等
        ↓
L3 跨域记忆
由两个及以上领域共同支持的阶段状态、关联和冲突
        ↓
L4 HoloAI 运行时上下文
按当前问题组合 Profile、领域记忆、跨域记忆和必要明细
```

各层职责：

- 原始数据负责证明。
- 模块信号负责压缩明细。
- 领域记忆负责描述单一领域内的稳定理解。
- 跨域记忆负责连接多个领域，但不创造新的事实。
- Profile 负责提供用户明确声明的身份、偏好、目标和边界。
- HoloAI 负责根据当前问题选择信息，不把所有记忆全部注入。

## 6. 统一记忆契约

目标模型建议收敛为统一的 `HoloMemoryRecord`，由 `scope` 和 `persistenceClass` 表达领域、跨域和时间性质，而不是继续扩展多套 JSON 模型。

核心字段：

```swift
struct HoloMemoryRecord {
    var id: String
    var scope: HoloMemoryScope
    var primaryDomain: HoloMemoryDomain?
    var sourceDomains: [HoloMemoryDomain]
    var subjectKey: String
    var claimKind: HoloMemoryClaimKind
    var persistenceClass: HoloMemoryPersistenceClass

    var displaySummary: String
    var aiUseSummary: String
    var prohibitedInferences: [String]

    var evidenceRefs: [HoloMemoryEvidenceRef]
    var upstreamMemoryIDs: [String]
    var counterEvidenceRefs: [HoloMemoryEvidenceRef]

    var validFrom: Date?
    var validTo: Date?
    var lastSupportedAt: Date?
    var expiresAt: Date?

    var confidenceScore: Double
    var freshnessScore: Double
    var state: HoloMemoryState
    var sensitivity: HoloMemorySensitivity
    var userDecision: HoloMemoryUserDecision

    var createdAt: Date
    var updatedAt: Date
    var schemaVersion: Int
}
```

### 6.1 Scope

- `domain`：只描述一个模块。
- `crossDomain`：引用两个及以上领域记忆。

### 6.2 Claim Kind

- `observedFact`：数据直接支持的事实摘要。
- `recurringPattern`：多个周期重复出现的模式。
- `phaseShift`：相较此前阶段发生变化。
- `association`：多个变化同时出现，但不声明因果。
- `tension`：当前行为与明确目标或偏好存在冲突。
- `hypothesis`：证据有限、需要继续验证的推测。
- `explicitPreference`：用户明确表达或确认的偏好。
- `lifeEvent`：用户明确记录的重要人生事实。

### 6.3 Persistence Class

- `currentState`：短期状态，快速衰减。
- `phase`：阶段性记忆，可持续数周至数月。
- `durable`：跨周期稳定模式。
- `permanentFact`：明确人生事件或用户确认事实。

### 6.4 稳定身份

记忆 ID 继续使用稳定语义身份：

```text
scope + primaryDomain/sourceDomains + claimKind + subjectKey
```

日期、报告序号、卡片 ID 和模型生成文案不得进入稳定身份。

## 7. 领域记忆生产链

```text
模块数据变化
→ 标记对应领域 dirty
→ 确定性统计生成候选信号
→ AI 解释候选含义
→ 本地验证证据、时间和表达边界
→ 按稳定身份合并或生成领域记忆
→ 更新支持证据、反例、时间状态和用户回执
```

### 7.1 规则与 AI 的边界

确定性代码负责：

- 金额、数量、频率、完成率、时间窗口和对照周期。
- 原始记录引用。
- 数据覆盖度和缺失状态。
- 置信度基础项、时间衰减、状态机和用户操作优先级。

AI 负责：

- 将多个信号解释成克制的领域结论。
- 判断是模式、阶段变化、关联、冲突还是待验证推测。
- 生成用户可读摘要和 AI 使用摘要。

AI 不负责：

- 自由计算金额和次数。
- 发明不存在的证据。
- 把一次事件固化为人格或长期事实。
- 绕过本地 Validator 直接写入 Store。

## 8. 各领域萃取职责

| 领域 | 重点萃取 | 明确禁止 |
|---|---|---|
| 财务 | 消费结构变化、重复商户/品类、固定支出、预算偏离、消费与目标的关系 | 不根据消费推断收入、阶层或人格；单次消费不固化为习惯 |
| 观点 | 反复主题、明确立场、立场变化、未解决问题、长期价值取向 | 不把 AI 总结当用户原话；一次随想不升级成人格标签 |
| 健康 | 睡眠、活动、站立、运动和恢复趋势、阶段变化 | 不做医疗诊断；不把缺失数据当异常 |
| 习惯 | 稳定节奏、中断场景、恢复方式、成功或失败条件 | 不使用“自律差”等道德评价 |
| 任务 | 完成节奏、积压变化、常见延期类型、工作负荷、执行偏好 | 不因少量逾期推断能力或心理问题 |
| 目标 | 当前重点、推进速度、目标调整、目标协同与冲突 | 不把系统建议自动写成用户目标 |
| 对话 | 用户明确表达的偏好、纠正、承诺和重要上下文 | 普通聊天不自动长期保存；AI 自己的回复不能成为证据 |
| Profile | 用户主动填写或确认的身份、偏好、目标和边界 | 后台 AI 不得静默改写 |

MemoryInsight 只用于发现值得重新检查的主题，不能作为新的事实证据。领域记忆必须最终追溯到原始模块记录。

## 9. 时间窗口与内部评分

### 9.1 领域时间窗口

- 健康、习惯：7 天、28 天、90 天。
- 财务：本周、本月、近 3 个月，并尊重账单周期等自然周期。
- 观点：30 天、90 天、半年，重点观察主题和立场演变。
- 任务：7 天、14 天、30 天，重点观察负荷和执行节奏。
- 目标：跟随目标自身生命周期。
- 对话：只处理明确表达，并设置较短候选观察期。

### 9.2 置信度与新鲜度分离

置信度回答“这条结论有多可靠”，新鲜度回答“它是否仍代表当前状态”。

```text
成立置信度
= 来源可靠性
× 证据覆盖度
× 跨周期一致性
× 用户确认加权
× 反例惩罚

当前召回分
= 语义相关度
× 新鲜度
× 成立置信度
× 当前场景适用度
```

衰减速度：

- 近期状态：7～30 天快速衰减。
- 行为模式：90～180 天缓慢衰减。
- 阶段目标：跟随目标状态，到期或完成后归档。
- 用户明确偏好：不自动衰减，直到用户修改。
- 人生事件：事实不衰减，但日常召回相关性可以降低。

### 9.3 用户可见表达

内部不得向普通用户展示百分比、评分条或排序分数。用户只看到：

- 最近形成。
- 持续了一段时间。
- 近期正在变化。
- 过去曾经如此。
- 目前观察到，仍需更多记录确认。

## 10. 反例、冲突与版本

新证据不直接覆盖旧记忆：

- 支持：增加独立周期支持并续期。
- 冲突：加入反例，降低当前有效性。
- 明确反转：旧记忆标记为 `superseded`，生成新的阶段记忆。

Holo 应能表达“以前是这样，但最近发生了变化”。历史版本只用于解释变化和开发者审计，不默认进入当前回答。

优先级按问题类型分别处理，不能用一条全局顺序覆盖所有场景。

身份、偏好和用户意图类：

```text
用户当前明确输入
> 用户纠正、拒绝和忘记记录
> 用户明确填写的 Profile
> 已确认记忆
> 自动推断的未确认记忆
```

当前状态和可计算事实类：

```text
用户当前明确输入
> 当前原始数据和确定性计算
> 用户明确目标与边界
> 已确认记忆
> 自动推断的未确认记忆
```

因此，Profile 可以约束“用户想实现什么”，但不能覆盖本期真实金额、睡眠、完成率和任务状态。

## 11. 跨域融合

跨域候选必须存在共同锚点：

- 同一目标。
- 同一主题。
- 同一时间阶段。
- 同一生活事件。
- 用户当前明确问题。

示例：

```text
目标记忆：正在持续减脂
健康记忆：最近四周入睡较晚
财务记忆：高热量快餐消费增加
习惯记忆：运动完成率下降
        ↓
跨域候选：
“减脂目标仍在持续，但睡眠、饮食和运动执行近期出现了同时偏离。”
```

### 11.1 四道门

1. 候选发现：两个及以上领域记忆具有共同锚点和时间重叠。
2. AI 综合解释：输出阶段状态、共同变化、协同、冲突或假设。
3. 本地验证：验证上游 ID、时间重叠、来源数量和因果表达。
4. 持久化判断：当前问题临时使用，或连续多个周期成立后保存。

### 11.2 防污染规则

- 跨域记忆只能引用领域记忆。
- 领域记忆只能引用原始模块证据。
- 跨域记忆不能作为新的事实来源递归生成更多记忆。
- 上游领域记忆过期、删除或被替代时，下游跨域记忆必须重新评估。
- 跨域记忆继承上游最高敏感等级和最严格同步策略。

### 11.3 因果边界

系统区分：

- 已记录事实。
- 观察到的关联。
- 待验证推测。
- 因果结论。

默认只允许前三层。没有足够证据或用户确认时，不得生成或保存确定因果。

## 12. HoloAI 查询路由

### 12.1 明确数字和事实问题

示例：“最近 14 天吃了多少麦当劳？”

- 直接查询原始数据并确定性计算。
- 记忆不能代替数字证据。
- 除非用户继续追问原因或趋势，否则不注入长期记忆。

### 12.2 单领域理解问题

示例：“我最近消费习惯怎么样？”

- 先读取相关领域记忆。
- 记忆过旧、覆盖不足或需要解释具体原因时，定向补查该模块明细。

### 12.3 开放式状态问题

示例：“我最近状态如何？”

```text
当前明确输入
→ Profile 中的目标与边界
→ 相关跨域记忆
→ 各领域近期记忆
→ 仅对缺口定向补查明细
```

系统按问题选择相关领域，不把全部记忆塞入 Prompt。

### 12.4 规划和建议问题

记忆用于个性化，实时数据用于约束。实际日期、金额、任务状态和日程必须读取当前数据。

### 12.5 用户回执

普通用户只看到简洁回执：

> 这次判断参考了你近期的健康、习惯和财务记录。

展开后显示：

- 使用了哪些记忆结论。
- 来源模块。
- 参考时间范围。
- 修改、忘记和不相关操作。

如果关键模块没有数据，Holo 必须说明判断范围有限，不能把“没有读取到”说成“没有问题”。

## 13. 静默更新与开关模型

### 13.1 用户侧只保留两个记忆开关

#### 自动形成记忆

控制：

- 领域记忆萃取。
- 跨域融合。
- 新记忆写入和自动更新。

关闭后停止形成和更新记忆，但不删除已有记忆。模块数据变化可以留下轻量 dirty 标记，供用户以后重新开启时补算；关闭期间不得生成新的派生内容，也不得调用记忆萃取或跨域融合 AI。

#### 记忆辅助回答

控制已有记忆是否进入：

- HoloAI 对话。
- 洞察。
- 目标规划。
- Agent 深度分析。

关闭后记忆仍保留并可管理，但不参与回答。

### 13.2 上层总闸

“HoloAI 数据授权”继续作为所有外部 AI 调用的总闸：

```text
数据授权关闭
→ 外部 AI 调用停止
→ 静默领域萃取和跨域融合停止
→ 原始本地记录不受影响
```

### 13.3 内部 Feature Flags

现有 `longTermMemoryEnabled` 可升级为记忆写入总闸。`episodicMemoryObservationEnabled`、各模块开关和跨域融合开关作为内部灰度能力，不向普通用户暴露。

### 13.4 更新节奏

- 本地统计信号：数据变化后低成本更新。
- 领域 AI 萃取：每天最多一次，只处理发生变化的模块。
- 跨域融合：每周一次，或开放式问题触发按需刷新。
- 重大目标变化和用户纠正：立即使相关结论失效并重新评估。
- 后台机会未执行：下次进入 App 后轻量恢复。

## 14. 用户控制与记忆生命周期

### 14.1 用户操作

- `不相关`：降低当前召回，未来有新证据可重新判断。
- `这不准确`：保存用户纠正，后续萃取必须把它作为反例。
- `忘记这件事`：删除记忆并建立持久 suppression，禁止近义重生。
- `清除全部 AI 记忆`：清除领域记忆、跨域记忆、反馈和 suppression，不删除原始业务数据。

### 14.2 原始数据联动

- 新增记录：对应领域标记 dirty。
- 修改记录：旧证据失效并重新计算。
- 删除记录：移除证据；证据不足时记忆减弱或归档。
- 模块同步失败：不更新结论，也不能把缺失解释成行为变化。

每条证据应携带来源稳定 ID 和版本指纹，以便判断原始记录是否变化。

## 15. 存储与跨设备策略

统一底座通过 `HoloMemoryRepository` 暴露单一接口，底层可按敏感等级选择存储策略。

目标策略：

- 普通领域记忆、跨域记忆和用户反馈采用结构化、可版本化存储。
- 普通记忆可进入用户私有 iCloud，实现跨设备一致体验。
- 使用稳定 ID、版本号和删除墓碑处理多设备并发。
- 用户删除、纠正和关闭自动记忆的操作优先于后台生成更新。
- 健康及其他敏感派生记忆在合规边界确认前默认只保存在本机。
- 含敏感领域的跨域记忆继承最严格存储和同步策略。

现有长期/情景 JSON Store 不继续扩展为最终形态。迁移必须支持：

- 原子迁移。
- 失败回滚。
- Schema 版本。
- 旧记忆稳定 ID 保留或映射。
- 用户确认、拒绝和忘记状态不丢失。

## 16. 失败与降级

- 网络或 AI 失败：保留 dirty 标记，下次重试。
- 单模块失败：其他模块继续更新，本轮覆盖度标记为部分。
- 输出格式错误：拒绝候选，不覆盖旧记忆。
- 引用不存在：拒绝候选。
- 时间窗口不重叠：拒绝跨域候选。
- 存储失败：保留上一个有效版本。
- 长时间未更新：只作为历史背景，不描述为最近状态。
- 数据授权或自动记忆关闭：不得旁路调用 AI。

这些失败默认不打扰普通用户。只有当用户的问题依赖缺失模块时，才用自然语言说明本轮参考范围有限。

## 17. Debug：AI 记忆实验室

在当前 Debug “AI 设置”中新增独立入口“AI 记忆实验室”，Release 不编译、不展示。

### 17.1 总览面板

- 数据授权、自动形成记忆、记忆辅助回答状态。
- 各领域最近成功更新时间。
- dirty 模块和计划运行时间。
- 领域、跨域、待确认、归档记忆数量。
- 最近失败原因与重试状态。
- Schema 和 Prompt 版本。

### 17.2 模块验证

逐步执行并查看真实链路：

```text
查看数据覆盖
→ 生成本地统计信号
→ 预览观察包
→ 执行领域萃取
→ 查看结构化返回
→ 查看 Validator 结果
→ 查看最终写入或拒绝原因
```

### 17.3 跨域验证

- 选择领域记忆手动执行融合。
- 查看共同锚点、选入和排除原因。
- 查看时间重叠、claim kind 和持久化判断。
- 查看完整上游领域记忆引用。

### 17.4 HoloAI 查询模拟器

输入问题后展示：

- 问题类型。
- 选择的领域。
- memory/detail 数据路径。
- 定向补查项。
- 过期、不相关和敏感内容排除原因。
- 内部置信度、新鲜度、相关度和排序分数。
- 最终上下文预览。

用于验证数字问题没有错误召回、开关能够阻断调用、敏感记忆不被无关召回、覆盖不足正确降级。

### 17.5 单条记忆检查器

- 当前结论和历史版本。
- 支持证据和反例。
- 原始记录引用有效性。
- 内部评分组成。
- 最近使用记录。
- 上游领域记忆和下游跨域记忆。
- 用户纠正、拒绝和忘记记录。

### 17.6 本地 Trace

保留本机环形追踪：

```text
原始数据变化
→ 模块标脏
→ 信号生成
→ AI 调用
→ 校验
→ 记忆写入
→ 查询召回
→ 回答使用
```

Trace 不进入 CloudKit、不上传后端、不持久保存完整敏感 Prompt。导出时只允许生成脱敏 JSON。

开发者页面必须调用真实生产 Service 和 Validator，不能维护另一套只用于展示的模拟逻辑。

## 18. 分阶段实施

### Phase 1：统一记忆底座

- 统一领域/跨域模型和 Repository。
- 置信度、新鲜度、召回相关度分离。
- 证据、反例、版本、纠正、忘记和删除墓碑。
- 用户开关收口。
- dirty 调度和失败恢复。
- 现有长期/情景记忆迁移。
- AI 记忆实验室基础总览和单条检查器。

### Phase 2：结构化领域

- 财务。
- 习惯。
- 健康。
- 目标。
- 模块实验室分步执行与 Trace。

### Phase 3：语义型领域

- 观点。
- 任务。
- 对话明确表达。
- 防止临时表达、普通聊天和一次事件错误固化。

### Phase 4：跨域融合与 HoloAI 路由

- 共同目标、主题和时间阶段索引。
- 动态状态回答。
- 跨域阶段记忆持久化。
- Chat、洞察、规划和 Agent 接入。
- 来源回执和查询模拟器完整链路。

## 19. 上线策略

```text
内部影子运行
→ 记忆中心可见
→ HoloAI 仅使用用户确认记忆
→ 开放低风险自动记忆
→ 开放跨域阶段记忆
```

影子模式正常生成候选，但不进入正式回答、不主动展示给用户。重点检查准确性、重复、越界、证据可追溯性和数据缺失语义。

## 20. 测试与验收

### 20.1 纯逻辑测试

使用 standalone Swift 测试覆盖：

- 稳定身份。
- 置信度和新鲜度计算。
- 过期、替代和反例状态机。
- 用户操作优先级。
- suppression 和近义重生拦截。
- Query Router 分类。
- 跨域时间重叠和因果表达 Validator。
- 敏感等级继承。

### 20.2 Fixture 测试

每个领域准备固定输入与预期结构，覆盖：

- 数据充足。
- 数据不足。
- 突然变化。
- 反例出现。
- 原始数据删除。
- 模型返回非法 evidence ID。
- 模型返回诊断、人格标签或确定因果。

### 20.3 集成验收场景

1. “最近 14 天吃了多少麦当劳”只查明细，不用记忆代替计算。
2. “我最近状态如何”组合多个有效领域，并说明覆盖范围。
3. 原始记录修改或删除后，领域和跨域记忆重新评估。
4. 用户选择“忘记”后，同一结论不以近义表达重生。
5. 关闭自动形成记忆后，不再发生后台 AI 萃取调用。
6. 关闭记忆辅助回答后，已有记忆不进入回答或规划。
7. 单模块失败不影响其他模块，不把缺失数据描述为正常。
8. 跨域结论每部分都能追溯到领域记忆和原始证据。
9. 关联不被表达成确定因果，健康内容不形成诊断。
10. Holo 能表达“以前如此，但最近正在变化”。
11. Debug 查询模拟器展示的路由、召回和最终上下文与真实回答链路一致。
12. Release 构建不包含 AI 记忆实验室入口和敏感 Trace 展示。

### 20.4 验证要求

- 纯逻辑策略优先使用 standalone `swiftc` 测试。
- 如果使用 Xcode 测试，必须确认测试实际执行，`Executed 0 tests` 不算通过。
- 后端 Prompt/registry 变更必须运行后端测试，并在 ECS 部署后验证生产 Prompt 版本和输出契约。

## 21. 内部质量指标

- 用户纠正率、忘记率和不相关率。
- 领域记忆被实际使用的比例。
- 开放式问题的跨域覆盖度。
- 重复记忆率。
- 过期记忆误召回率。
- 静默更新耗时、AI 成本和失败率。
- 明细问题被错误路由到记忆的比例。

这些指标只用于内部质量改进，不向普通用户展示。

## 22. 非目标

本轮不做：

- 不让向量相似度直接决定事实。
- 不把所有原始数据复制进记忆库。
- 不把 MemoryInsight 或 AI 历史回答当事实证据。
- 不让跨域记忆递归强化自身。
- 不自动生成医疗、心理、人格或阶层判断。
- 不要求普通用户理解内部评分和状态机。
- 不让静默记忆任务阻塞主流程。

## 23. 后端与发布边界

领域萃取和跨域融合需要新增或升级后端 Prompt purpose、registry、默认 Prompt 和 Validator 契约。实施时必须：

1. 同步 iOS 后备模板与后端生效 Prompt。
2. 升级 Prompt 版本。
3. 完成后端契约测试。
4. 部署 HoloBackend 到 ECS。
5. 验证生产 Prompt 元数据和真实返回结构。

只完成本地 iOS 或本地后端代码不代表功能已经生产生效。
