## 一、认识 HoloAI

HoloAI 是 Holo APP 的智能助手系统。它能够理解你的自然语言，帮你完成记账、建任务、习惯打卡、记录心情等日常操作，还能自动分析你的数据、生成周期性洞察报告，让你对自己的生活状态一目了然。

**HoloAI 能做什么：**

| 类别 | 功能 | 触发方式 |
|------|------|---------|
| 日常操作 | 记账、建任务、打卡、记心情、记笔记 | 你主动对 AI 说 |
| 数据查询 | 查看任务列表、习惯状态、消费分析 | 你主动对 AI 说 |
| 周期洞察 | 生成日/周/月洞察报告 | 你点击生成，或系统自动生成 |
| 智能学习 | 记住你的分类习惯，越用越准 | 自动运行 |
| 异常检测 | 消费飙升、习惯断裂、任务过载预警 | 自动运行 |
| 跨域关联 | 发现习惯与消费、情绪与支出的隐藏关联 | 自动运行 |

## 二、快速开始：配置 AI 服务

HoloAI 需要接入大语言模型（LLM）才能工作。首次使用前，请完成以下配置。

### 2.1 进入配置页面

有 **三种方式** 进入 AI 配置页面：

| 入口 | 操作路径 |
|------|---------|
| 聊天页 | 点击聊天页右上角齿轮图标 |
| 设置页 | 首页右上角人像 → 设置 → 通用 → 「AI 助手」 |
| 记忆长廊 | 记忆长廊 → 回放 Tab → 若未配置，点击「去配置 AI」 |

### 2.2 选择 AI 服务商

HoloAI 支持以下 **5 种 AI 服务商**（均为 OpenAI 兼容协议）：

| 服务商 | 默认模型 | 说明 |
|--------|---------|------|
| DeepSeek | deepseek-chat | 推荐，性价比高 |
| 通义千问 | qwen-turbo | 阿里云大模型 |
| Moonshot | moonshot-v1-8k | 月之暗面 |
| 智谱 | glm-4-flash | 智谱 AI |
| 自定义 | 自行填写 | 支持任何 OpenAI 兼容 API |

### 2.3 填写配置

| 配置项 | 说明 |
|--------|------|
| API Key | 你的服务商 API 密钥，安全存储于设备 Keychain，不会上传 |
| Base URL | 服务商接口地址（自定义服务商需手动填写） |
| 模型名称 | 使用的具体模型（自定义服务商需手动填写） |
| 温度 | 控制回复的随机性，范围 0~1，默认 0.7 |
| 最大 Token | 单次回复的最大长度，范围 256~8192，默认 2048 |

### 2.4 测试连接

配置完成后，点击「测试连接」按钮验证 API 是否可用。显示"连接成功"即表示配置正确。

## 三、AI 对话助手 — 自然语言操作

这是 HoloAI 最核心的功能。你只需用日常语言说出需求，AI 会自动理解并执行。

### 3.1 如何进入对话

- **首页底部导航栏中央橙色按钮**（sparkles 图标）
- **记忆长廊 → 回放 Tab → "继续问 AI"**
- **Holo One 快捷动作**（若配置为"AI 对话"）

### 3.2 快捷操作栏

进入对话后，输入框上方有一排快捷按钮，点击即可自动发送对应指令：

| 按钮 | 发送内容 | 执行操作 |
|------|---------|---------|
| 记一笔消费 | "帮我记一笔消费" | AI 引导你输入消费信息 |
| 创建任务 | "帮我创建一个任务" | AI 引导你输入任务信息 |
| 记录心情 | "记录我现在的心情" | AI 引导你描述心情 |
| 习惯打卡 | "帮我打卡" | 直接打卡（仅一个习惯时）或让你选择 |
| 本周总结 | "生成本周总结" | 生成本周 AI 洞察回放 |
| 记笔记 | "帮我记一条笔记" | AI 引导你输入笔记内容 |
| 今日任务 | "今天有什么待办" | 列出今日待办任务 |
| 习惯状态 | "今天习惯完成了吗" | 展示今日各习惯打卡状态 |

#### 意图 1：记录支出

**意义**：用自然语言快速记一笔消费，无需手动选择分类和填写表单。

**如何使用**：直接说出消费内容，包含金额和用途即可。

| 输入示例 | 说明 |
|---------|------|
| "午饭35" | 记录 35 元餐饮支出 |
| "打车25块" | 记录 25 元交通支出 |
| "买了一件衣服 399" | 记录 399 元购物支出 |
| "超市买了点零食 56" | 记录 56 元购物支出 |

**AI 会做什么**：
1. 识别你的意图是"记一笔支出"
2. 提取金额和分类关键词
3. 自动匹配到对应的分类科目（如"午饭"→"餐饮/午餐"）
4. 记入默认账户
5. 返回确认信息

**如果分类无法识别**：
- AI 会将交易暂归"待确认"分类，并提示你"⚠️ 无法识别分类「XXX」，已暂归「待确认」，点击卡片可修改"
- 你可以点击消息中的交易卡片，手动选择正确的分类
- 系统会记住你的选择，下次自动应用（详见第七章）

**如果缺少金额**：
- AI 会追问："请告诉我具体的金额"

#### 意图 3：创建任务

**意义**：用自然语言快速创建待办任务，支持自动解析日期和提醒。

**如何使用**：

| 输入示例 | 说明 |
|---------|------|
| "提醒我明天买牛奶" | 创建任务"买牛奶"，截止日期为明天 |

5. 返回确认信息

**确认信息示例**：
- 有具体时间："已创建任务：开会（5月6日 15:00，将提前 15 分钟提醒你）"

### 3.4 多指令一次输入

你可以在一句话中包含多个操作，AI 会自动拆分并依次执行。

| 输入示例 | AI 拆分结果 |

**限制**：当前版本不支持把"查询"和"操作"混在一句话里，请拆成两句发送。

### 3.5 澄清机制

当 AI 不确定你的意图时，会主动追问而非盲目执行：

| 场景 | AI 回复 |
|------|--------|
| 意图不明确 | "我没完全理解，可以换个方式再说一次吗？" |
| 低置信度（< 70%） | "我不太确定你想要做什么，可以再描述得清楚一些吗？" |
| 无法识别 | "我没能识别出可执行操作，请拆开重试。" |
| 查询与操作混合 | "当前版本暂不支持把查询和执行操作混在一句话里，请拆成两句发送。" |

## 四、AI 数据分析

HoloAI 可以对你各个模块的数据进行智能分析，用自然语言提问即可获得深度洞察。

### 4.1 支持的分析域

| 分析域 | 关键词示例 | 分析内容 |
|--------|-----------|---------|
| 财务 | 消费、支出、花了多少、预算 | 收支统计、Top 分类、月度趋势、预算执行 |
| 习惯 | 习惯、打卡、完成率、坚持 | 完成率、连续打卡、挣扎习惯、趋势对比 |
| 任务 | 任务、待办、逾期、拖延 | 完成率、逾期统计、优先级分布、每日趋势 |
| 想法/情绪 | 心情、情绪、标签、想法 | 情绪分布、Top 标签、文本情感分析 |
| 跨模块 | 总结、复盘、综合分析、最近过得怎么样 | 四域联动分析、跨域关联发现 |

### 4.2 如何使用

**直接对 AI 提问**，AI 会自动识别你的问题属于哪个分析域。

| 提问示例 | 分析域 | AI 行为 |
|---------|--------|---------|
| "这个月花了多少？" | 财务 | 计算本月总支出、Top 分类、日均消费等 |
| "最近跑步坚持得怎么样？" | 习惯 | 计算完成率、连续天数、趋势对比 |
| "我最近拖延严重吗？" | 任务 | 统计逾期率、完成率、高优先级完成情况 |
| "我最近心情怎么样？" | 想法/情绪 | 统计情绪分布、Top 标签、情感倾向 |
| "和上个月比怎么样？" | 自动对比 | 自动计算对比期，生成对比分析 |
| "总结一下我最近的生活状态" | 跨模块 | 四个域的数据联动综合分析 |

### 4.3 时间范围智能解析

AI 会自动理解你提到的时间范围：

| 你的表述 | AI 解析的时间范围 |
|---------|-----------------|
| "今天" | 今日 |
| "本周/这周" | 本周一到今天 |
| "上周" | 上周一到上周日 |
| "本月/这个月" | 本月1日到今天 |
| "上个月" | 上月1日到上月末日 |
| "最近7天" | 最近7天 |
| "最近30天" | 最近30天 |
| "今年" | 今年1月1日到今天 |
| "去年" | 去年全年 |
| 未提及时间 | 默认最近30天 |

**自动对比**：当你提到"和上期比"或暗示对比时，AI 会自动计算一个等长的对比区间。

### 4.4 分析结果的展示

分析结果以 **流式 Markdown** 格式展示，通常包含：
- 核心指标总结
- 趋势分析
- 对比变化
- 可操作建议

同时在消息中展示对应的分析卡片：
- **Summary 卡片**：域概览 + 核心指标列表
- **Breakdown 卡片**：分类占比饼图 + 标签-值-百分比
- **Trend 卡片**：趋势方向 + 最高/最低值
- **Comparison 卡片**：当前值 vs 上期值 + 变化量（涨跌颜色区分）
- **Highlights 卡片**：绿色亮点 + 橙色警告

### 4.5 无数据时的提示

如果指定时间范围内没有数据：
- AI 回复："在 XXX 期间没有找到可分析的数据。你可以换一个时间范围试试。"

## 五、AI 记忆洞察（回放）

记忆洞察是 HoloAI 的高级功能，能够基于你的周期数据（财务、习惯、任务、想法）自动生成结构化的 AI 洞察报告。

### 5.1 什么是记忆洞察

记忆洞察是一份 AI 生成的周期性生活报告，包含：
- **标题和摘要**：对本周期生活状态的整体评价
- **洞察卡片**（1~8 张）：按不同维度展开分析
- **建议追问**：AI 推荐的后续提问方向

### 5.2 支持的周期

| 周期 | 覆盖范围 | 数据聚合维度 |
|------|---------|-------------|
| **日报** | 当天 | 财务/习惯/任务/想法的当日数据 |
| **周报** | 本周一到今天（不足3天自动回退到上周） | 四维数据 + 上周对比 + 异常检测 + 跨域关联 |
| **月报** | 本月1日到今天（不足7天自动回退到上月） | 四维数据 + 上月对比 + 异常检测 + 跨域关联 |

**智能回退**：如果当前周期数据太少（周不足3天、月不足7天），系统会自动使用上一个完整周期。

### 5.3 如何生成洞察

**方式一：在记忆长廊中生成**（推荐）

1. 点击首页底部导航栏的"记忆长廊"按钮
2. 选择「回放」Tab
3. 在 HeroCard 中切换到"周"或"月"
4. 点击「生成本周回放」或「生成本月回放」按钮
5. 等待 AI 生成（通常 5~15 秒）

**方式二：在 AI 对话中生成**

1. 打开 AI 对话
2. 输入"生成本周总结"或"复盘这个月"
3. AI 会在后台生成洞察，并回复确认信息

**方式三：系统自动生成**（需开启设置，详见第八章）

### 5.4 洞察卡片的 8 种类型

每份洞察报告包含 1~8 张卡片，每张卡片聚焦一个维度：

| 卡片类型 | 图标 | 颜色 | 内容说明 |
|---------|------|------|---------|
| **习惯** | 跑步人形 | 绿色 | 习惯完成率、连续打卡、挣扎习惯 |
| **财务** | 人民币符号 | 橙色 | 消费总结、Top 分类、预算执行 |
| **任务** | 勾选圆圈 | 蓝色 | 任务完成情况、逾期统计、高优先级进度 |
| **观点** | 对话气泡 | 蓝色 | 情绪分布、标签、想法摘要 |
| **里程碑** | 旗帜（实心） | 红色 | 连续打卡 N 天、累计记账 N 笔等重要里程碑 |
| **跨域关联** | 循环箭头 | 橙色 | 发现跨模块的隐藏关联（如习惯↔消费） |
| **总览** | 柱状图 | 灰色 | 本周期整体状态概览 |
| **异常** | 感叹号 | 蓝/橙/红 | 异常情况预警，按严重度分色 |

### 5.5 异常检测卡片

异常卡片会自动检测以下情况，并按严重度分级显示：

| 异常类型 | 触发条件 | 严重度 |
|---------|---------|--------|
| **消费飙升** | 单日支出 > 日均 2 倍且 > 100 元 | 🟠 warning（2~5倍）/ 🔴 critical（>5倍） |
| **总预算超支** | 总预算使用率 ≥ 100% | 🔴 critical |
| **分类预算预警** | 分类预算使用率 ≥ 80% | 🟠 warning |
| **分类预算超支** | 分类预算使用率 ≥ 100% | 🔴 critical |
| **习惯断裂** | 连续 ≥3 天未完成每日打卡习惯 | 🟠 warning |
| **任务过载** | 活跃任务 ≥10 且逾期 ≥3 | 🟠 warning（3~5）/ 🔴 critical（>5） |

### 5.6 跨域关联卡片

系统会自动检测 6 种跨域关联，并在洞察报告中展示：

| 关联类型 | 发现什么 |
|---------|---------|
| 习惯 ↔ 财务 | "习惯完成率下降的同时外卖支出上升" |
| 任务 ↔ 财务 | "逾期任务增多伴随支出上升" |
| 情绪 ↔ 习惯 | "负面情绪增多时习惯完成率下降" |
| 任务 ↔ 习惯 | "任务和习惯完成率同频波动" |
| 情绪 ↔ 消费 | "负面情绪期间支出突增" |
| 工作日/周末模式 | "工作日和周末消费模式差异显著" |

### 5.7 上期回顾

洞察报告会自动注入上一期的内容，形成连续性：
- **上期建议**：AI 上次给出的追问建议（最多3条）
- **上期异常**：上期报告中的异常卡片标题
- **上期摘要**：上期报告的摘要（前160字）

这让你可以追踪问题的改善情况。

### 5.8 洞察的 6 种状态

HeroCard 有 6 种显示状态：

| 状态 | 显示内容 | 可执行操作 |
|------|---------|-----------|
| **未配置** | "配置 AI 后可以生成个性化的周/月回放" | 「去配置 AI」 |
| **空闲** | 兜底文案或"已有记录，可以让 AI 帮你整理成回放" | 「生成本周/月回放」 |
| **生成中** | 进度条 + "AI 正在阅读你的记账、习惯、任务和观点" | 无（按钮禁用） |
| **已完成** | 洞察标题 + 摘要 + 卡片列表 | 「继续问 AI」+「重新生成」 |
| **有新记录** | 已有洞察标题 + 蓝色"有新记录"标签 | 「刷新洞察」+「继续问 AI」 |
| **失败** | 错误信息 | 「重试」 |

### 5.9 洞察缓存与去重

- **Hash 缓存**：如果你的数据没有变化（快照 hash 一致），系统不会重复生成
- **Jaccard 去重**：新洞察与近期3条相似度 > 85% 时自动跳过
- **Stale 标记**：数据变化导致 hash 不同时，标记为"有新记录"，提示你刷新
- **强制刷新**：在对话中输入"重新生成本周总结"可强制重新生成

### 5.10 规则兜底

即使未配置 AI 或生成失败，回放 Tab 也会展示基于规则生成的兜底文案：
- 根据"习惯趋势 × 消费趋势 × 是否有想法"组合，生成 18 种场景文案
- 确保你在任何情况下都能看到有意义的总结

### 5.11 记忆长廊的完整结构

记忆长廊有三个 Tab：

**回放 Tab**：
- HeroCard（AI 洞察主卡片，支持周/月切换）
- 今日展柜（今日数据速览）
- 最近 7 天封面流

**地图 Tab**：
- 全量记忆数 + 记录天数 + 洞察数
- 13 周热力图（点击某天查看当日统计：消费/习惯/任务/观点）

**明细 Tab**：
- 5 种筛选标签：全部 / 记账 / 习惯 / 待办 / 观点
- 时间线列表：日摘要 → 里程碑 → 高亮事件
- 点击某条记忆可查看详情

## 七、智能分类学习

### 7.1 功能意义

AI 在帮你记账时会自动匹配分类科目。但有时你的表述不在系统预设的分类中（如"家政"）。智能分类学习让系统记住你的纠正，下次自动应用，越用越准。

### 7.2 完整工作流程

**场景示例："家政花了200"**

**第一次使用**：
1. 你说"家政花了200"
2. AI 提取意图：记支出，金额 200，分类候选"家政"
3. 系统在分类表中查找"家政"：精确匹配 ❌ → 同义词匹配 ❌ → 学习映射 ❌（首次）→ 模糊匹配 ❌
4. 兜底：归入"待确认"分类
5. AI 回复："已记录支出 ¥200。⚠️ 无法识别分类「家政」，已暂归「待确认」，点击卡片可修改"
6. 你点击卡片，手动选择"购物/日用"
7. 系统学习映射：`家政 → 购物/日用`

**第二次使用**：
1. 你说"家政300"
2. 系统查找"家政"：精确匹配 ❌ → 同义词匹配 ❌ → **学习映射 ✅**（confidence 0.9）
3. 自动分类为"购物/日用"
4. AI 回复："已记录支出 ¥300（家政） → 现金"

### 7.3 四级分类匹配策略

系统按以下优先级依次尝试，命中即返回：

| 优先级 | 策略 | 置信度 | 说明 |
|--------|------|--------|------|
| 1 | 精确匹配 | 1.0 | 二级分类名完全一致（忽略大小写和空格） |
| 2 | 同义词匹配 | 0.95 | 系统内置约 100+ 个同义词映射 |
| 3 | 用户学习映射 | 0.9 | 你之前纠正过的分类对应关系 |
| 4 | 模糊匹配 | 0.6~ | Levenshtein 编辑距离算法，取 Top 5 候选 |
| — | 无匹配 | 0 | 归入"待确认"，暂存候选词供后续学习 |

### 7.4 覆盖的分类科目

系统预设了完整的二级科目体系：

**支出（9 大一级分类）**：

| 娱乐 | 电影、游戏、视频、音乐、KTV、旅游、健身 |
| 居住 | 房租、房贷、水费、电费、燃气、物业、网费、家电、装修 |

## 八、被动交互 — 系统自动运行

HoloAI 有一系列被动交互功能，在后台自动运行，无需你手动触发。

### 8.1 后台自动生成洞察

**意义**：系统在后台自动生成日/周/月洞察报告，你打开记忆长廊时即可直接查看。

**如何开启**：
1. 进入设置页（首页右上角人像 → 设置）
2. 在「AI 回放」区块找到「后台自动生成」
3. 打开开关

**触发条件**：
- 后台任务：iOS 系统调度 `BGAppRefreshTask`，最早 1 小时后执行（iOS 不保证准时执行）
- 前台补偿：如果你打开 APP 时发现后台任务未执行，系统会自动补生成

**生成逻辑**：
- 日报：当天的数据，已存在且状态为 ready 则跳过
- 周报：智能计算周期范围（不足 3 天自动回退到上周）
- 月报：智能计算周期范围（不足 7 天自动回退到上月）

### 8.2 定时通知提醒

**意义**：在指定时间推送本地通知，提醒你查看周期洞察。

**如何配置**：

| 配置项 | 路径 | 默认值 |
|--------|------|--------|
| 每周提醒 | 设置 → AI 回放 → 「每周提醒」 | 关闭（默认周一 9:00） |
| 每月提醒 | 设置 → AI 回放 → 「每月提醒」 | 关闭（默认每月 1 日 9:00） |

**详细配置选项**：

| 通知类型 | 可配置项 |
|---------|---------|
| 周报提醒 | 开关 + 星期几（周一到周日）+ 时间（整点） |
| 月报提醒 | 开关 + 每月几号 + 时间（整点） |

**注意**：
- 首次开启会请求通知权限（`.alert` + `.sound`）
- 如果通知权限被拒绝，开关会自动回退关闭
- 通知内容："本周记忆已准备好" / "本月记忆已准备好"

### 8.3 自动异常检测

**意义**：在生成洞察时自动检测异常情况，确保你不会错过重要问题。

异常检测在生成洞察时自动运行，无需额外配置。检测以下 5 种异常：

| 异常 | 触发条件 | 严重度 |
|------|---------|--------|
| 消费飙升 | 单日支出 ≥ 日均 2 倍且 > ¥100 | 🟡 warning / 🔴 critical |
| 总预算超支 | 总预算使用率 ≥ 100% | 🔴 critical |
| 分类预算预警 | 分类预算使用率 ≥ 80% | 🟡 warning |
| 习惯断裂 | 连续 ≥3 天未完成每日打卡习惯 | 🟡 warning |
| 任务过载 | 活跃任务 ≥10 且逾期 ≥3 | 🟡 warning / 🔴 critical |

### 8.4 自动跨域关联

**意义**：发现你生活中不同模块之间的隐藏关联。

在生成洞察时自动运行 6 条跨域关联规则：

| 关联规则 | 发现什么 | 触发条件 |
|---------|---------|---------|
| 习惯 ↔ 财务 | 习惯完成率下降 + 外卖支出上升 | 习惯完成率降 >10% 或餐饮占比升 >5pp |
| 任务 ↔ 财务 | 逾期任务增多 + 支出上升 | 逾期率 >20% 且支出环比升 >15% |
| 情绪 ↔ 习惯 | 负面情绪增多 + 习惯完成率下降 | 负面情绪 >40% 且完成率降 >10% |
| 任务 ↔ 习惯 | 两者完成率同频波动 | 两者都 <50% 或都 >70%，且差距 <30pp |
| 情绪 ↔ 消费 | 负面情绪 + 支出突增 | 负面情绪 >40% 且支出环比升 >10% |
| 工作日/周末 | 消费模式差异 | 日均消费比 >1.5 倍 |

### 8.5 自动去重与缓存

**意义**：避免重复生成相同内容的洞察，节省 API 调用和时间。

| 机制 | 说明 |
|------|------|
| **Snapshot Hash** | 对完整数据计算 SHA256 hash，数据没变就不重新生成 |
| **Jaccard 去重** | 新洞察与近期 3 条相似度 >85% 时自动跳过 |
| **Stale 标记** | 数据变化后标记"有新记录"，提示你刷新 |

### 8.6 高亮检测

在生成洞察时自动检测以下高亮事件：

| 高亮类型 | 触发条件 |
|---------|---------|
| 连续打卡成就 | 习惯连续打卡达到 3/7/14/21 天 |
| 消费异常 | 当日消费 > 7 日日均 × 1.5 |
| 习惯全勤日 | 当天所有活跃习惯全部完成 |
| 重要任务完成 | 完成优先级 ≥ 高的任务 |

### 8.7 里程碑检测

自动检测并记录以下里程碑：

| 里程碑类型 | 阈值 |
|-----------|------|

## 十、常见问题

### Q1：AI 回复"请先设置默认账户"怎么办？

记账需要指定一个默认账户。请前往"设置 → 账户管理"，创建至少一个账户并将其设为默认。

### Q2：AI 回复"AI 服务尚未配置"怎么办？

你需要先配置 AI 服务商和 API Key。请参考第二章"快速开始"。

### Q3：为什么洞察生成失败？

可能的原因：
- API Key 无效或余额不足
- 网络连接问题
- AI 服务商暂时不可用

建议：点击"重试"按钮，或检查 AI 配置中的"测试连接"。

### Q4：洞察显示"有新记录"是什么意思？

你上次生成洞察后，又新增了记账、打卡等数据。系统检测到数据变化，建议你刷新洞察。点击「刷新洞察」即可。

### Q5：AI 记不住我上次说了什么？

当前版本每次对话独立，没有跨对话记忆。你可以通过编辑 HoloProfile（设置 → 个人档案）写入 AI 需要长期记住的信息。

### Q6：为什么后台自动生成没有按时执行？

iOS 的 `BGAppRefreshTask` 由系统调度，不保证准时执行。系统会在你打开 APP 时检查并补偿生成。确保已开启"后台自动生成"开关。

### Q7：分类被 AI 识别错了怎么办？

点击消息中的交易卡片，手动修改分类。系统会自动学习你的纠正，下次不会再错。

### Q8：通知权限被拒绝了怎么办？

请前往 iOS 系统设置 → Holo → 通知，手动开启通知权限。开启后重新在 APP 中打开提醒开关。

### Q9：可以同时使用多个 AI 服务商吗？

当前版本同一时间只能配置一个 AI 服务商。你可以在 AI 配置页面随时切换。

### Q10：自定义 Prompt 模板后效果变差了怎么办？

在 Prompt 编辑器中，可以通过"恢复默认"功能重置为系统预设模板。版本过低的模板会自动回退到默认值。
