# 💭 观点模块 - 功能拆解与任务清单

> 本文档基于 PRD 和详细设计，将「观点」模块拆解为可执行的任务清单，细化到每个数据模型、每个 UI 组件、每个交互细节。

---

## 一、需求确认汇总

### 1.1 核心决策

| 维度 | 决策 |
|------|------|
| **入口位置** | 首页「观点」模块（已有按钮占位，id="thoughts"） |
| **数据存储** | 共享 Core Data + iCloud 同步 |
| **AI 能力** | MVP 暂不做，后续迭代 |
| **MVP 范围** | 核心可用：记录 + 标签 + 心情 + 图片 + 思维链 |
| **编辑策略** | 可编辑可删除，删除时静默断开引用链 |

### 1.2 交互细节确认

| 维度 | 决策 |
|------|------|
| **新增入口** | 底部 Sheet 编辑器 |
| **引用交互** | 内嵌引用卡片 + 底部关联区域（两者结合） |
| **心情模板** | 独立装饰字段，不影响筛选和可视化 |
| **标签限制** | MVP 不限制数量，后续根据使用数据调整 |
| **列表视图** | 默认最新在前，可按标签筛选（不按日分组） |
| **删除影响** | 确认后静默断开引用链，无占位符 |
| **富文本** | 精简模式：加粗、斜体、列表（无代码块/引用块） |
| **图片支持** | MVP 支持（拍照/相册） |
| **卡片信息** | 预览模式：2-3 行内容 + 标签 + 心情 + 日期 + 引用数 |
| **引用数量** | 无限制 |
| **搜索能力** | 多维搜索：内容、标签、心情、日期范围、排序 |

## 二、功能架构拆解

### 2.1 模块层级结构

```
观点模块 (Thoughts)
├── 列表视图层 (ThoughtListView)
│   ├── 底部 Sheet 新增入口
│   ├── 标签筛选栏
│   └── 无限滚动列表
│
├── 编辑器层 (ThoughtEditor)
│   ├── 精简富文本工具栏
│   ├── 图片插入
│   ├── 标签输入
│   ├── 心情选择器
│   └── 引用旧想法
│
├── 详情页层 (ThoughtDetailView)
│   ├── 完整内容展示
│   ├── 内嵌引用卡片
│   ├── 底部关联区域（反向链接）
│   └── 编辑/删除操作
│
└── 数据层 (ThoughtRepository)
    ├── Core Data 实体
    ├── 增删改查方法
    └── 引用关系管理
```

## 三、数据模型设计

### 3.1 Core Data 实体定义

#### Thought 实体

| 属性名 | 类型 | 说明 | 可选 |
|--------|------|------|------|
| `id` | UUID | 唯一标识符 | ❌ |
| `content` | String | 富文本内容（Markdown 格式） | ❌ |
| `createdAt` | Date | 创建时间 | ❌ |
| `updatedAt` | Date | 最后更新时间 | ❌ |
| `mood` | String | 心情值：happy/sad/angry/calm/thinking/inspired | ✅ |
| `orderIndex` | Int16 | 排序索引（用于自定义排序） | ✅ |
| `imageData` | Binary Data | 图片数据（可选，存储压缩后的图片） | ✅ |
| `isDeleted` | Boolean | 软删除标记（默认 false） | ✅ |

#### Thought 关系

| 关系名 | 目标实体 | 类型 | 说明 |
|--------|----------|------|------|
| `tags` | ThoughtTag | To-Many | 标签关联 |
| `references` | ThoughtReference | To-Many | 引用的其他想法（正向链接） |
| `referencedBy` | ThoughtReference | To-Many | 引用本想法的其他想法（反向链接） |

#### ThoughtTag 实体（标签）

| 属性名 | 类型 | 说明 | 可选 |
|--------|------|------|------|
| `id` | UUID | 唯一标识符 | ❌ |
| `name` | String | 标签名称（如 "产品思考"） | ❌ |
| `color` | String | 标签颜色值（16 进制） | ✅ |
| `usageCount` | Int16 | 使用次数（用于排序） | ✅ |

| 关系名 | 目标实体 | 类型 | 说明 |
|--------|----------|------|------|
| `thoughts` | Thought | To-Many | 使用该标签的想法列表 |

#### ThoughtReference 实体（引用关系）

| 属性名 | 类型 | 说明 | 可选 |
|--------|------|------|------|
| `id` | UUID | 唯一标识符 | ❌ |
| `createdAt` | Date | 创建时间 | ❌ |

| 关系名 | 目标实体 | 类型 | 说明 |
|--------|----------|------|------|
| `sourceThought` | Thought | To-One | 引用发起方（谁引用了别人） |
| `targetThought` | Thought | To-One | 被引用方（谁被别人引用） |

**设计说明**：
- 使用独立的 `ThoughtReference` 实体而非 UUID 数组，便于 Core Data 管理和反向链接查询
- `sourceThought` → `targetThought` 表示 "source 引用了 target"
- 删除 target 时，自动删除所有指向它的 ThoughtReference 记录
