# HoloAI 本周观察阻塞态治理设计

日期：2026-07-09
状态：待实施

## 目标

修复 HoloAI 顶部「本周观察」失败卡片中关闭与重试无反馈的问题，并让生成失败可诊断、可恢复、不阻塞聊天。

成功标准：

- 点击关闭后卡片立即消失，当天不再出现；新生成的未读观察仍可重新出现。
- 点击重试后立即显示生成中状态并禁用重复点击。
- 重试成功后原地显示新观察；失败后恢复可重试状态，并显示用户可理解的错误。
- 空响应、非法 JSON、Schema 不合格不再统一显示为含糊的「JSON 解析失败」。
- 卡片任何失败都不影响消息列表、快捷栏和输入框交互。

## 已确认根因

`ChatView` 在关闭和重试后递增 `weeklyCardVersion`，但视图树没有读取该状态，不能可靠触发 `WeeklyObservationCard.shouldDisplay` 重新计算。

重试链路还存在两个放大问题：

- `try?` 吞掉生成错误，失败后没有显式状态更新。
- 卡片没有重试中状态，用户看不到点击反馈，也可以重复触发请求。

截图中的「JSON 解析失败：」冒号后为空，说明空响应被错误归类为普通 JSON 失败，导致真正故障层级不可见。

## 方案

### 1. 卡片展示与操作状态

在 `ChatView` 维护明确的本地卡片状态：

- `isWeeklyCardDismissed`：本次页面生命周期内立即隐藏。
- `isRetryingWeeklyObservation`：控制 loading、禁用重复点击。
- `weeklyObservationError`：保存本次重试的可展示错误。
- `weeklyCardVersion`：仅在确需重新读取持久化观察时作为显式刷新标识，并由视图读取。

展示条件为：

1. 本地已关闭时立即不显示。
2. 否则读取持久化展示决策。
3. 生成出新的未读观察时，允许清除本地关闭并重新展示。

关闭动作先更新本地状态，再写入当天隐藏标记。即使持久化写入异常，当前页面也不会继续阻塞。

### 2. 重试状态机

重试行为按以下顺序执行：

1. 若正在重试，忽略重复点击。
2. 立即进入 `retrying` 并清除旧错误。
3. 刷新有效记录日并请求强制生成。
4. 成功后刷新卡片数据，退出 `retrying`。
5. 失败后保留失败卡片，退出 `retrying`，展示分类后的错误。

所有状态切换在主线程执行。取消任务不显示为业务失败。

### 3. 解析错误分类

解析器提供可诊断结果，而不是只返回 `nil`：

- `emptyResponse`：响应为空或只有空白。
- `invalidJSON`：无法提取或解码 JSON。
- `invalidSchema`：JSON 可解码但不符合洞察约束。

服务层保存简短用户文案，同时在调试日志保留安全截断后的原始响应和具体解码原因。UI 不展示模型原始内容，避免噪声和潜在隐私泄漏。

### 4. 同类阻塞风险扫描

扫描 HoloAI 页面内满足以下组合的交互入口：

- 用户主动点击后启动异步任务；
- 使用 `try?` 或空 `catch`；
- 没有 loading、失败反馈或防重复点击。

只修复会阻断主要交互或让按钮表现为无效的高风险入口；普通后台刷新不在本次范围，避免无关重构。

## 测试

先写失败测试，再写生产代码：

- 关闭后展示决策立即为 false。
- 重试期间重复点击不会发起第二个任务。
- 重试成功后退出 loading 并触发数据刷新。
- 重试失败后退出 loading 且保留可重试状态。
- 空字符串和纯空白响应返回 `emptyResponse`。
- 非法 JSON 与 Schema 不合格可被区分。

最后执行相关单元测试、Holo iOS Debug 构建，并检查未提交改动范围，确保不包含现有记忆长廊工作。

## 非目标

- 不调整本周观察卡片视觉样式。
- 不改变本周观察生成周期、配额或 Prompt。
- 不提交或修改当前工作区已有的记忆长廊行动按钮改动。
- 除非排查证据确认生产后端返回异常，否则不改后端；若涉及后端代码，必须单独部署并做线上验证。
