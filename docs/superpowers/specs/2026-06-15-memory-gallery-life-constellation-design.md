# 记忆长廊洞察区「生活星图」改版设计

日期：2026-06-15
状态：待实现

## 背景

当前记忆长廊洞察区的问题不是单纯“不好看”，而是叙事层级混在一起：

- Agent 深度分析结果像一张独立卡片被插在页面里，用户不容易理解它为什么出现在记忆长廊。
- AI 回放、Daily Sense、里程碑、高光、Agent 结果互相并列，缺少一个统一的“这段时间发生了什么”的主叙事。
- 里程碑与高光里存在偏刺眼的冷数据提醒，例如“今日消费比日均高 XX%”，容易让用户感到被评判，而不是被理解。
- 健康数据即将接入 Agent，但前端目前还没有稳定的健康入口和状态承接。

本次改版目标是把洞察区从“AI 生成结果列表”改为“生活变化地图”：用户先看到 Holo 对近期生活信号的整体观察，再按需展开回放、证据和模块详情。

## 产品目标

1. **让记忆长廊更像记忆长廊**
   首屏不是数据看板，也不是报告列表，而是一张能表达生活信号关系的“生活星图”。

2. **让 Agent 成为解释能力，而不是突兀卡片**
   Agent 深度分析默认折叠，作为证据与继续追问入口，不直接抢第一视觉。

3. **让五个模块同级出现**
   习惯、财务、任务、思考、健康都作为星图中的信号星出现。健康未授权或未完成 Agent 接入时，也保留明确位置。

4. **让里程碑/高光变温和**
   不再用“消费超过 XX%”这类刺眼提醒作为主内容。它们应成为“可回看的片段”，偏观察、激励、复盘，而不是警报。

## 页面结构

洞察 Tab 分为四层：

### 1. 生活星图首屏

替代当前 AI 回放 Hero + Agent 深度分析卡片的堆叠结构。

内容包括：

- 当前周期：本周、本月、本季度、自定义周期。
- 中心主线：一句话总结近期生活状态，例如“节奏变晚，但仍在恢复”。
- 五颗信号星：
  - 习惯
  - 财务
  - 任务
  - 思考
  - 健康
- 星星之间的弱连线：表达“这些信号可能共同构成一个生活状态”，不表达确定因果。

视觉规则：

- Light Mode：纸质星图。米白纸张、细线连接、柔和星点、无发光效果。
- Dark Mode：夜空星图。深色背景、发光星点、弱连线。
- 两个模式共享同一信息结构，只替换材质语言。

### 2. 模块解释卡

点击任意信号星后，星图下方的解释卡切换到对应模块。

示例：

- 习惯：最近完成节奏、断裂/恢复迹象。
- 财务：近期支出节奏、外出日、预算压力提醒。
- 任务：完成/拖延/收尾状态。
- 思考：观点记录时间、主题变化、夜间集中度。
- 健康：授权状态、睡眠、步数、站立、运动恢复入口。

交互规则：

- 第一版不弹半屏 Sheet，避免打断星图体验。
- 点击星星只切换下方解释卡。
- 解释卡里再提供二级动作：打开模块页、继续问 AI、查看证据状态。

### 3. 折叠解释层

AI 回放和深度分析都在星图下方，默认折叠。

AI 回放：

- 负责把星图讲成一段周期故事。
- 继续支持周期切换、重新生成、继续问 AI。
- 后续可逐步接入 Agent 结果，但不把 Agent 原始结果直接塞到首屏。

深度分析：

- 负责展示证据、分析状态和继续追问入口。
- 无 Agent 结果时显示“暂无证据化分析”。
- 有 Agent 结果时展示已验证结论和证据来源。
- 健康 Agent 接入前，健康证据状态显示为“接入中”或“暂未纳入证据化分析”。

### 4. 可回看的片段

旧“里程碑与高光”改名为“可回看的片段”。

它不再承担“重大成就/异常警报”的主叙事，而是作为星图下方的素材层：

- 一次重要任务完成。
- 一段习惯恢复期。
- 一次外出日支出峰值。
- 一次夜间观点集中记录。
- 一个健康数据授权或恢复观察点。

## 健康入口与状态

健康是第五颗信号星，必须与习惯、财务、任务、思考同级。

状态分三类：

1. **未授权**
   - 星点使用虚线样式。
   - 文案：`等待健康数据`
   - 动作：`打开健康` / `去授权`

2. **接入中**
   - 星点使用虚线或半实心样式。
   - 文案：`健康证据接入中`
   - 说明：健康会用于睡眠、步数、站立和运动恢复分析，但暂未纳入 Agent 证据化结论。

3. **已接入**
   - 星点使用实心样式。
   - 文案根据健康状态生成，例如 `睡眠偏少`、`步数恢复`、`运动恢复稳定`。
   - 深度分析折叠区展示健康证据状态。

## 里程碑/高光内容准则

### 命名

原名称：`里程碑与高光`

新名称：`可回看的片段`

备选名称：

- `这一段值得回看`
- `本周留下的片段`
- `近期片段`

第一版建议使用：`可回看的片段`。

### 文案原则

不再使用审判感、警报感、冷数字冲击感强的表达。

避免：

- `今日消费比日均高 80%`
- `消费异常`
- `超过预算太多`
- `习惯失败`
- `任务拖延严重`

改为：

- `这天的外出支出比较集中`
- `这周有一天的花费明显更像外出日`
- `预算节奏需要留意一下`
- `习惯节奏出现短暂停顿`
- `任务收尾比计划慢了一点`

### 财务提醒的语气

财务片段应从“异常警告”改成“节奏观察”。

示例映射：

| 旧表达 | 新表达 |
| --- | --- |
| 今日消费比日均高 80% | 这天的外出支出比较集中 |
| 消费异常 | 花费节奏有一个小高点 |
| 支出超过日均 | 这天更像一次集中消费日 |
| 预算压力较大 | 预算节奏值得稍微留意 |

只有在用户主动查看详情或证据时，才展示具体金额、比例和对比基线。

### 激励与观察

“片段”应该优先记录：

- 状态变化：从断裂到恢复、从分散到集中。
- 阶段门槛：连续性、完成度、稳定主题。
- 用户相关性：和用户近期关注目标有关。
- 可证据化：背后有真实数据支撑。

不应把任何统计异常都包装成高光。

## 数据与兼容策略

第一版尽量复用现有数据，控制实现范围。

可复用：

- `MemoryGalleryViewModel` 当前周期、回放状态、Agent 渲染结果。
- `MemoryInsight` 当前 AI 回放。
- `DailySenseSnapshot` 的健康、习惯、财务等状态信号。
- `HighlightDetector` / `MilestoneDetector` 的原始素材，但需要重新筛选和重写展示文案。
- 现有 `HealthView` 作为健康星的落点。

后续接入：

- Agent Health Tool 注册后，健康星从“接入中”切换为“已接入”。
- 深度分析折叠区展示健康证据状态。
- 星图主线可逐步由 Agent 结果驱动，而不是仅由旧 AI 回放或规则摘要驱动。

## 空态与降级

无 AI 回放：

- 星图仍显示，但中心主线使用规则兜底摘要。
- AI 回放折叠区显示“还没有生成本周期回放”。

无 Agent 结果：

- 深度分析折叠区显示“暂无证据化分析”。
- 不在首屏展示空 Agent 卡片。

健康未授权：

- 健康星保留，但为虚线状态。
- 点击后解释卡说明“授权后可把睡眠、步数和运动恢复纳入生活星图”。

数据很少：

- 星图可弱化连线，避免暗示不存在的关联。
- 中心主线使用保守文案：`记录还不多，先从几个生活信号开始`。

## 测试与验收

验收重点：

- 洞察 Tab 首屏不再出现突兀的独立深度分析卡。
- AI 回放与深度分析默认折叠。
- 健康作为第五颗星稳定出现，不因未授权或 Agent 未接入而消失。
- Light/Dark Mode 都能清楚展示星图，文字不重叠。
- 点击五颗星能切换解释卡。
- “可回看的片段”不再使用刺眼的消费百分比主文案。
- 具体金额和百分比只在详情或证据层展示。

建议验证：

- Swift 编译通过。
- 记忆长廊洞察 Tab 在 Light/Dark Mode 截图检查。
- 健康三态截图：未授权、接入中、已接入状态。
- 财务片段文案检查，确认不出现“消费异常”“高 XX%”作为主文案。
