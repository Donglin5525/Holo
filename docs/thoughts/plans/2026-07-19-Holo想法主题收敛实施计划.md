# Holo 想法模块 · 主题收敛实施计划（Review 修正版）

> 对应 PRD：`docs/thoughts/plans/2026-07-19-Holo想法主题收敛PRD.md`

## Phase 1：领域契约与兼容语义

### 1.1 Topic 状态

- 为 `Topic.TopicStatus` 增加 `classification`。
- 所有“可见主题”口径纳入 `classification`。
- 新增 `fetchClassificationTopics`、启用/停用分类主题和批量初始化方法。
- “未归类”保持虚拟节点，不创建特殊 Topic。

### 1.2 主题本地变更

- 改名：Topic 标题与整棵标签路径前缀同步。
- 合并：关系、来源标签、标签路径和状态同步；目标保持 classification。
- 删除：分类关系解除，原路径降级到 `未分类/`，不删除手动标签。
- 新分类结果只替换 classification Topic 关系，不破坏历史 Topic。

### 1.3 纯逻辑安全层

新增可 standalone 测试的 `ThoughtThemeConstraint`：

- 主题名归一化匹配；
- 未知主题降级；
- 子标签去前缀、去空、去重、限长；
- 最终路径只能以合法主题或 `未分类` 开头。

## Phase 2：AI 分类链路

### 2.1 Prompt 双端同步

- iOS `PromptManager.swift`：`thoughtOrganization` 升 v3。
- 后端 `defaultPrompts.json`：同步结构化输入/输出契约。
- 后端 `promptRegistry.js`：登记 `thought_organization: 3`。
- Prompt 明确把用户正文和列表当数据，不服从其中指令。

### 2.2 ThoughtOrganizationService

- 读取 classification Topic 列表。
- 发送 JSON 数据：`activeTopics`、`existingTags`、`rejectedTags`、`thoughtContent`。
- 解析 `selectedTopic + suggestedTags`。
- 通过纯逻辑安全层生成最终路径。
- 原子替换未确认 AI 标签，并写入单一分类主题关系。

## Phase 3：主题发现复用

- `fetchConvergenceCandidates` 只返回没有 classification Topic 的想法。
- 收敛 Prompt 的现有主题只使用 classification Topic。
- 用户确认新建议时，把新主题设为 classification。
- 队列清空且未归类达到 5 条时触发一次建议生成；签名去重继续避免重复请求。
- 建议数组与输入签名一并本地持久化，逐条处理后同步扣除，支持冷启动恢复。
- UI 不再 `autoApply`，任何新主题都需要用户确认。

## Phase 4：用户界面

### 4.1 Onboarding

- 三页升级为四页：欢迎 → 能力 → 主题 → AI 授权。
- 新增 `OnboardingTopicSetupPage`，默认 4 个预设，允许增删自定义。
- 继续时本地创建 classification Topic；整体跳过不写入。

### 4.2 主题管理

- 新增 `TopicManagementView`。
- 支持启用/停用、创建、改名、合并、删除、重新整理未归类。
- 设置页与知识树都能进入。

### 4.3 知识树

- 顶层展示 classification Topic 与固定未归类。
- 主题内按路径展示子标签。
- 兼容旧标签放入“个人与历史标签”，不丢数据。
- 未归类有建议时显示角标，入口打开现有确认页。

## Phase 5：验证

- standalone：主题校验、幻觉主题、全角路径、重复标签、空标签、无主题池。
- standalone：待确认建议编码、冷启动恢复与处理完成清理。
- Core Data：classification 可见性、未归类口径、关系单选替换、改名/合并/删除。
- 后端：Prompt 注入契约和版本。
- iOS：完整 Simulator build。
- 真机：Onboarding、老用户启用、打标、未归类、管理操作、洞察确认。

## 发布边界

本次包含 `HoloBackend/` Prompt 与版本改动。代码在本地通过不代表线上生效；必须部署后端并验证生产 Prompt 版本，才可完成线上验收。
