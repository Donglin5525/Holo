# 今日看板优化方案

## Context
今日看板存在多处 UI 和逻辑问题：首页入口按钮显示丑陋的百分比文字、标题未对齐、健康数据位置不合理、打卡逻辑不区分习惯类型（计数类/测量类当成了简单打卡）、待办未同步未完成任务、心情标签太少且不支持自定义。

---

## 修改范围

### 1. 首页入口按钮优化
**文件**: `Components/DailyKanbanEntryButton.swift`

- 移除进度环内部的 `\(Int(overallPercent * 100))%` 文字和 `+ Text("%")` 拼接
- 进度环内保持空白，仅显示进度弧线
- 保留下方两个小标签 `habitProgress` 和 `kanbanProgress`
- 移除按钮下方的 `Text("今日看板")` 文字标签

### 2. 看板标题对齐
**文件**: `Views/DailyKanban/DailyKanbanView.swift`

- `headerView` 中 "今日看板" 文字使用 `ZStack` + `.frame(maxWidth: .infinity)` 覆盖居中，而非依赖 `HStack` 的 Spacer，确保标题在视觉上严格居中不受左右控件宽度影响

### 3. 健康数据移至最底部
**文件**: `Views/DailyKanban/DailyKanbanView.swift`

- 调整 `ScrollView` 内卡片顺序：Hero → Budget → Habit → Task → Mood → **Health**
- 将 `KanbanHealthSection` 移到 `KanbanMoodSection` 之后

### 4. 打卡板块支持多种习惯类型
**文件**: `Views/DailyKanban/KanbanHabitSection.swift`

当前只展示打卡型习惯的 toggle 按钮，需要根据习惯类型渲染不同交互：

| 习惯类型 | 当前行为 | 修改后行为 |
|---------|---------|-----------|
| 打卡型 (checkIn) | toggle 勾选 | 保持不变 |
| 计数类 (numeric/sum) | toggle 勾选（错误） | 显示今日累计值 + "+1" 按钮，复用 `HabitRepository.incrementCount()` |
| 测量类 (numeric/latest) | toggle 勾选（错误） | 显示 "记录" 按钮，点击弹出数值输入 Sheet，复用 `HabitRepository.addNumericRecord()` |

实现要点：
- 移除 `activeHabits` 的 `isCheckInType` 过滤，展示所有每日频率的活跃习惯
- `habitRow` 根据 `habit.isCheckInType` / `habit.isCountType` / `habit.isMeasureType` 分发不同行视图
- 新增 `@State private var todayValues: [UUID: Double]` 存储数值型习惯的今日值
- 新增 `@State private var showValueInput: Bool` 和 `editingHabit` 管理测量类输入弹窗
- 弹窗 UI 复用 `HabitQuickCheckInView` 中的 `valueInputSheet` 模式
- `loadStatus()` 需额外加载数值型习惯的今日值（`habitRepo.getTodayValue(for:)`）

### 5. 今日待办支持同步未完成任务
**文件**: `Views/DailyKanban/KanbanTaskSection.swift` + `Models/TodoRepository+Kanban.swift`

**查询扩展** - `TodoRepository+Kanban.swift`:
- 新增 `getUncompletedRecentTasks(limit:)` 方法：获取未来 3 天内到期的未完成任务（排除已出现在 plannedTasks 和 dueTodayTasks 中的）
- 新增 `getUnplannedOpenTasks(limit:)` 方法：获取无到期日的未完成任务（最近创建的）

**UI 调整** - `KanbanTaskSection.swift`:
- 新增"近期待办"子区域，展示未完成的近期任务
- 每个任务行右侧增加"加入今日"按钮，调用 `todoRepo.planTask(task, for: Date())`
- 已加入今日的任务从"近期待办"区域消失（移到"今日待办"区域）

### 6. 心情板块增强
**文件**: `Views/DailyKanban/KanbanMoodSection.swift`

**增加心情标签** - 扩充 `moods` 数组，新增：
- 😴 疲惫 (tired)
- 😰 焦虑 (anxious)
- 🥳 兴奋 (excited)
- 🙏 感恩 (grateful)

（从 6 个扩充到 10 个，分两行 5 个排列）

**支持自定义标签**:
- 新增 `@State private var selectedTags: [String] = []` 和 `@State private var showTagInput: Bool = false`
- 心情选择器下方增加标签行，显示常用标签 chips（如 "工作"、"生活"、"运动"、"社交" 等）
- 每个标签为可选中/取消的 pill 按钮
- 末尾增加 "+" 按钮允许输入自定义标签
- 保存时将标签一并传入 `ThoughtRepository.create()` 的 `tags` 参数

**数据存储**:
- 复用 `ThoughtTag` Core Data 模型，通过 `ThoughtRepository` 管理标签关联
- 标签输入可参考 `TagInputView.swift` 的 UI 模式
